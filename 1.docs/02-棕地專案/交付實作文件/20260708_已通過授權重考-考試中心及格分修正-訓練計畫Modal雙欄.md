## 二、背景與需求摘要

### 2.1 現況問題

| 問題 | 現況 | 正確做法 |
|------|------|----------|
| 及格分判斷 | `ExamDashboard.tsx` 硬編碼 `score >= 60` / `score < 60` | 依訓練計畫 `passing_score`；後端回傳 `is_passed` |
| 已通過重考 | `start_exam` 擋住 `is_passed=true` | 需主管/稽核**授權**後才可重考 |
| 重考紀錄 | `exam_history` 已記每次交卷 | 授權行為需另表；每次交卷持續寫入 history |
| 編輯 Modal | `TrainingPlanManager` 單欄過長 | 教材區移右欄，左右並排 |

### 2.2 業務規則（鐵律）

1. **未通過**：學員可自動重考（既有行為，不變）
2. **已通過**：學員**不可**自行重考；須具權限者授權
3. **一次授權 = 一次重考機會**；學員提交後授權旗標自動清除
4. **重考不需再報到**（沿用第一次報到紀錄）
5. **及格分數**以 `training_plans.passing_score` 為準（各計畫可不同，預設 60）
6. **每次交卷**都寫入 `exam_history`（含逐題 JSON 快照）

### 4.2 `can_start_exam` 計算邏輯（後端唯一真相）

```python
def compute_can_start_exam(
    plan: TrainingPlan,
    record: Optional[ExamRecord],
    today: date,
) -> bool:
    if plan.is_archived:
        return False
    if today < plan.training_date:
        return False  # pending

    has_submitted = record and record.submit_time is not None

    if not has_submitted:
        # 尚未交卷：在開放期間可考
        if plan.end_date and today > plan.end_date:
            return False
        return True

    # 已交卷
    if not record.is_passed:
        return True  # 未通過：自動可重考（可忽略 end_date，沿用現有 start_exam 邏輯）

    if record.retake_authorized:
        return True  # 已通過但已授權重考

    return False  # 已通過且未授權
```


```
Body: { "emp_id", "plan_id" }
```

- `retake_authorized = true`
- 學員**尚未**因本次授權開始重考（可簡化為：`record.start_time` 在授權後未被更新，或檢查是否有未消耗的 authorization 列）

- `exam_records.retake_authorized = False`
- 對應 `exam_retake_authorizations` 列：`revoked_at = now`、`revoked_by = current_user.emp_id`

```python
# 標記對應 authorization 為已消耗

    ExamRetakeAuthorization.record_id == record_id,
    ExamRetakeAuthorization.consumed_at.is_(None),
```

## 六、重考紀錄策略（完整說明）

| 事件 | 寫入位置 | 說明 |
|------|----------|------|
| 每次交卷（含自動重考、授權重考） | `exam_history` | 每次新增一筆；`details` 存逐題快照 JSON |
| 最新成績狀態 | `exam_records` | 覆寫；`attempts` = 提交次數 |
| 主管/稽核授權行為 | `exam_retake_authorizations` | 記錄誰、何時、原因 |
| 授權被使用 | `exam_retake_authorizations.consumed_at` | 學員提交授權重考時填入 |
| 授權被撤銷 | `revoked_at` / `revoked_by` | 管理員撤銷時填入 |

**範例時間軸**：

```
第1次考 55分 FAIL  → exam_history #1, record: FAIL
第2次考 72分 PASS  → exam_history #2, record: PASS
主管授權重考       → authorizations #1, retake_authorized=true
第3次考 80分 PASS  → exam_history #3, record: PASS, 授權消耗
```

**查詢入口（既有，不需重複開發）**：

- 挑戰次數：`exam_history` 筆數（`my_exams.attempts`）
- 考試歷程：成績中心「查看詳情」→ `GET /exam/history/{history_id}`
- 授權稽核：查 `exam_retake_authorizations`（可後續加管理端列表）

#### 8.3.2 表單內容改雙欄

將 `<form>` 內現有欄位與 `PlanMaterialsSection` 拆分：

```tsx
<form onSubmit={handleSubmit} className="flex flex-col flex-1 min-h-0">
  <div className={clsx(
    "flex flex-1 min-h-0 overflow-hidden",
    isEditing && editId ? "flex-col lg:flex-row" : "flex-col"
  )}>
    {/* 左欄：訓練計畫表單 */}
    <div className="flex-1 overflow-y-auto p-6 space-y-4 lg:border-r border-gray-100">

      {/* 不含 PlanMaterialsSection */}
    </div>

    {/* 右欄：教材（僅編輯既有計畫） */}
    {isEditing && editId && (
      <div className="w-full lg:w-[42%] xl:w-[40%] shrink-0 overflow-y-auto p-6 bg-gray-50/50">

  </div>
```
