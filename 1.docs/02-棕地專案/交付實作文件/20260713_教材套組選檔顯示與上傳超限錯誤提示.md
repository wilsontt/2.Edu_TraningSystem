# 教材套組：選檔檔名顯示與上傳超限錯誤提示

**日期**：2026-07-13  
**狀態**：✅ 已實作  
**規格脈絡**：[教材上傳列管與教材庫 PLAN](../plans/已完成/20260617_教材上傳列管與教材庫_PLAN.md) §5.12（三層 min 單檔上限）

---

## 1. 目的

1. 編輯套組（訓練計畫／教材庫）選檔後清楚列出檔名與大小，對齊建立套組體驗並提高可讀性。  
2. 上傳失敗時明確標示觸發原因（教材類型／檔案格式／系統硬上限／單次批次總量），**禁止**僅因 HTTP 200 而顯示「傳輸成功」。

---

## 2. 範圍

| 項目 | 說明 |
|------|------|
| 前端 | `SelectedFilesList`、`MaterialSetEditPanel`、`MaterialSetUploadPanel`、`FileTransferModal` |
| 後端 | `_file_size_limit_exceeded_message`、`_batch_upload_total_exceeded_message`；`create_set` / `add_set_files` |

---

## 3. 行為定案

- 有效單檔上限仍為 `min(類型, 格式, 系統硬上限)`。  
- `POST .../sets/{id}/files` 若全部進 `failed`：前端呼叫 `endTransferError`，不呼叫 `endTransferSuccess`。  
- 錯誤文案需含「觸發」層級與各層參考上限。

---

## 4. 驗收

- [ ] 教材庫編輯套組選檔後可見檔名＋大小（`text-sm`、深色字）。  
- [ ] 訓練計畫建立套組選檔列表同樣清楚。  
- [ ] 「簡報教材」套組上傳超過 20MB 之 mp4：傳輸視窗顯示失敗與類型上限說明，列表／NAS 無新檔。  
- [ ] 建立套組單檔或總量超限：同樣明確失敗原因。
