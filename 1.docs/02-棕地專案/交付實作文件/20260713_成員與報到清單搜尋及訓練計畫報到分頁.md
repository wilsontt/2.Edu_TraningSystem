# 成員／報到清單搜尋 ＋ 訓練計畫報到統計分頁

## 1. 目的

於下列三處人員清單補齊「員工編號、姓名、部門」前端搜尋，並將訓練計畫報到統計 Modal 的分頁行為對齊報到總覽。

## 2. 範圍

| 畫面 | 元件 | 變更 |
|------|------|------|
| 角色管理 → 成員清單 | `frontend/src/components/admin/RoleManager.tsx` | 搜尋（員編／姓名／部門）；既有分頁改為先過濾再分頁 |
| 報到總覽 → 報到統計 | `frontend/src/components/attendance/AttendanceOverviewPage.tsx` | 同上搜尋；分頁以過濾後筆數為準 |
| 訓練計畫 → 報到統計 | `frontend/src/components/admin/TrainingPlanManager.tsx` | 同上搜尋；新增 `Pagination`（移除僅靠 `max-h-60` 捲動） |

不變更後端 API；資料已由既有 stats／角色成員 API 一次載入，搜尋為前端過濾。

## 3. 權責

| 角色 | 權責 |
|------|------|
| 前端 | 實作搜尋框、過濾、分頁重置 |
| 驗收 | 三處 UI 行為一致 |

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| 前端過濾 | 不另打 API，對已載入陣列做 `includes`（大小寫不敏感） |
| 符合 N 位 | 標題旁顯示搜尋命中筆數；分頁 `totalItems` 為過濾後筆數 |

## 5. 作業內容

1. 各 Modal／卡片清單上方新增搜尋框，placeholder：`搜尋員工編號、姓名或部門...`
2. 過濾欄位：
   - 角色成員：`emp_id`、`name`、`department`
   - 報到清單：`emp_id`、`name`、`dept_name`
3. 搜尋或切換統計卡片時重置頁碼為 1
4. 訓練計畫報到統計：對齊報到總覽之 `Pagination`（每頁筆數可調）
5. 「列印目前清單」仍依卡片篩選（應到／實到／未到／請假）打後端，**不受**前端搜尋關鍵字限制（與既有列印 API 行為一致）
