# 角色管理：在職成員數、Admin 清單可見、成員分頁項次

## 1. 目的

修正角色管理三項問題：

1. 角色卡片「成員人數」與成員清單不一致（應排除停用人員）。
2. Admin 角色有 1 人，但成員清單空白（break-glass `admin` 為管理帳號被過濾）。
3. 成員清單人數多時難閱讀，需項次與分頁。

## 2. 範圍

| 層級 | 變更 |
|------|------|
| 後端 | `GET /admin/roles` 的 `user_count` 僅計 `status=active`；新增 `GET /admin/roles/{role_id}/users` |
| 前端 | `RoleManager.tsx` 改用專用成員 API；成員 Modal 加項次與共用 `Pagination` |
| 測試 | `tests/test_role_active_member_count.py`（離線語意測試） |

## 3. 權責

- 開發：角色管理 API／UI
- 驗收：admin 登入後於「角色管理」確認 Admin／系統管理／User 等角色

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| 在職成員 | `users.status == "active"` |
| 管理帳號 | `users.is_trainee == False`（如 break-glass `admin`、AD JIT 管理員） |
| 訓練員帳號 | `users.is_trainee == True`（預設 `/admin/users` 名單） |

## 5. 作業內容

### 5.1 根因

- 卡片人數：`len(role.users)` → 含停用。
- 成員清單：`GET /admin/users` 預設 `include_inactive=false` 且 `trainees_only=true` → 排除停用與管理帳號。
- 兩邊過濾條件不同 → 數字與清單不一致；Admin 成員被完全藏起。

### 5.2 後端

**`GET /admin/roles`**

- `user_count` = 該角色下 `status=active` 的使用者數（含管理帳號）。

**`GET /admin/roles/{role_id}/users`（新增）**

- 僅回傳在職成員（`apply_active_user_filter`）。
- **不**套用 `apply_trainee_filter`（管理帳號可見）。
- 回傳結構：`{ role_id, role_name, user_count, users: [...] }`。

### 5.3 前端

- 開成員 Modal → 呼叫 `/admin/roles/{id}/users`。
- 新增成員候選名單 → `/admin/users?trainees_only=false`（仍排除停用）。
- 成員列顯示項次（跨頁連續）。
- 使用 `frontend/src/components/common/Pagination.tsx`。
- 標題顯示「共 N 位成員」。
