## 1. 目的

1. 多場訓練合併報到（可跨開始日；場次日 OR 關鍵字；已選購物車）→ 一組 QR 寫入多筆 `attendance_records`。  
2. append-only 報到歷程（`attendance_checkin_events`）；`checkin_time` 永不覆蓋。  
3. 批次可關閉／重開。  
4. 報到總覽單計畫 QR **不綁 Owner**（有 `menu:attendance-overview` 即可）。

---

## 2. 部署（必含前端重建）

> **症狀**：Modal 仍顯示「訓練日期」「此日期尚無進行中的訓練計畫」、沒有「關鍵字」欄 → **前端仍是舊 bundle**，僅拉後端或未 rebuild `training-frontend` 不夠。

### 2.1 程式分支

```bash
cd /path/to/2.Edu_TraningSystem
git fetch origin
git checkout cursor/batch-merge-or-select-3b7d
# 或合併 PR #5 後再部署 main
```

### 2.2 DB 遷移（後端）

```bash
cp data/education_training.db data/education_training.db.bak-$(date +%Y%m%d)
cd backend
.venv/bin/python3 migrations/add_attendance_checkin_batch_and_events.py
```

ds1／Docker：

```bash
cd /opt/apps/enterprise-portal/deploy   # 依實際路徑
docker compose exec training-backend python migrations/add_attendance_checkin_batch_and_events.py
```

### 2.3 重建並重啟前端＋後端（必做）

映像需從本子專案建置（同層需有 `0.shared-ui`）。於企業入口 deploy 目錄：

```bash
# 確認 compose 的 build context 已指向含本分支原始碼之目錄後：
docker compose build --no-cache training-frontend training-backend
docker compose up -d --force-recreate training-frontend training-backend
```

瀏覽器對報到總覽頁 **強制重新整理**（Ctrl+Shift+R／清快取）。

### 2.4 確認已是新版 UI（缺一即未部署成功）

開啟「多場訓練合併報到」後應同時看到：

| 新版應有 | 舊版（未更新） |
|----------|----------------|
| 提示「關鍵字／場次日 OR」 | 無此提示 |
| 「關鍵字（搜尋計畫）」 | 無此欄 |
| 「場次／報到日」 | 「訓練日期」 |
| 「標籤（批次名稱，選填）」 | 「標籤（選填）」 |
| 開啟後直接列出進行中計畫 | 「此日期尚無進行中的訓練計畫」 |

**報到完成頁**（掃 QR → 登入 → 合併報到）應出現**藍底區塊「報到人」**（部門 · 員編 · 姓名）。若完全沒有此區塊，表示 `:5173` 的 Vite 仍跑舊程式，請在該主機執行：

```bash
cd /path/to/2.Edu_TraningSystem
git fetch && git checkout cursor/batch-merge-or-select-3b7d && git pull
# 後端重啟（含 checked_in_user 欄位）
# 前端：停掉舊 vite 後重開
cd frontend && npm run dev -- --host 0.0.0.0 --port 5173
```

驗證後端是否已含報到人欄位（需有效 JWT）：

```bash
curl -s -X POST "http://127.0.0.1:8000/api/exam/attendance/batches/<batch_id>/checkin" \
  -H "Authorization: Bearer <token>" -H "Content-Type: application/json" | jq .checked_in_user
```
