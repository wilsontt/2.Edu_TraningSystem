# AD 整合：系統管理者登入實作計劃 (PLAN)

**文件類型**：實作計劃（PLAN，僅設計，不含程式碼）
**建立日期**：2026-06-12
**狀態**：草案（待核可後進入技術設計與實作）
**對應需求**：IT 管理者改以 AD 群組 `IT_Admin` 登入管理端；**保留單一本地管理帳號（break-glass）** 作為 AD 故障時之緊急登入，並套用 ISO 27001 密碼政策。

> 本計劃依使用者全域規範採 7 大結構：目的／範圍／權責／名詞解釋／作業內容／參考文件／使用表單。

---

## 1. 目的

1. 將「IT 系統管理者」之**身分驗證**改由企業 AD（Active Directory）負責，僅允許 AD 群組 **`IT_Admin`** 之成員登入管理端。
2. **保留單一本地管理帳號（break-glass）**：當 AD 故障且短時間無法復原時，仍可由此帳號以**本地密碼**登入管理端。此帳號**不可刪除、不可停用至無法登入**，並套用 ISO 27001 密碼政策。
3. **保留本地 RBAC**（角色、功能選單、部門可視範圍）作為**授權**依據；AD 僅負責「**是誰**」與「**能否進入管理端**」。
4. 一併清理現存安全債（JWT 金鑰硬編碼、驗證碼後門 `0000`），確保上線安全基線。

> **重要前提（已確認）**：本系統本地使用者主鍵為 `emp_id`（純數字員工編號），而 **AD 登入帳號（sAMAccountName）≠ 員工編號**。因此 AD 登入採**方案 B**：以 `users.ad_username` 對映既有 `emp_id`，避免 JIT 佈建產生重複人員。詳見 §5.3。

## 2. 範圍

### 2.1 涵蓋範圍（In Scope）

- 新增「管理端 AD 登入」流程（LDAPS bind + 群組檢查）。
- AD 驗證通過後之 **JIT（Just-In-Time）使用者對映／佈建**：以 `ad_username` 對映既有 `emp_id`（方案 B），更新登入時間與角色。
- **本地 ADMIN 帳密登入**與 **ISO 27001 密碼政策**（複雜度、長度 ≥ 12、90 天到期、失敗鎖定）。
- 認證設定**環境變數化**（AD 連線、JWT 金鑰、密碼政策、後門開關）。
- 既有 `auth.py` 權限判斷與超級角色判定之相容處理；**關閉管理角色之免密登入路徑**。

### 2.2 不涵蓋範圍（Out of Scope）

- **一般員工登入維持現狀**（工號＋圖形驗證碼免密），本計劃不改動。
- AD 群組與本地角色之**細粒度多對多映射**（本期僅單一群組 `IT_Admin` → 單一管理角色）。
- AD 密碼修改、AD 帳號生命週期管理（屬 AD／IT 既有流程）。
- **多個本地管理帳號**：本期僅保留**單一** break-glass 本地 ADMIN，不另開多個 supervisor。
- **密碼歷史比對**（不重用前 N 次舊密碼）：本期**不實作**。
- SSO / OIDC（Entra ID、ADFS）：本期採 **LDAPS**，OIDC 留待未來評估。

### 2.3 已確認決策

| 決策點 | 結論 |
|--------|------|
| 認證協定 | **LDAPS**（帳密直連 DC，內網單機，改動最小） |
| 套用範圍 | **僅管理端／特權角色**走 AD；一般員工不變 |
| 准入群組 | 僅 AD 群組 **`IT_Admin`** |
| 授權來源 | **本地 RBAC**；AD 只決定「能否進管理端」 |
| AD↔本地對映 | **方案 B**：AD 帳號（sAMAccountName）≠ emp_id，以 `users.ad_username` 對映既有 `emp_id` |
| 帳號佈建 | **JIT**：AD 登入且屬 IT_Admin → 以 `ad_username` 找既有 user；找不到才依規則建立 |
| 本地 break-glass | **保留單一**本地 ADMIN（沿用既有 `admin`），`is_protected` 不可刪停 |
| 密碼複雜度 | 數字＋大寫＋小寫＋特殊符號，長度 **≥ 12** |
| 密碼到期 | **90 天**；到期採「**強制改密碼才放行**」 |
| 失敗鎖定 | 連續 **6 次** → 鎖定 **15 分鐘** |
| 密碼歷史 | **不實作** |

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| **AD** | Active Directory，企業目錄服務 |
| **LDAPS** | LDAP over SSL/TLS（埠 636），加密的目錄查詢／驗證協定 |
| **bind** | 以帳號密碼向 AD 進行繫結驗證，成功即代表帳密正確 |
| **DN / Base DN** | Distinguished Name；Base DN 為搜尋起點（如 `DC=yourco,DC=local`） |
| **UPN** | User Principal Name，如 `user@yourco.local` |
| **sAMAccountName** | AD 的傳統登入帳號（如 `it01`） |
| **memberOf** | AD 使用者所屬群組清單屬性 |
| **JIT 佈建** | 首次登入時即時於本地建立／更新使用者資料 |
| **RBAC** | 本系統角色基礎存取控制（角色＋功能＋部門範圍） |
| **break-glass** | 緊急用本地管理帳號；平時少用，AD 故障時作為唯一保險絲登入管理端 |
| **ISO 27001 密碼政策** | 本案指：複雜度（數字＋大小寫＋特殊符號）、長度 ≥ 12、90 天到期、失敗鎖定 |
| **emp_id** | 本地使用者主鍵，純數字員工編號（與 AD 帳號不同） |

## 5. 作業內容

### 5.1 整體流程

本系統管理端登入分**兩條路徑**，一般員工免密登入不變。

#### 路徑 A：IT 管理者 — AD 登入（日常）

```
管理端登入頁 → 選「AD 登入」→ POST /api/auth/login/admin
  1. 檢查 AD_ENABLED（false → 503「AD 整合未啟用」）
  2. LDAPS bind 驗證帳密（失敗 → 401）
  3. 讀取 memberOf，檢查 IT_Admin（否 → 403）
  4. 以 ad_username 對映本地 users（方案 B，見 §5.3）
  5. 簽發 JWT（auth_src=ad）
```

#### 路徑 B：本地 ADMIN — 帳密登入（break-glass，不受 AD_ENABLED 影響）

```
管理端登入頁 → 選「本地緊急登入」→ POST /api/auth/login/local
  1. 驗證 emp_id + password（bcrypt）
  2. 檢查 is_protected 且具管理角色
  3. 檢查 locked_until（鎖定中 → 423）
  4. 檢查密碼是否過期 90 天 → 強制改密碼流程（§5.4.2）
  5. 簽發 JWT（auth_src=local）或「須改密」之有限 token
```

#### 路徑 C：一般員工 — 免密登入（不變）

```
POST /api/auth/login（工號＋驗證碼）
```

#### 管理角色免密登入關閉

- 具**管理端功能**之角色（如 `系統管理`、`Admin`），**不得**再經路徑 C 登入。
- 路徑 C 若偵測到管理角色 → **403**「請使用 AD 或本地緊急登入」。
- **例外**：`is_protected=true` 之 break-glass 帳號僅走路徑 B，不走路徑 C。

#### AD 帳號 ≠ emp_id 對映（方案 B）

| 步驟 | 行為 |
|------|------|
| 1 | AD 驗證成功，取得 `sAMAccountName`（如 `it01`） |
| 2 | 查 `users.ad_username = 'it01'` |
| 3a | **找到** → 以該筆 `emp_id` 簽 JWT，更新 `last_login_at` |
| 3b | **找不到** → **403**「AD 帳號尚未對映至本地員工，請聯絡系統管理員」；**禁止**以 AD 帳號當 emp_id 新建使用者 |

> 對映由系統管理員於「人員管理」維護 `ad_username` 欄位（或 IT 提供對照表批次匯入）。JIT **不自動新建** emp_id，避免與既有 HR 員編脫鉤。


### 5.2 設定環境變數化（新增 `backend/app/config.py`）

以 `pydantic-settings` 集中讀取，**禁止硬編碼**：

| 變數 | 用途 | 範例 |
|------|------|------|
| `JWT_SECRET_KEY` | 取代 `auth_utils.SECRET_KEY` 硬編碼 | `openssl rand -hex 32` 產生 |
| `AD_ENABLED` | AD 登入總開關 | `true` |
| `AD_SERVER_URI` | DC 位址 | `ldaps://dc.yourco.local:636` |
| `AD_BASE_DN` | 搜尋起點 | `DC=yourco,DC=local` |
| `AD_DOMAIN` | UPN／網域 | `yourco`（組 `user@yourco.local`） |
| `AD_ADMIN_GROUP` | 准入群組 CN | `IT_Admin` |
| `AD_ADMIN_ROLE_NAME` | 對應本地角色名稱 | `系統管理` |
| `LOGIN_BYPASS_ENABLED` | 驗證碼後門 `0000` 開關（預設關閉） | `false` |
| `PASSWORD_MIN_LENGTH` | 本地 ADMIN 密碼最短長度 | `12` |
| `PASSWORD_MAX_AGE_DAYS` | 密碼最長有效天數 | `90` |
| `LOGIN_MAX_FAILED` | 連續失敗次數上限 | `6` |
| `LOGIN_LOCKOUT_MINUTES` | 鎖定分鐘數 | `15` |
| `BREAK_GLASS_EMP_ID` | break-glass 帳號 emp_id | `admin` |

> `.env` 必須列入 `.gitignore`（現況已忽略 `.env`，需再確認）。

### 5.3 AD 驗證服務（新增 `backend/app/services/ad_auth.py`）

- 套件：`ldap3`（需加入 `requirements.txt`）。
- 函式：
  - `authenticate_ad(username, password) -> dict | None`：LDAPS bind；成功回 `{ad_username, display_name, groups, mail}`。
  - `is_in_admin_group(groups) -> bool`：比對 `AD_ADMIN_GROUP`（以 `CN=IT_Admin,` 前綴判斷，忽略大小寫）。
- 連線失敗、帳密錯誤、查無使用者一律回 `None`（端點轉成 401）。
- **不負責** emp_id 對映；對映由登入端點查 `users.ad_username`（§5.1 方案 B）。

### 5.3.1 密碼政策服務（新增 `backend/app/services/password_policy.py`）

- `validate_password_complexity(password) -> None`：不符則拋錯。
  - 長度 ≥ `PASSWORD_MIN_LENGTH`（12）
  - 含小寫、大寫、數字、特殊符號各至少一個
- `is_password_expired(password_changed_at) -> bool`：超過 `PASSWORD_MAX_AGE_DAYS`（90）為 true。
- **不實作**密碼歷史比對。
- 雜湊：沿用 `auth_utils.pwd_context`（bcrypt）。

### 5.4 新增端點

#### 5.4.1 `POST /api/auth/login/admin`（AD 登入）

- Request：`{ username, password }`（username = AD sAMAccountName）。
- 行為：依 §5.1 路徑 A；對映成功後簽 JWT，`auth_src=ad`。
- `AD_ENABLED=false` → **503**（不影響路徑 B）。

#### 5.4.2 `POST /api/auth/login/local`（本地 break-glass）

- Request：`{ emp_id, password }`。
- 僅允許 `is_protected=true` 且具管理角色之帳號（預設 `BREAK_GLASS_EMP_ID=admin`）。
- 失敗計數：連續 **6 次** → 設定 `locked_until = now + 15 分鐘`。
- 密碼過期（90 天）：回傳 `{ must_change_password: true, change_token: "..." }`，**不發完整管理 JWT**；須呼叫改密 API 後才取得完整 token。
- **不受 `AD_ENABLED` 影響**（AD 全斷仍可登入）。

#### 5.4.3 `POST /api/auth/password/change`（改密）

- Request：`{ change_token 或 已登入 JWT, old_password, new_password }`。
- 新密碼須通過 §5.3.1 複雜度；成功後更新 `password_hash`、`password_changed_at`，清除 `must_change_password`。
- 改密成功後簽發完整 JWT。

#### 5.4.4 既有 `/login` 調整

- 若使用者角色屬管理端角色 → **403**「請使用 AD 或本地緊急登入」。
- 一般 `User` 角色不受影響。

### 5.5 資料模型微調（需遷移，`models.User`）

| 新欄位 | 型別 | 用途 |
|--------|------|------|
| `auth_source` | String，預設 `local` | 區分 `local` / `ad`（最近登入來源） |
| `ad_username` | String，nullable，**unique** | AD sAMAccountName；**對映 emp_id 用（方案 B）** |
| `last_login_at` | DateTime，nullable | 稽核用 |
| `password_hash` | String，nullable | bcrypt；**僅 break-glass 等本地密碼帳號** |
| `password_changed_at` | DateTime，nullable | 計算 90 天到期 |
| `must_change_password` | Boolean，預設 false | 首次／重設後強制改密 |
| `failed_login_count` | Integer，預設 0 | 失敗鎖定計數 |
| `locked_until` | DateTime，nullable | 鎖定到期 |
| `is_protected` | Boolean，預設 false | break-glass；**禁止刪除、禁止停用** |

**break-glass 初始化**（遷移或 `init_db` 補丁）：

- 將 `emp_id = admin`（或 `BREAK_GLASS_EMP_ID`）設 `is_protected=true`、掛管理角色。
- 首次部署須設定**符合 ISO 政策之初始密碼**，並設 `must_change_password=true`（首次登入必改）。

**人員管理 UI**（技術設計）：

- IT 人員維護時可編輯 `ad_username`，與既有 emp_id、角色設定並存。

> 遷移依 `1.docs/00-專案總覽/資料庫遷移/MIGRATION_GUIDE.md` 模式；執行前備份資料庫。

### 5.6 安全債清理（同批處理）

1. `auth_utils.SECRET_KEY` → 改讀 `settings.jwt_secret_key`（**上線前必修**）。
2. 登入與 QRcode 登入之 `answer == "0000"` 後門 → 受 `LOGIN_BYPASS_ENABLED` 控制，生產環境關閉。
3. `check_permission` / `check_any_permission` 內超級角色字串 → 收斂為集中常數。
4. 移除 `auth.py` 內敏感 `print(...)` 除錯輸出。
5. break-glass 帳號：`is_protected` 帳號於人員管理 API **禁止 delete／禁止 status=inactive**。
6. 管理角色 **禁止** 經免密 `/login` 進入（§5.1）。

### 5.7 驗收條件

| # | 案例 | 期望 |
|---|------|------|
| 1 | 正確 IT_Admin AD 帳密，且 ad_username 已對映 emp_id | 200，取得 token |
| 2 | 正確 AD 帳密但非 IT_Admin | 403 |
| 3 | AD 帳密錯誤 | 401 |
| 4 | AD 帳密正確但 ad_username 未對映本地 | 403，提示聯絡管理員 |
| 5 | `AD_ENABLED=false`，呼叫 `/login/admin` | 503 |
| 6 | `AD_ENABLED=false`，本地 break-glass 帳密正確 | 200（路徑 B 不受影響） |
| 7 | AD 全斷（DC 不可達），本地 break-glass | 仍可登入 |
| 8 | 密碼不符複雜度或長度 < 12 | 400，拒絕設定 |
| 9 | 密碼超過 90 天 | 須改密才取得完整 JWT |
| 10 | 連續 6 次本地登入失敗 | 423 或 401 + 鎖定 15 分鐘 |
| 11 | 管理角色經免密 `/login` | 403 |
| 12 | 一般員工免密 `/login` | 不受影響 |
| 13 | 刪除或停用 `is_protected` 帳號 | API 400/403 拒絕 |
| 14 | 程式碼掃描 | 無硬編碼金鑰；後門預設關閉 |

## 7. 使用表單（欄位說明）

### 7.1 管理端 AD 登入請求 `AdminLoginRequest`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `username` | string | 是 | AD sAMAccountName（如 `it01`），**非 emp_id** |
| `password` | string | 是 | AD 密碼（僅用於即時 bind，不落地儲存） |

### 7.2 本地 break-glass 登入請求 `LocalAdminLoginRequest`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `emp_id` | string | 是 | 本地員工編號（如 `admin`） |
| `password` | string | 是 | 符合 ISO 27001 政策之本地密碼 |

### 7.3 改密請求 `PasswordChangeRequest`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `old_password` | string | 是 | 舊密碼 |
| `new_password` | string | 是 | 新密碼（複雜度 + ≥12） |
| `change_token` | string | 否 | 過期強制改密時使用 |

### 7.4 登入回應（完整 JWT）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `access_token` | string | JWT |
| `token_type` | string | `bearer` |
| `user.emp_id` | string | 本地 emp_id（AD 登入經 ad_username 對映後） |
| `user.name` | string | 顯示名稱 |
| `user.role` | string | 本地 RBAC 角色 |
| `user.functions` | string[] | 功能碼清單 |
| `auth_src` | string | `ad` / `local` |

### 7.5 須改密回應（90 天到期）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `must_change_password` | boolean | `true` |
| `change_token` | string | 限時 token，僅可呼叫改密 API |

### 7.6 `users` 新增／調整欄位

| 欄位 | 型別 | 預設 | 說明 |
|------|------|------|------|
| `auth_source` | string | `local` | 最近登入來源 |
| `ad_username` | string | null | AD 帳號，**unique**，對映 emp_id |
| `last_login_at` | datetime | null | 最近登入 |
| `password_hash` | string | null | bcrypt |
| `password_changed_at` | datetime | null | 改密時間 |
| `must_change_password` | boolean | false | 強制改密 |
| `failed_login_count` | int | 0 | 失敗次數 |
| `locked_until` | datetime | null | 鎖定至 |
| `is_protected` | boolean | false | break-glass 保護 |

## 8. 風險與營運規範

| 項目 | 說明 |
|------|------|
| UPN 格式 | 需 IT 確認 bind 用 `user@domain.local` 或 `DOMAIN\user` |
| 群組巢狀 | IT_Admin 若為巢狀群組，`memberOf` 可能不含；必要時用 `LDAP_MATCHING_RULE_IN_CHAIN` |
| 憑證信任 | LDAPS 需信任 DC 憑證 |
| **AD 帳號 ≠ emp_id** | 須事先維護 `ad_username` 對映；未對映者無法 AD 登入（403） |
| **break-glass（已保留）** | 單一本地 ADMIN（`is_protected`）；密碼由資訊安全／IT 保管，平時封存；**登入須寫稽核 log** |
| AD 故障 | 路徑 B 仍可登入；IT 日常走 AD，故障時切 break-glass |
| 鎖定 | 6 次失敗鎖 15 分鐘；避免唯一 break-glass 被暴力破解（仍須強密碼） |
| 密碼到期 | 強制改密才放行，避免鎖死唯一保險絲 |
| 對映維護 | 新進 IT 須在「人員管理」填 `ad_username` 並掛管理角色 |
