# 成績中心導覽、報到總覽篩選與歷史記錄精簡 — 實作計劃 (PLAN)

**文件類型**：棕地實作計劃  
**建立日期**：2026-07-10  
**狀態**：✅ 已實作（2026-07-10）  
**關聯議題**：部門成績查看個人成績導覽錯誤、報到總覽僅顯示進行中計畫、歷史記錄與學習分析成績趨勢重複

> 本文件依 2026-07-10 使用者回饋定稿撰寫，不新增額外需求。  
> **與 T13 測試紀錄之差異**：T13 曾將「部門成績 → 查看個人成績」改為導向**總覽**頁籤；本次定稿改回導向**歷史記錄**（與個人登入後預設行為一致）。

---

## 1. 目的

1. **部門成績 → 查看個人成績**：具跨部門／部門範圍權限者（總稽核、業務、系統管理等）從「部門成績」點選成員後，應進入該員的**歷史記錄**頁籤，而非總覽。
2. **報到總覽**：支援依訓練計畫狀態（正在進行中／已過期／已封存／全部）篩選，並加上計畫名稱搜尋，便於回頭檢視已結束或已封存計畫的報到統計。
3. **成績中心 → 歷史記錄**：移除上半部「選擇計畫／成績趨勢」區塊（學習分析頁已有成績趨勢），僅保留搜尋排序與訓練清單表格。

## 5. 作業內容

### 5.1 議題 A：部門成績 → 歷史記錄導覽

#### 5.1.1 現況與根因

`ReportDashboard.tsx` 兩處「查看個人成績」`<Link>` 硬編碼 `tab=overview`：

| 位置 | 情境 |
|------|------|
| 約 L1403 | 部門統計手風琴 → 成員成績列表 |
| 約 L1533 | 訓練計畫維度 → 成員列表 |

`PersonalScorePage` 雖預設 `localTab = 'history'`，但 URL 的 `tab` 參數優先（`activeTab = urlTab ?? localTab`），故實際落在總覽。

#### 5.1.2 修改

**檔案**：`frontend/src/components/admin/ReportDashboard.tsx`

將兩處連結之 `tab=overview` 改為 `tab=history`：

```tsx
to={`/reports/personal?emp_id=${record.emp_id}&tab=history&emp_name=${encodeURIComponent(record.name || '')}&dept_name=${encodeURIComponent(record.dept_name || '')}`}
```

部門統計列之 `dept_name` 可 fallback 至 `(item as DepartmentStat).name`（維持既有邏輯）。

#### 5.1.3 不需修改

- `PersonalScorePage.tsx`：預設頁籤、URL 解析、`titlePrefix` 已正確
- 後端 `/exam/personal/history?emp_id=`：權限與資料範圍既有邏輯不變

### 5.2 議題 B：報到總覽狀態篩選與搜尋

#### 5.2.1 現況與根因

`AttendanceOverviewPage.tsx`：

- `const [activeTab] = useState<TabStatus>('active')` — **無 setter**，永遠為進行中
- `api.get('/training/plans', { params: { status: 'active' } })` — API 固定
- UI 僅靜態 badge「目前僅顯示：正在進行中」
- 檔案註解已寫「與訓練計畫管理相同分頁」，但 UI 未實作

後端 `GET /training/plans?status=` 已支援 `active`／`expired`／`archived`／`all`（`training.py` L272–308），權限含 `menu:attendance-overview`。

#### 5.2.2 修改

**檔案**：`frontend/src/components/attendance/AttendanceOverviewPage.tsx`

**State**

```tsx
type PlanStatusFilter = 'active' | 'expired' | 'archived' | 'all';
const [planStatusFilter, setPlanStatusFilter] = useState<PlanStatusFilter>('active');
const [searchTerm, setSearchTerm] = useState('');
```

**資料載入**

```tsx
useEffect(() => {
  const fetchPlans = async () => {
    setLoading(true);
    const res = await api.get('/training/plans', {
      params: { status: planStatusFilter },
    });
    setPlans(res.data || []);
    setStatsMap({});
    setModalPlanId(null);
    // ...
  };
  fetchPlans();
}, [planStatusFilter]);
```

**前端搜尋**（參考 `ExamStudio.tsx` L398–428、`TrainingPlanManager.tsx` L637–638）

```tsx
const filteredPlans = useMemo(() => {
  const k = searchTerm.trim().toLowerCase();
  if (!k) return plans;
  return plans.filter(
    (p) =>
      p.title.toLowerCase().includes(k) ||
      p.training_date.includes(k) ||
      (p.year ?? '').toLowerCase().includes(k),
  );
}, [plans, searchTerm]);
```

表格改渲染 `filteredPlans`；`modalPlan` 查找仍用 `plans` 全集合。

**UI 替換**

- 移除綠色靜態 badge（L272–274）
- 新增工具列（置於 header 下方）：
  - `<select>` 選項：正在進行中／已過期／已封存／全部
  - `<input type="search">` placeholder：`搜尋訓練計畫...`
  - 可選：「更新」按鈕重新 fetch（與考卷工坊一致）

**空狀態文案**（依 `planStatusFilter`）

| 狀態 | 文案範例 |
|------|----------|
| active | 目前尚無正在進行中的訓練計畫 |
| expired | 目前尚無已過期的訓練計畫 |
| archived | 目前尚無已封存的訓練計畫 |
| all | 目前尚無訓練計畫 |

**封存唯讀**

將 `absenceReasonReadOnly` 判斷中之 `activeTab === 'archived'` 改為：

```tsx
Boolean(modalPlan?.is_archived) || planStatusFilter === 'archived';
```

（或僅依 `modalPlan?.is_archived`，因單筆計畫屬性已足夠。）

### 5.3 議題 C：歷史記錄移除成績趨勢

#### 5.3.1 現況與根因

`PersonalScoreHistory.tsx` 上半部包含：

1. 標題「個人成績歷史」— **保留**
2. 搜尋／排序列 — **保留**
3. 「選擇計畫」checkbox 區（L322–370）— **移除**
4. 「成績趨勢」折線圖（L372–422）— **移除**

`PersonalLearningAnalysis.tsx` 已有「成績趨勢」柱狀圖（3／6／12 個月切換，L206–257），功能重疊。

此外 `fetchPlansHistory()` 對當頁每筆記錄呼叫 `GET /exam/record/{id}/detail`，僅為繪製趨勢圖，造成不必要請求與載入延遲。

#### 5.3.2 刪除清單

| 類別 | 項目 |
|------|------|
| import | `recharts` 全系列（`LineChart`, `Line`, `XAxis`, `YAxis`, `CartesianGrid`, `Tooltip`, `Legend`, `ResponsiveContainer`） |
| interface | `PlanTrendData`, `DetailHistoryItem`, `TrendChartPoint` |
| state | `plansData`, `selectedPlanIds` |
| 函式 | `fetchPlansHistory`；`fetchHistory` 內對其的呼叫 |
| 衍生邏輯 | `COLORS`, `chartData`, `handlePlanToggle`, `selectAllPlans`, `clearAllPlans`, `maxAttempts` 迴圈 |
| JSX | 「選擇計畫」區塊、「成績趨勢」圖表區塊 |

#### 5.3.3 保留清單

| 項目 |
|------|
| `fetchHistory` → `GET /exam/personal/history` |
| 搜尋 debounce（350ms）、排序、分頁 |
| 成績清單表格（ITEM 序號、考試次數、查看詳情、開放重考） |
| `PlanHistoryModal`、`AuthorizeRetakeModal` |

#### 5.3.4 效能預期

移除後，歷史記錄頁每次載入僅 **1 次** list API（加分頁參數），不再對每列 record 打 detail API。
