2026/06/18 教育訓練 建議修改項目：

1. 教材上傳/下載：
   1. 要進行上傳/下載時，顯示`要求登入` NAS 的驗證。
      1. 要求登入，即可知道是誰在進行傳輸，同時也可以抓到IP。
      2. 登入可用`NAS本地` 的帳號，或是`AD網域` 的帳號。
   2. 開始進行傳輸時，要有`傳輸進度` % 的顯示。
      1. 顯示傳輸的視窗，要有取消中止傳輸的按鈕。
      2. 要離開傳輸視窗，只能`右上的X`，或底下的`取消傳輸`的按鈕。
      3. 正在傳輸的過程中，禁止按`F5、Ctrl＋R` 網頁重整。
   3. 完成傳輸後，顯示傳輸成功。然後關閉傳輸視窗後，要Close Session，主要用於終止連線、釋放系統資源或結束正式會議。
2. 系統與NAS之間，因應安全性問題，應該不要一直用SMB來連線者。
3. QRCode產生：
   1. 產生時，給前端使用者掃描用的QRCode碼，只要短碼即可，http://IP/training/login 即可。
      1. 此處IP是系統server的IP。
   2. QRCode長碼，可能會讓瀏覽器產生誤判為安全性問題（如Safari就如此），而中止連線，所以QRCode長碼僅給系統用。
   3. 前端頁面不用顯示QRCode的有效時間。
4. 行動裝置的 UI排版要調整。
   1. 提示詞：`跨平台＋行動裝置＋IOS-Android`，重新調整前端頁面，不要動要商業邏輯。
   2. 可先調整某一個頁面（如考試中心、成績中心），OK後才再進行其他的頁面。
   3. 然後可以將此作法，寫成skill，日後只要呼叫skill即可。
5. 訓練計畫-編輯訓練計畫：
   1. 授課單位 - 個人授課對象 是有關聯的，個人授課對象清單預設是空的。
   2. so，授課單位勾了那個單位，個人授課對象就顯示該單的人，未勾選單位的人就不要顯示出來。
   3. 如果要單獨對 某一人 或 某些個人 對象，就在底下 新增一個`新增個人對象`按鈕，來新增個人對象。
   4. 勾選了新增個人對象，就`顯示在個人授課對象的卡片`裡。
6. 進入考試中心，然後跳到其他某一個頁面，然後再回到考試中心：
   1. 結果考試中心的按鈕不能按。
   2. 需要重整頁面後才能按（如`立即報到/開始考試`按鈕）。
7. 完成修改後，在`claude code 中進行 /security-review`。
