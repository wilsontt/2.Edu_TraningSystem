## 1. 目的

### 1.1 及格分硬編碼問題
前端 `ExamDashboard`、`PersonalScoreHistory` 等多個元件對「及格」判斷硬編碼 `score >= 60`，導致以下缺陷：
- 不同訓練計畫設定不同及格分（如 70、80、90 分）時，前端仍按 60 判斷，與實際不符
- 每增加新計畫需修改多個檔案，後續維護困難

**正確做法**：後端根據訓練計畫的 `passing_score` 欄位判斷是否通過（`is_passed`），前端完全移除硬編碼 60，改用 API 回傳值。

### 1.2 已通過者重考授權流程缺失
現況：學員通過考試後，`start_exam` 會直接擋住，無法重考。但業務需求是：
- **已通過者不可自行重考**
- **需主管或稽核人員授權**才能重考一次
- 一次授權 = 一次機會，提交後自動清除

目前缺少：
- 授權流程定義（誰授權、如何授權、記錄在哪）
- 授權 UI（成績中心的「開放重考」按鈕）
- 授權狀態跟蹤（哪些已通過者已獲授權）

**正確做法**：新增授權表、API、UI，實現「申請→授權→重考→消耗」的完整流程。

### 1.3 訓練計畫編輯 Modal 過長
現況：編輯訓練計畫時，Modal 單欄堆疊，教材管理區（上傳、列表、編輯卡片）放在最底部，導致需大量垂直捲動，UX 不佳。

**正確做法**：編輯模式改為左右雙欄（Modal 寬度 `max-w-6xl`）：
- 左欄：訓練計畫基本設定（名稱、日期、受課對象等）
- 右欄：教材管理（上傳、列表、編輯）
- 新增模式維持單欄 `max-w-2xl`（無 planId，無教材區）

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| `passing_score` | 訓練計畫設定的及格分數（整數，預設 60），儲存於 `training_plans` 表 |
| `is_passed` | 考試紀錄是否通過（`score >= passing_score`）；API 回傳，由後端統一計算 |
| `can_start_exam` | 後端判斷的「是否可開始考試」；考慮計畫狀態、日期、提交狀態、通過狀態、授權狀態 |
| `retake_authorized` | `exam_records` 新欄位 (BOOLEAN)；指本次考試是否已獲得重考授權 |
| `exam_retake_authorizations` | 新表，記錄每次授權行為：誰授權、何時、原因、何時消耗、何時撤銷 |
| `btn:exam:authorize-retake` | 新功能碼，標記具授權能力的使用者 |
| **授權**（Authorize） | 主管/稽核主動開放某員工對已通過考試的重考機會 |
| **消耗**（Consumed） | 學員因授權重考而提交答卷；系統自動清除 `retake_authorized` 旗標、填入 `consumed_at` 時間 |
| **撤銷**（Revoked） | 授權人在學員尚未使用前，收回授權資格 |

每次提交照舊：

**新增**：若提交前 `record.retake_authorized == True`：

```python
# 消耗授權
record.retake_authorized = False

# 標記對應 authorization 為已消耗
pending_auth = db.query(ExamRetakeAuthorization).filter(
    ExamRetakeAuthorization.record_id == record_id,
    ExamRetakeAuthorization.consumed_at.is_(None),
    ExamRetakeAuthorization.revoked_at.is_(None),
).order_by(ExamRetakeAuthorization.authorized_at.desc()).first()

if pending_auth:
    pending_auth.consumed_at = now
```

```tsx
<form onSubmit={handleSubmit} className="flex flex-col flex-1 min-h-0">
  <div className={clsx(
    "flex flex-1 min-h-0 overflow-hidden",
    isEditing && editId ? "flex-col lg:flex-row" : "flex-col"
  )}>
    {/* 左欄：訓練計畫表單 */}
    <div className="flex-1 overflow-y-auto p-6 space-y-4 lg:border-r border-gray-100">

      {/* 不含 PlanMaterialsSection */}
```
