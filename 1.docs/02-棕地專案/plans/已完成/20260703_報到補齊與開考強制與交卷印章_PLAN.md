## 1. 目的

1. **補齊歷史報到資料**：凡已有交卷紀錄、卻無`attendance_records` 者，以**該員工該訓練計畫的第一次考試時間**補登報到列，使報到統計與實際受測人數一致。
2. **落實報到鐵律**：維持「登入 → 考試中心 → 先報到 → 再開考」流程，**後端強制**、**前端不得放行繞路**，防止再產生「有考試、無報到」缺口。
3. **交卷成績 UI**：交卷後立即顯示分數，並在分數下方加上與成績預覽一致的**通過／未通過印章**（PASS / FAIL）。

## 4. 名詞解釋

| 名詞                     | 說明                                                                                               |
| ------------------------ | -------------------------------------------------------------------------------------------------- |
| `attendance_records`   | 報到紀錄表；`UNIQUE(emp_id, plan_id)`                                                            |
| `exam_records`         | 考試總記錄；含 `start_time`、`submit_time`                                                     |
| `exam_history`         | 每次交卷快照；`submit_time` 可還原**第幾次**交卷                                           |
| **第一次考試時間** | 補登 `checkin_time` 的依據；見 §5.1 取值順序                                                    |
| **假相容**         | `get_attendance_status` 在無報到列時，因有交卷而回傳 `is_checked_in: true` 但不寫 DB（將移除） |

## 5. 作業內容

### 5.1 Phase A — 歷史資料補齊

#### 補登對象

```sql
SELECT er.emp_id, er.plan_id
FROM exam_records er
LEFT JOIN attendance_records ar
  ON ar.emp_id = er.emp_id AND ar.plan_id = er.plan_id
WHERE er.submit_time IS NOT NULL
  AND ar.id IS NULL;
```

#### `checkin_time` 取值順序

1. `MIN(exam_history.submit_time)` — 最早一次交卷（重考後`exam_records.start_time` 會被覆寫，**不可**單靠此欄代表第一次）
2. `exam_records.start_time` — 僅在無`exam_history` 時
3. `exam_records.submit_time` — 最後 fallback

#### 實作

| 項目           | 內容                                                                               |
| -------------- | ---------------------------------------------------------------------------------- |
| 檔案           | `backend/migrations/backfill_attendance_from_exam.py`                            |
| 風格           | 與 `add_attendance_absence_reasons.py` 相同（sqlite3 或 SQLAlchemy，可重複執行） |
| `ip_address` | `NULL`（預設；不寫 `"backfill"` 除非審核另定）                                 |
| 執行前         | 備份 `data/education_training.db`                                                |
| 執行           | `cd backend && .venv/bin/python3 migrations/backfill_attendance_from_exam.py`    |

#### 驗證

```sql
-- 預期：0
SELECT COUNT(*) FROM exam_records er
LEFT JOIN attendance_records ar
  ON ar.emp_id = er.emp_id AND ar.plan_id = er.plan_id
WHERE er.submit_time IS NOT NULL AND ar.id IS NULL;
```

抽查：員工 `100057` 等先前異常帳號應有報到列，且 `checkin_time` 為第一次考試時間。

### 5.2 Phase B — 後端強制報到檢查

#### B1. `start_exam`

- **檔案**：`backend/app/routers/exam_center.py`
- **位置**：計畫驗證通過後、建立／更新`ExamRecord`**之前**
- **邏輯**：查無`attendance_records`（當前`emp_id` +`plan_id`）→`HTTP 403`，`detail`:`請先完成報到後再開始考試`
- **不做**：自動建立報到列

#### B2. `submit_exam`

- **檔案**：同上
- **位置**：評分邏輯之前或之後皆可，但須在`commit` 之前拒絕
- **邏輯**：同 B1，無報到列 →`403`（防 API 略過`start_exam` 直接交卷）

#### B3. 移除 `get_attendance_status` 假相容

- **刪除**：約 1292–1302 行「有`exam_records.submit_time` 則回傳`is_checked_in: true`」區塊
- **行為**：僅依真實`attendance_records` 回傳

#### 共用輔助（建議）

```python
def _require_attendance_record(db, emp_id: str, plan_id: int) -> None:
    """無報到列則 raise HTTPException(403)"""
```

供 `start_exam`、`submit_exam` 呼叫，避免重複查詢邏輯。

#### 重考情境

- 已報到且`attendance_records` 已存在 → 不重複建立、不更新`checkin_time`
- backfill 後舊考生重考 → 沿用補登列，**不需再報到**

## 7. 執行順序與部署注意

```
1. 備份 data/education_training.db
2. Phase A：執行 backfill + 驗證 SQL（缺口數 = 0）
```
