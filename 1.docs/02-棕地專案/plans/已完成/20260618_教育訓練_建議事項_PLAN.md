## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| **互動傳輸（interactive）** | 教材上傳／下載；使用者提供 NAS/AD 帳密，每次傳輸建立 SMB Session |
| **服務帳號（service）** | 考卷 TXT 讀寫 NAS；憑證來自環境變數，使用者不輸入 |
| **備份帳號（backup）** | 排程備份 job 專用；憑證存於排程設定（加密） |
| **Close Session** | SMB 連線結束、釋放 handle；禁止長期掛載 |
| **file_transfer_audit_logs** | 檔案傳輸稽核表；記錄 emp_id、IP、動作、檔案、結果 |
| **QR 方案 A** | QR 圖僅編碼登入頁 URL，不含 UUID token |
| **LDAPS 管理登入** | AD PLAN 範圍；IT 管理者登入 DC，與 NAS SMB 驗證不同協定、不同用途 |

## 5. 作業內容

### 5.1 建議事項 #1：教材上傳／下載

**範圍**：僅**教材**（`teaching/`、`teaching_materials`）；不含考卷 TXT（見 §5.2）。

**流程**：

```
JWT 已登入 → 觸發上傳或下載（含批次 ZIP）
  → NAS 登入視窗（NAS 本地帳號 或 AD 網域帳號；NAS 已入 DC）
  → 後端 SMB interactive 模式驗證並傳輸
  → 傳輸視窗：進度 %、可取消
  → 成功 → 關閉視窗 → Close SMB Session → 寫入 Audit Log
```

**傳輸視窗規格**：

| 項目 | 規格 |
|------|------|
| 進度 | 顯示傳輸 **%** |
| 取消 | 「取消傳輸」按鈕；`AbortController` 中斷 |
| 關閉方式 | 僅右上角 **X** 或「取消傳輸」；**禁用**點擊遮罩關閉 |
| 防重整 | 傳輸中 `beforeunload` 警告（F5／Ctrl+R） |
| 完成 | 顯示傳輸成功後關閉 |

**Audit Log**（`file_transfer_audit_logs`）至少記錄：`emp_id`、`client_ip`、`nas_username`、`action`（upload/download/cancel）、`resource_type=teaching_material`、`plan_id`、`filename`、`bytes`、`status`、`error_message`。

詳細 API、前端元件見 [`20260617_教材上傳列管與教材庫_PLAN.md`](../20260617_教材上傳列管與教材庫_PLAN.md) §5.4a。

### 5.2 建議事項 #1A：考卷工坊 TXT 上傳（討論補充）

- **不要求**使用者 NAS 登入（避免考卷工坊操作過於繁瑣）。
- 後端以**服務帳號**（環境變數 `EXAM_SMB_*` 或共用 `SERVICE_SMB_*`）短連線寫入 `{plan_id}/exams/`。
- **Audit Log**：記 `emp_id`（JWT）、`client_ip`、檔名、`plan_id`；`nas_username` 為 null 或標記 `service`。
- 讀取（列表／預覽）同服務帳號 + Audit（若有下載行為）。

### 5.3 建議事項 #2：SMB 不得長期連線

- 系統**禁止**啟動時永久 SMB 掛載或維持永久連線。
- 儲存層介面：`connect(credentials) → operation → disconnect()`。
- 三種 credentials 來源見 NAS PLAN §5.3。

### 5.4 建議事項 #2A：排程備份專用 NAS 帳號（討論補充）

- 於**排程管理介面**新增：`backup_nas_username`、`backup_nas_password`（加密儲存）。
- 僅排程備份 job 使用；與互動傳輸、考卷服務帳號分離。
- job 開始 connect → 備份完成 disconnect。

### 5.5 建議事項 #3：QR Code（方案 A）

| 項目 | 定案 |
|------|------|
| QR 圖內容 | `http://{server IP}/training/login`（含部署子路徑；IP 為系統 server） |
| 掃碼後 | 一般登入頁 → 員工編號 + 圖形驗證碼 → JWT |
| 長碼 UUID | **不再嵌入 QR 圖**；`login_tokens`／token QR 流程廢止或僅保留歷史 |
| 有效時間 | **前端不顯示**（`QRCodeManager`、`QRCodeLoginPage` 移除 `expires_at` 顯示） |
| Safari | 短 URL 避免長 UUID 觸發安全警告 |

**取捨**：失去 Admin 場次 token 控管；任何知悉登入頁網址者皆可登入（與直接開瀏覽器相同）。

### 5.6 建議事項 #4：行動裝置 UI

- 提示詞：`跨平台＋行動裝置＋iOS-Android`；**僅調整 layout**（Tailwind responsive、safe-area），**不變**商業邏輯與 API。
- **第一階段**：考試中心（`ExamDashboard`）、成績中心。
- **第二階段**：其餘頁面依第一階段模式擴展。
- 成熟後將作法寫成 **Cursor skill**，供日後呼叫。

### 5.7 建議事項 #5：訓練計畫—受課對象 UX

現行 `TrainingPlanManager` 與下列規格不一致，需改前端（必要時對齊後端儲存語意）：

| # | 規格 |
|---|------|
| 5.7.1 | **個人授課對象**清單預設為**空** |
| 5.7.2 | 勾選**授課單位**後，個人清單**僅顯示**該單位人員；未勾選單位者不顯示 |
| 5.7.3 | 新增「**新增個人對象**」按鈕，可跨單位指定個人 |
| 5.7.4 | 透過「新增個人對象」加入者，顯示於**個人授課對象卡片**（建議 badge 區分「來自單位」vs「額外新增」） |

**後端語意**（技術設計定案）：`plan_target_departments` 與 `plan_target_users` 如何表達「單位內全員 implicit」vs「明確個人 explicit」須與 UX 一致。

### 5.8 建議事項 #6：考試中心按鈕失效（P0）

**現象**：進入考試中心 → 跳至其他頁 → 返回考試中心 →「立即報到／開始考試」等按鈕不可按；需 F5 恢復。

**原因**：`ExamDashboard`／`CheckInButton` 僅 mount 時 fetch；modal／狀態可能殘留。

**修復方向**：

1. `useLocation()`：pathname 回到 `/` 時 **refetch** 考試列表與報到狀態。
2. 離開頁面時 reset `showNotCheckedInModal` 等狀態。
3. 必要時 `key={location.key}` 強制 remount（治標）。

### 5.9 建議事項 #7：security-review

- 各項實作完成、合併前，於 Claude Code 執行 **`/security-review`**。
- 產出存放：`1.docs/02-棕地專案/reviews/`。
- 重點：NAS 憑證傳遞、Audit、QR 簡化、受課對象權限。

### 5.10 驗收條件（本 PLAN 總覽）

| # | 案例 | 期望 |
|---|------|------|
| 1 | 教材上傳未 NAS 登入 | 拒絕或先彈登入窗 |
| 2 | 教材 NAS 帳密錯誤 | 失敗提示；不寫 NAS |
| 3 | 教材傳輸成功 | Audit 有記錄；SMB Session 已關閉 |
| 4 | 考卷 TXT 上傳 | 不彈 NAS 登入；Audit 有 emp_id + IP |
| 5 | 排程備份 | 使用排程設定之專用帳號；job 後 disconnect |
| 6 | 系統 idle | 無殘留 SMB 長連線 |
| 7 | QR 掃碼 | 進入 `/training/login`；QR 圖無 UUID |
| 8 | QR 前端 | 不顯示有效時間 |
| 9 | 考試中心來回導航 | 按鈕可正常操作，無需 F5 |
| 10 | 受課對象 UX | 符合 §5.7 四點 |
| 11 | 行動裝置 | 考試中心、成績中心可讀可操作 |
| 12 | security-review | 已執行並有 review 文件 |

細部教材驗收見教材 PLAN §5.11；NAS 驗收見 NAS PLAN §5.9。

## 8. 風險與待確認

| 項目 | 說明 |
|------|------|
| NAS 密碼經後端轉 SMB | 須 HTTPS；密碼不寫 DB；僅當次或短時 session |
| 三種 NAS 帳號權限 | IT 須在 NAS 劃分互動、考卷服務、備份帳號權限 |
| QR 方案 A | 失去場次 token 控管 |
| 受課對象後端語意 | 單位勾選與 `target_user_ids` 儲存策略須技術設計定案 |
| 考卷僅 Audit | 無法對應「誰的 AD 帳號寫入 NAS」，僅能追蹤系統 emp_id |
| 實作順序 | 建議 P0 先修 #6，再進行 NAS／QR 大改 |
