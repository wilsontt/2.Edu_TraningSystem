## 1. 目的

1. 將考卷工坊上傳之 **TXT 考卷** 與 **教材（PDF 等）** 統一存放於**遠端 NAS（SMB）**，本地不再保留副本。
2. 提供**排程備份**：依設定時間，將 **`education_training.db` ＋ 當日 materials 快照** 打包為單一 ZIP，存至 NAS。
3. 提供**排程設定介面**：可設定備份時間與**保留份數**（rotation），超出份數自動清除最舊備份。
4. 順帶修正既有**路徑分裂**問題（`backend/data/materials` vs 根 `data/`），並將參考用 PDF 由 `data/教材/` 移出至文件區。

## 2. 範圍

### 2.1 涵蓋範圍（In Scope）

- 上傳檔案儲存後端抽象化：本地檔案系統 → **SMB（NAS）**。
- 考卷與教材**統一目錄骨架**（同計畫下分 `exams/`、`teaching/`；`teaching/` 子結構見教材 PLAN）。
- 排程備份服務：DB ＋ materials 快照 → ZIP → NAS；保留份數輪替。
- 排程設定之**管理介面**與設定持久化。
- 還原（restore）作業 SOP（文件化；還原可半自動）。

### 2.2 不涵蓋範圍（Out of Scope）

- NAS 本身之 RAID／硬體備援與機房異地備份（屬基礎設施）。
- 資料庫由 SQLite 改為其他 DBMS（本期維持 SQLite）。
- 邏輯層 JSON 匯出／匯入：**列為選配**（見 5.8），本期不作為主要 DR 手段。
- 教材上傳列管、教材庫、版本處理、下載檔名等業務功能（見教材 PLAN）。

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| **NAS** | 網路儲存設備 |
| **SMB** | Server Message Block，Windows 檔案共享協定（亦稱 CIFS） |
| **materials** | 上傳檔案根目錄，含考卷 TXT 與教材 |
| **快照（snapshot）** | 備份當下 materials 之檔案複本集合 |
| **rotation／保留份數** | 僅保留最近 N 份備份，超出刪最舊 |
| **L1 實體備份** | 直接打包 DB 檔與檔案（本計劃主軸） |
| **L2 邏輯匯出** | DB 結構化資料匯出為 JSON（本計劃選配） |
| **DATA_ROOT** | 執行期資料根路徑（環境變數，統一本地／容器） |
| **interactive 模式** | 教材上傳／下載；credentials 由使用者當次提供（NAS 本地或 AD） |
| **service 模式** | 考卷 TXT 讀寫；credentials 來自環境變數 `EXAM_SMB_*` |
| **backup 模式** | 排程備份 job；credentials 來自排程設定 `backup_nas_*` |
| **Close Session** | SMB 連線結束、釋放 handle；禁止長期維持連線 |

## 5. 作業內容

### 5.1 統一儲存目錄結構

NAS 上（SMB 掛載點之下）採：

```
{MATERIALS_ROOT}/
└── {year}/
    └── {plan_id}/
        ├── exams/      # TXT 考卷原始檔
        └── teaching/   # 教材實體檔（子目錄結構見教材 PLAN §5.1）

{BACKUP_ROOT}/
└── education_training_backup_{YYYYMMDD_HHmm}.zip
```

> `MATERIALS_ROOT`、`BACKUP_ROOT` 均指向 NAS（SMB）路徑，以環境變數設定。

### 5.2 路徑統一（修正既有債）

- `backend/app/routers/exam.py` 現用相對路徑 `Path("data/materials")`，依啟動 cwd 落在 `backend/data/materials`，與 `database.py` 的根 `data/` 分裂。
- **改為**：讀取環境變數 `MATERIALS_ROOT`（NAS）作為唯一根；本地不再寫檔。
- `database.py` 之 DB 路徑維持專案根 `data/`（DB 仍為本地單檔，由備份服務上傳 NAS）。

### 5.3 儲存抽象層（SMB）

- 新增儲存服務（如 `backend/app/services/storage.py`），介面：`connect(credentials) → save / open / list / delete → disconnect()`。
- **禁止**系統啟動時永久 SMB 掛載或維持永久連線；每次操作結束須 **Close Session**。
- 後端實作建議採 **程式內 SMB 客戶端**（如 `smbprotocol`），以利按請求建立／釋放連線。若採 OS 掛載，僅限 job 期間掛載，不得常駐。
- **三種連線模式**（credentials 來源）：

| 模式 | 用途 | credentials 來源 | 使用者 NAS 登入 |
|------|------|------------------|-----------------|
| `interactive` | 教材上傳／下載 | 前端傳入 NAS 本地或 AD 帳密（HTTPS） | **每次傳輸要** |
| `service` | 考卷 TXT 讀寫 | 環境變數 `EXAM_SMB_*`（或 `SERVICE_SMB_*`） | **不要** |
| `backup` | 排程備份 ZIP | 排程設定 `backup_nas_username`／`backup_nas_password` | 不適用 |

- 考卷上傳流程（`/admin/exams/upload` 等）以 **service** 模式寫入 `exams/`；**移除本地落地**；寫入後 **Audit**（`emp_id` + IP，見建議事項 PLAN §5.2）。
- 教材實體檔以 **interactive** 模式透過教材 PLAN API 寫入 `teaching/`。
- 每次傳輸（含失敗／取消）寫入 `file_transfer_audit_logs`（詳見建議事項 PLAN §7.1）。

### 5.4 教材實體檔與業務規格（交叉引用）

- 本 PLAN 僅確保 `teaching/` 目錄可透過儲存層讀寫；**不**定義教材上傳欄位、類型、版本、教材庫 UI。
- 教材完整業務規格見 [`20260617_教材上傳列管與教材庫_PLAN.md`](../20260617_教材上傳列管與教材庫_PLAN.md)。
- 備份 ZIP 含 DB 時，自然包含 `teaching_materials` 表與 `teaching/` 下實體檔。

### 5.5 排程備份服務

- 新增備份服務，工作內容：
  1. 對 SQLite 執行一致性備份（`sqlite3 .backup` 或線上備份 API，避免寫入中複製）。
  2. 擷取**當日 materials 快照**（自 NAS materials 複製，或就地打包）。
  3. 打包 DB ＋ 快照為單一 ZIP，含 `manifest.json`（時間、schema 版本、檔案清單、來源路徑）。
  4. 上傳 ZIP 至 `BACKUP_ROOT`（NAS）。
  5. 依**保留份數**刪除最舊 ZIP。
  6. 寫入備份紀錄（成功／失敗、檔名、大小、耗時）。
- 排程器選型（技術設計定案）：`APScheduler`（隨後端進程）或主機層 `cron`／工作排程器。建議 **APScheduler**，以利由設定介面動態調整。

### 5.6 排程設定介面（管理端）

- 新增設定頁（需 `系統管理` 權限），可設定：
  - 是否啟用排程備份
  - 備份頻率與時間（每日／每週＋時:分）
  - 保留份數（N）
  - NAS 目的地（唯讀顯示或可編輯，視權責）
  - **排程備份專用 NAS 帳號**（`backup_nas_username`、`backup_nas_password`；密碼加密儲存，不可明文入版控）
- 設定持久化：新增設定表或沿用既有設定機制（技術設計定案）。
- 提供「**立即備份一次**」按鈕與「**備份紀錄**」清單。
- 「立即備份」與排程 job 均使用 **backup** 模式帳號；job 結束須 disconnect。

### 5.7 設定環境變數

| 變數 | 用途 | 範例 |
|------|------|------|
| `DATA_ROOT` | 執行期本機資料根（DB volume；Docker） | `/data`（容器內；非 SMB） |
| `SMB_SERVER` | SMB 主機名或 IP（不可含路徑） | `10.9.82.22` |
| `SMB_SHARE` | SMB **共享名稱**（建議不含 `/`） | `CrownWork` |
| `MATERIALS_ROOT` | 共享內教材／考卷**邏輯相對路徑**（一律 `/` 分段） | `教育訓練教材及考卷/materials` |
| `BACKUP_ROOT` | 共享內備份**邏輯相對路徑** | `教育訓練教材及考卷/backups` |
| `EXAM_SMB_USERNAME`／`EXAM_SMB_PASSWORD` | 考卷 TXT **service** 模式 | 環境變數／密鑰管理 |
| `backup_nas_*` | 排程備份 **backup** 模式 | 排程設定表（加密），見 §7.1 |

> **路徑慣例（Win／macOS／Linux Docker 共用）**：`MATERIALS_ROOT`／`BACKUP_ROOT` **不是**本機掛載點或 UNC，而是 SMB 共享內相對路徑；禁止 `D:\...`、`/mnt/...`、`\\server\share\...`。UNC 僅由 `storage._unc()` 組出。詳見 [NAS與路徑跨平台慣例](../../00-專案總覽/NAS與路徑跨平台慣例.md)。
>
> SMB 帳密屬機敏，須走環境變數／密鑰管理或加密設定表，不可明文入庫、不可進版控。
> **interactive** 模式之使用者 NAS 密碼僅用於當次連線，不寫入 DB。
> 開發用 `backend/.env`（以 `config.py` 檔案位置載入）；生產 Docker 以環境變數注入（映像不含 `.env`）。

### 5.8 邏輯匯出／匯入（選配，非本期主軸）

- 如未來需跨環境遷移或單表修復，可再提供 `GET /admin/backup/export`（JSON full）與 `POST /admin/backup/import`（dry_run／merge／replace）。
- 註記：JSON **不含**教材實體檔，實體檔一律依賴 NAS 與 ZIP 快照。

### 5.9 驗收條件

| # | 案例 | 期望 |
|---|------|------|
| 1 | 上傳 TXT 考卷 | 落在 NAS `{plan_id}/exams/`，本地無副本 |
| 2 | 教材實體檔寫入 | 透過儲存層可寫入 `{plan_id}/teaching/`（業務驗收見教材 PLAN） |
| 3 | 考卷 materials 列表／預覽／刪除 | `exams/` 子目錄行為正確 |
| 4 | 設定每日 02:00、保留 7 份 | 屆時自動產生 ZIP 至 NAS |
| 5 | 連續備份至第 8 份 | 自動刪除最舊，恆維持 7 份 |
| 6 | 「立即備份」 | 即時產生 ZIP 並記錄 |
| 7 | ZIP 內容 | 含可還原之 DB ＋ materials 快照 ＋ manifest |
| 8 | 還原演練 | 依 SOP 由 ZIP 還原後系統可正常運作 |
| 9 | NAS 不可達 | 備份失敗有明確紀錄與告警，不影響線上服務 |
| 10 | `data/教材/` | 已移至 `1.docs/00-專案總覽/參考教材/`，文件連結同步 |
| 11 | 教材上傳未 NAS 登入 | 拒絕（interactive 模式） |
| 12 | 考卷 TXT 上傳 | 不彈 NAS 登入；service 模式成功；Audit 有 emp_id |
| 13 | 排程備份執行 | 使用排程設定之 backup 帳號；job 後無殘留連線 |
| 14 | 系統 idle | 無長期 SMB 掛載或永久連線 |

### 5.10 還原 SOP（概要，細節於技術設計補完）

1. 自 NAS 取得目標 ZIP，校驗 `manifest.json`。
2. 停止後端寫入。
3. 還原 `education_training.db`（覆蓋前先另存現況）。
4. 還原 materials 至 `MATERIALS_ROOT`。
5. 啟動後端，依驗收案例抽測。

## 7. 使用表單（欄位說明）

### 7.1 備份排程設定 `BackupScheduleConfig`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `enabled` | boolean | 是 | 是否啟用排程備份 |
| `frequency` | string | 是 | `daily` / `weekly` |
| `time_of_day` | string | 是 | `HH:mm`（24 小時） |
| `weekday` | int | 否 | `frequency=weekly` 時，0–6 |
| `retention_count` | int | 是 | 保留份數 N（≥1） |
| `destination` | string | 是 | NAS 備份路徑（`BACKUP_ROOT`） |
| `backup_nas_username` | string | 是 | 排程備份專用 NAS 帳號（本地或 AD） |
| `backup_nas_password` | string | 是 | 加密儲存；設定／更新時傳入，回應不返回明文 |

### 7.2 備份紀錄 `BackupRecord`

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | int | 主鍵 |

| `created_at` | datetime | 產生時間 |
| `size_bytes` | int | 檔案大小 |
| `status` | string | `success` / `failed` |
| `message` | string | 失敗原因或摘要 |

### 7.3 備份包內 `manifest.json`

| 欄位 | 型別 | 說明 |
|------|------|------|
| `schema_version` | string | 結構版本，還原相容性判斷 |

| `materials_count` | int | 快照檔案數 |
| `files` | string[] | 快照檔案清單 |

## 8. 風險與待確認

| 項目 | 說明 |
|------|------|
| SMB 方案 | 建議 `smbprotocol` 按需連線；禁止常駐 OS 掛載 |
| 三種帳號權限 | IT 須在 NAS 劃分互動、考卷服務、備份帳號之共享權限 |
| 使用者 NAS 密碼 | interactive 模式經 HTTPS 傳後端，不落地；安全審查見建議事項 #7 |
| SQLite 一致性 | 備份須用 `.backup`／線上備份 API，避免寫入中複製造成毀損 |
| 大檔與時間 | materials 增大後 ZIP 時間／空間成本；必要時改增量備份 |
| NAS 認證機敏 | SMB 帳密需密鑰管理，嚴禁入庫／入版控 |
| 無本地副本風險 | 本地不留檔，NAS 不可達時上傳將失敗；需明確錯誤與告警 |
| 告警機制 | 備份失敗如何通知（Log／Email／前端紅點）待業主決定 |
| 還原權限 | 還原為高風險操作，是否限 break-glass 或 IT 手動執行待定 |
