**文件類型**：棕地實作計劃

**最後修訂**：2026-07-11（follow-up：部門 details 套用 `plan_status`）

**關聯議題**：

1. 成績中心「總覽」「學習分析」與「歷史記錄」數字對不起來（含個人視角與部門成績跨人查詢）

## 1. 目的

1. **統一個人三頁籤資料範圍**：「總覽」「歷史記錄」「學習分析」在**同一員工、同一訓練計畫狀態篩選**下，後端統計應基於**同一批 `ExamRecord`**，避免切換頁籤時數字互相矛盾。
2. **修正學習進度分母**：學習分析「總計畫數」應涵蓋該員**實際應考範圍**（部門受課 **＋** 個人指定受課），與報到總覽受課對象邏輯一致。
3. **釐清並標示計數單位**：畫面區分「**考試場次**」（`ExamRecord` 筆數）與「**訓練計畫數**」（不重複 `plan_id`）。
4. **補強個人頁前端請求一致性**：總覽、學習分析於 `empId`／篩選／`trend_period` 變更時取消舊請求（`AbortController`）。
5. **修復部門成績「計畫統計」展開**：`GET /admin/reports/plan` 回傳 `plan_id`，使點選計畫列可載入 details。
6. **消除 `ReportDashboard` React 警告**：績效表 `.map()` 改為帶 `key` 的 `Fragment`。
7. **定稿跨部門／跨人導覽**：進部門成績清個人檢視、從部門點個人帶入 `plan_status`、URL merge、標題統一、範圍外 403。

### 5.5 議題 E：部門成績「計畫統計」點擊無反應

#### 5.5.1 現況與根因

**現象**：部門成績 →「計畫統計」→ 點任訓練計畫列，無展開、無載入考生列表。「各部門統計」正常。

**根因**：`GET /admin/reports/plan`（`get_plan_statistics`）回應**未含 `plan_id`**：

```976:982:backend/app/routers/report.py
            stats.append({
                "name": r.title,
                "date": r.training_date,
                "count": total,
                "avg_score": round(r.avg_score or 0, 1),
                "pass_rate": round(pass_rate, 1)
            })
```

查詢有 `group_by(TrainingPlan.id)` 但 **SELECT 未取 `TrainingPlan.id`**。  
對照部門統計有 `dept_id`（約 L772）。

**前端連鎖失效**（`ReportDashboard.tsx` L1228–1240）：`plan_id` 為 `undefined` → `if (!itemId) return` → 無展開、不呼叫 details。

#### 5.5.2 修改

**檔案**：`backend/app/routers/report.py` → `get_plan_statistics`

```python
results = db.query(
    models.TrainingPlan.id,           # 新增
    models.TrainingPlan.title,
    models.TrainingPlan.training_date,
    func.count(models.ExamRecord.id).label("count"),
    ...
).join(...).filter(plan_status_expr)
# ...
stats.append({
    "plan_id": r.id,                  # 新增
    "name": r.title,
    "date": r.training_date.isoformat() if r.training_date else None,
    "count": total,
    "avg_score": round(r.avg_score or 0, 1),
    "pass_rate": round(pass_rate, 1),
})
```

**前端**（`ReportDashboard.tsx`）：

- 「查看個人成績」`<Link>` 加 `plan_status`（R11）
- 切換「各部門統計」↔「計畫統計」時清 `expandedDept`／`expandedPlan`
- `PlanStat.plan_id` 必填語意；缺則不可展開

**不需修改**：`GET /admin/reports/plan/{plan_id}/details` 已存在。

### 5.6 議題 F：`ReportDashboard` React `key` 警告

#### 5.6.1 現況與根因

`paginatedStats.map` 回傳無 key 的 `<>...</>`；內層 `<tr key={idx}>` 對 map 無效。僅在部門成績掛載且有資料時出現。

```tsx
import { Fragment } from 'react';

const rowKey = activeTab === 'department'

  : `plan-${(item as PlanStat).plan_id ?? idx}`;

return (
  <Fragment key={rowKey}>
    <tr>...</tr>
    {isExpanded && itemId && <tr>...</tr>}
  </Fragment>
);
```

## 10. 風險與注意事項

| 項目 | 說明 |
|------|------|
| 歷史記錄預設範圍縮小 | 個人歷史預設改為「進行中」；須 UAT 確認；或改 R1 預設為 `all` |
| 計畫統計 `date` 格式 | 後端若改 ISO 字串，確認前端 `toLocaleDateString` 仍正常 |
| 計畫統計進階欄位 | `completion_rate`、`effectiveness_grade`、`popularity_score` 仍可能為 `-`；非本次範圍 |
| **部門 details 無 plan_status** | ✅ 已於 2026-07-11 follow-up 補齊（details 帶 `plan_status`，切換狀態清 `deptDetails`） |

| URL merge 遺漏 | Admin 選人若整包覆寫會弄丟 `plan_status`／違反 R6a；實作必用 `prev` merge |
