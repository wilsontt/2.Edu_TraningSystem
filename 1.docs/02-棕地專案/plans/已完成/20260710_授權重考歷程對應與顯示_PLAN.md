## 1. 目的

1. 讓「授權開放重考」在**考試歷程**可正確對應到**第 N＋1 次交卷**（非列表上的第 N 次）。
2. 在成績中心「考試歷程記錄」顯示**授權者**；「成績詳情」顯示授權者、時間、原因。
3. 釐清權限：部門範圍與跨部門範圍皆可授權；**不需要** Admin／系統管理者自我授權（管理帳號不應受訓考試）。

## 2. 範圍

### 2.1 涵蓋（In Scope）

| 項目 | 內容 |

### 2.2 不涵蓋（Out of Scope）

| 項目 | 說明 |
|------|------|

| 授權紀錄獨立管理頁 | 本次不做列表／查詢頁 |
| 變更「未通過可自動重考」 | 維持既有 |
| 變更 Admin `is_trainee=false` 隔離 | 維持管理帳號不進考試中心；本 PLAN 僅拿掉自我授權例外 |

### 2.3 業務規則（定稿）

| # | 規則 |
|---|---|
| R1 | 成績中心列表一列 = 該員×該計畫**最後一次交卷**（第 N 次） |
| R2 | 「開放重考」准許再考一次 → 對應即將發生的 **第 N＋1 次** |
| R3 | 一次授權 = 一次重考；交卷後消耗授權 |
| R4 | 歷程「授權者」掛在 **N＋1 那一列**（綁 `exam_history.id`） |
| R5 | 詳情顯示：授權者、時間、原因；**不顯示**是否已消耗 |
| R6 | 授權資格與成績中心「部門成績」一致：`menu:report` **且** `role_scope_type` 為 `all` 或 `department`（或 `Admin`）；**且**目標在 `_can_view_emp_id` 範圍內；**不**另以 `btn:exam:authorize-retake` 作為顯示／API 門檻 |
| R7 | **禁止自我授權**（含 Admin／系統管理者）；管理帳號本就不應有受訓考試成績需自我授權 |

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| 第 N 次 | 授權當下列表／`exam_records` 顯示的最後一次交卷 |
| 第 N＋1 次 | 授權後學員實際再考並交卷的那一次（新 `exam_history`） |
| `consumed_history_id` | 授權被消耗時，對應那次交卷的 `exam_history.id` |
| 是否已消耗 | `consumed_at` 是否有值；本 PLAN **不在成績詳情 UI 顯示** |
| 授權者 | `authorized_by`（建議 API 一併回傳姓名供顯示） |

#### 5.1.3 既有資料回填（建議，可選但建議做）

執行前備份 `data/education_training.db`。

### 5.2 後端邏輯

#### 5.2.1 `submit_exam`（消耗授權）

現況：設 `consumed_at = now`。  
調整：在 `db.add(history_record)` 並 `flush` 取得 `history_record.id` 後：

```python
pending_auth.consumed_at = now
pending_auth.consumed_history_id = history_record.id
```

#### 5.2.2 歷程 API（供考試歷程 Modal）

擴充回傳 `history[]` 每一筆（至少下列之一端點，以現有 `get_exam_record_detail`／組裝 history 的路徑為準）：

```json
{
  "id": 12,
  "submit_time": "...",
  "total_score": 80,
  "is_passed": true,
  "retake_authorization": null
}
```

或有授權時：

```json
"retake_authorization": {
  "authorized_by": "B002",
  "authorized_by_name": "王主管",
  "authorized_at": "2026-07-09T11:00:00",
  "reason": "稽核複測"
}
```

對應方式：`ExamRetakeAuthorization.consumed_history_id == history.id`。  
無對應 → `null`（前端顯示「—」）。

> 詳情 API **可不回傳** `consumed_at`／「是否已消耗」給前端顯示用（內部欄位可保留）。

#### 5.2.3 單次歷程詳情 `GET /exam/history/{history_id}`

同樣附上該次的 `retake_authorization`（同上結構，**不含**是否已消耗的 UI 欄位）。

#### 5.2.4 移除 Admin 自我授權例外

**檔案**：`backend/app/routers/exam_center.py`（`authorize_retake`）

**改回**：

```python
# 禁止自我授權（管理帳號不應受訓；一般主管亦不可授權自己）
if req.emp_id == current_user.emp_id:
    raise HTTPException(
        status_code=400,
        detail="不可授權自己重考，須由其他主管或稽核人員執行",
    )
```

**刪除**「Admin／系統管理者允許自我授權」分支。

#### 5.2.5 權限（2026-07-10 定稿對齊）

**單一模型**（與「部門成績」tab 可見規則一致，不另開功能碼門檻）：

| 條件 | 說明 |
|------|------|
| 資格 | `role === 'Admin'`，或（`menu:report` **且** `role_scope_type ∈ { all, department }`） |
| 操作邊界 | 目標 `emp_id` 須通過 `_can_view_emp_id` |
| 跨部門／全域 | 可授權範圍內全部員工 |
| 部門範圍（主管、稽核等） | 可授權，限範圍內員工 |
| 自我授權 | 一律禁止（R7） |

`btn:exam:authorize-retake` 功能碼**保留於** `system_functions`（供權限管理 UI 顯示／未來細調），**不作**開放重考按鈕或 API 的必備條件。

**後端**：`exam_center._has_authorize_retake_permission(db, user)` 與 `access_scope.get_role_scope_type` 對齊 `GET /auth/me` 的 `role_scope_type`。

**前端**：`PersonalScorePage` 的 `canAuthorizeRetake` 與 `canViewTeamReport` 使用相同判斷。

## 11. 權限對齊修正（2026-07-10）

### 11.1 問題

初版實作誤將「開放重考」與「部門成績可視」拆成兩套權限：前者要求 `btn:exam:authorize-retake`（或 Admin），導致總稽核等已具跨部門看成績權限者看不到按鈕。

### 11.2 定稿（與 7/10 討論一致）

1. **具有跨部門／全域資料範圍**（`role_scope_type = all`）且具 `menu:report` → 可授權範圍內任何人。
2. **部門範圍角色**（`role_scope_type = department`，如主管、稽核）→ 也可授權，限範圍內員工。
3. **Admin** → 維持 bypass（與部門成績 tab 相同）。
4. 實際授權時仍檢查 `_can_view_emp_id`；禁止自我授權（R7）。

### 11.3 程式落點

| 檔案 | 變更 |
|------|------|
| `backend/app/access_scope.py` | 新增 `get_role_scope_type()`（與 `/auth/me` 一致） |
| `backend/app/routers/exam_center.py` | `_has_authorize_retake_permission(db, user)` 改為範圍模型 |
| `backend/app/routers/auth.py` | `/me` 改用 `get_role_scope_type` |
| `frontend/.../PersonalScorePage.tsx` | `canAuthorizeRetake` 與 `canViewTeamReport` 同判斷 |

## 12. 考試時間語意對齊（2026-07-10）

### 12.1 問題

`exam_center` 曾使用 `_now_taipei_naive()` 寫入交卷／授權／報到時間，但前端 `parseBackendDateTime()` 假設後端為 **UTC naive**，導致顯示多 8 小時（例如下午交卷顯示為隔天凌晨）。

### 12.2 定稿

| 項目 | 規則 |
|------|------|
| 後端寫入 | `_now_utc_naive()`（`datetime.utcnow()`） |
| 前端顯示 | `parseBackendDateTime()` → `toLocaleString('zh-TW')` |
| 涵蓋 | `submit_exam`、`start_exam`、`check_in`、`authorize_retake`、`revoke_retake` |
| 測試 | `tests/test_exam_time_semantics.py` |

### 12.3 程式落點

- `backend/app/routers/exam_center.py`：`_now_utc_naive` 取代 `_now_taipei_naive`
- `frontend/.../DeptMemberScoreModal.tsx`、`BatchPrintPage.tsx`：改用 `parseBackendDateTime`
- `1.docs/00-專案總覽/資料庫遷移/MIGRATION_GUIDE.md`：§考試中心時間語意
