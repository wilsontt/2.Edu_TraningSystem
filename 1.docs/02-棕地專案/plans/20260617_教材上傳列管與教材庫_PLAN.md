## 1. 目的

1. 建立教材**套組**（DB 中繼資料）與 NAS 實體檔的雙層架構：一標題可多檔、可綁多個訓練計畫，達成列管與搜尋取用。
2. 支援**分批、分日、分檔**累加（編輯套組內新增檔案）；**不做版本鏈**。
3. 提供**教材庫**：套組／檔案雙檢視、跨計畫搜尋（含**訓練計畫名稱**）、跨頁勾選、批次下載；下載檔名為 `original_filename`。
4. 規格以**本棕地 PLAN 為唯一準據**，與綠地需求文件脫鉤。

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| **教材套組（Set）** | 一筆業務教材：標題、類型、標籤等；底下含多個檔案 |
| **套組內檔案（File）** | 屬於某套組的單一實體檔與其中繼資料 |
| **通用教材** | 套組未綁定任何訓練計畫（`set_plans` 為空） |
| **使用中** | 套組或檔案 `is_active=true`；列表／下載預設僅含使用中 |
| **original_filename** | 上傳原始檔名；下載檔名來源 |
| **有效單檔上限** | min(系統硬上限, 類型上限, 格式上限) |

### 5.4 上傳／下載限額

#### 5.4.1 單檔大小（三層取最小值）

```
有效單檔上限 = min(
  ① TEACHING_MATERIAL_MAX_FILE_BYTES（系統硬上限）,
  ② material_types.max_file_bytes（該次選定教材類型）,
  ③ material_file_formats.max_file_bytes（該檔副檔名）
)
```

| 層級 | 設定位置 | 角色 |
|------|----------|------|
| ① 系統硬上限 | 環境變數 | 天花板（預設 1 GiB） |
| ② 教材類型上限 | 教材主檔 → 教材類型 | 業務微調（如操作手冊 50MB） |
| ③ 允許格式上限 | 教材主檔 → 允許格式 | 依副檔名（如 md 5MB） |

上傳失敗時錯誤訊息應標明觸頂來源（類型或格式）。範例：簡報(20MB)+MD(5MB)→實際 **5MB**。

#### 5.4.2 單次請求與批次（與單檔上限分開）

| 項目 | 預設 |
|------|------|
| 單次上傳檔數 | ≤ 5 |
| 單次上傳總量 | ≤ 1 GiB |
| 套組累積檔數 | **不設硬上限** |
| 批次下載檔數 | ≤ 10 |
| 批次下載總量 | ≤ 1 GiB |
| ZIP 產生逾時 | ≤ 120 秒 |

環境變數：`TEACHING_MATERIAL_MAX_FILE_BYTES`、`TEACHING_MATERIAL_MAX_BATCH_UPLOAD_*`、`TEACHING_MATERIAL_MAX_BATCH_DOWNLOAD_*`、`EXAM_TXT_MAX_FILE_BYTES`。

### 5.4a 教材傳輸安全與 NAS 驗證

（Wave 1 已實作，Wave 2 沿用）每次上傳／下載前 NAS 登入、`nas_session_token`、FileTransferModal、Audit、`file_transfer_audit_logs`。詳見建議事項 PLAN §5.1。

### 5.7 下載與檔名（Wave 1／2 共用）

| 場景 | 規則 |
|------|------|
| 單檔下載 | `original_filename`（`filename*` RFC 5987） |

## 5.12 教材套組（Wave 2，2026-07-09 核可）

### 5.12.1 儲存目錄結構

```
{MATERIALS_ROOT}/
└── {year}/
    └── sets/
        └── {set_id}/
            └── teaching/
                └── {material_type_slug}/
                    └── {file_id}.{ext}
```

- 多計畫綁定時**共用同一份 NAS 檔**（不依 plan_id 複製）。
- `exams/` 考卷 TXT 路徑不變（見 NAS PLAN）。

### 5.12.2 資料表

#### `teaching_material_sets`（套組主檔）

| 欄位 | 說明 |
|------|------|
| `id` | PK |
| `title` | 顯示名稱（整組共用） |
| `material_type_id` | 教材類型 |
| `description`, `tags` | 選填 |
| `year` | 年度 |
| `uploaded_by`, `uploaded_at` | 建立者／時間 |
| `is_active` | 軟刪整組 |

#### `teaching_material_files`（套組內檔案）

| 欄位 | 說明 |
|------|------|
| `id` | PK；NAS 檔名依此 |
| `set_id` | FK → sets |
| `original_filename`, `stored_filename`, `storage_path` | 檔名與路徑 |
| `file_format`, `file_size_bytes` | 格式與大小 |
| `uploaded_by`, `uploaded_at` | |
| `is_active` | 軟刪單檔 |

#### `teaching_material_set_plans`（套組 ↔ 計畫，M2M）

| 欄位 | 說明 |
|------|------|
| `set_id`, `plan_id` | 複合唯一 |

**通用**：此表無列。  
**綁定後**：列表顯示計畫名稱（多個以逗號或標籤）；**不再顯示「通用」**。

### 5.12.3 操作流程

| 情境 | 行為 |
|------|------|
| 新建套組 + N 檔 | 1 set + N files；套組檢視 **1 列** |
| 再補檔案 | 編輯套組 → 新增檔案（**不因標題相同自動合併**） |
| 編輯套組 | 改標題／類型／標籤；**增刪綁定計畫**；**列出檔案→移除**；**新增檔案** |
| 套組內同名檔 | **是否覆蓋？** Yes→覆寫該 file 之 NAS+中繼資料；No→跳過該檔 |
| 訓練計畫頁上傳 | 建立時**至少綁定該計畫**；編輯可加其他計畫 |
| 教材庫上傳 | 預設 0 計畫（通用）；可選綁一個或多個計畫 |

### 5.12.4 教材庫檢視與搜尋

| 模式 | 內容 |
|------|------|
| **套組檢視** | 一列一套組：標題、檔案數、綁定計畫名稱、類型 |
| **檔案檢視** | 一列一檔：檔名、所屬套組標題、計畫、大小 |

**keyword** 搜尋（兩種模式皆適用）：

- 套組：`title`、`description`、`tags`
- 檔案：`original_filename`
- **訓練計畫：`TrainingPlan.title`**（join `teaching_material_set_plans`）

篩選：`material_type_id`、`file_format`、`plan_id`（含該計畫之套組）、`uploaded_at` 區間。

**跨頁勾選、批次下載確認窗**（Wave 1 行為沿用；勾選對象依檢視模式為 set 或 file，技術設計定案）。

### 5.12.5 API 端點（草案）

Router：`/admin/teaching-materials`（或 `/admin/teaching-material-sets`，技術設計定案）

| 端點 | 說明 |
|------|------|
| `POST /sets` | 建立套組 + 首批檔案 |
| `GET /sets` | 套組列表（`view=set`，分頁篩選） |
| `GET /files` | 檔案列表（`view=file`） |
| `GET /sets/{id}` | 套組詳情 + 使用中檔案列表 |
| `PUT /sets/{id}` | 更新套組中繼資料 |
| `PUT /sets/{id}/plans` | 設定綁定計畫 id 列表 |
| `POST /sets/{id}/files` | 套組內新增檔案（含覆蓋 Yes/No） |
| `DELETE /sets/{id}/files/{file_id}` | 套組內移除單檔（軟刪） |
| `DELETE /sets/{id}` | 軟刪整組 |
| `GET /files/{id}/download` | 單檔下載 |
| `POST /batch-download` | 批次 ZIP（file id 列表） |

**廢止（Wave 2）**：`on_conflict`、`deactivate_and_new`、`replace_in_place`、`conflict-check`（全庫同名）。

### 5.12.6 前端畫面

1. **訓練計畫編輯頁**：套組列表（該計畫綁定之套組）、上傳新套組或編輯既有套組。
2. **教材庫**：套組／檔案檢視切換；上傳（可選計畫）；編輯套組面板（檔案增減、計畫多選）。
3. **系統管理**：教材主檔（沿用）。

### 5.12.7 遷移（自 Wave 1）

每筆舊 `teaching_materials`（`is_active=true`）：

1. 建立 1 `teaching_material_sets`（title、type、plan 等自舊筆）
2. 建立 1 `teaching_material_files`（檔案中繼資料、路徑可保留或搬至 `sets/{set_id}/`）
3. 若舊筆有 `plan_id` → 寫入 `teaching_material_set_plans`
4. 舊表標記遷移完成或唯讀封存

### 5.12.8 驗收條件（Wave 2）

| # | 案例 | 期望 |
|---|------|------|
| S1 | 上傳 3 檔新建套組 | 套組檢視 1 列、檔案數 3 |
| S2 | 編輯套組再新增 3 檔 | 同套組 6 檔，仍 1 列 |
| S3 | 第二次新建同標題套組 | **不**自動合併；為另一套組 |
| S4 | 套組內上傳同名檔，選 No | 舊檔保留，新檔不上傳 |
| S5 | 套組內上傳同名檔，選 Yes | 該檔覆寫，筆數不增 |
| S6 | 通用套組綁定計畫 A | 列表顯示計畫 A 名稱，非「通用」 |
| S7 | 套組綁定計畫 A、B | 兩計畫名稱皆可見；keyword 搜尋任一名稱可找到 |
| S8 | 解除所有計畫綁定 | 恢復顯示「通用」 |
| S9 | 訓練計畫頁上傳 | 至少綁定該計畫 |
| S10 | keyword 搜尋計畫名稱 | 可找到綁定該計畫之套組 |
| S11 | 套組／檔案檢視切換 | 資料一致、檔案數正確 |
| S12 | 編輯移除套組內 1 檔 | 該檔軟刪，其餘不變 |
| S13 | 單檔上限 | 依 min(系統,類型,格式) 拒絕並提示 |
| S14 | 無舊版衝突二選一跳窗 | 全流程不出現 |
| S15 | 遷移後舊教材 | 可搜尋、可下載 |
