## 1. 目的

1. 建立教材**目錄卡**（DB 中繼資料）與 NAS 實體檔的雙層架構，達成「圖書館式」列管：上傳時分類登記、日後可搜尋取用。
2. 支援**分批、分日、分檔**累加上傳；同名衝突時由使用者選擇處理方式。
3. 提供**教材庫**總覽：跨計畫搜尋、跨頁勾選、確認後批次下載；下載檔名恢復為使用者原始檔名。
4. 規格以**本棕地 PLAN 為唯一準據**，與綠地需求文件脫鉤。

## 5. 作業內容

### 5.1 儲存目錄結構

在 NAS 備援 PLAN 定義之 `{year}/{plan_id}/` 之下，`teaching/` 採：

```
{MATERIALS_ROOT}/
└── {year}/
    └── {plan_id}/
        └── teaching/
            └── {material_type_slug}/     # 依教材類型分子目錄
                └── {id}.{ext}            # 例：42.pdf
```

- `material_type_slug` 來自 `material_types.slug`（如 `操作手冊`、`法規與標準`）。
- 教材按**訓練計畫**分目錄，避免所有檔案擠在同一層。
- 備援輔助：ZIP `manifest`（NAS PLAN）+ 可選定期匯出 `teaching_materials_latest.csv` 至 `{BACKUP_ROOT}/catalog/`。

> `exams/`（TXT 考卷）目錄結構見 NAS 備援 PLAN；考卷不走 `teaching_materials` 列管。

> **修訂（2026-07-04）**：白名單改存 `material_file_formats`，由系統管理維護；前後端不得硬編碼副檔名清單。詳見主檔維護 PLAN。

**預設種子（可於系統管理增減）**

| 類別 | 副檔名 | 預設單檔上限 |
|------|--------|--------------|
| PDF | `.pdf` | 50 MB |
| Word | `.doc`, `.docx` | 50 MB |
| Excel | `.xls`, `.xlsx` | 30 MB |
| PowerPoint | `.ppt`, `.pptx` | 50 MB |
| Markdown | `.md` | 5 MB |
| 純文字 | `.txt`（`teaching/` 僅存檔，**不**觸發考卷解析） | 5 MB |

### 5.4 上傳／下載限額

> **修訂（2026-07-04）**：為支援影片上傳調高預設硬上限與批次總量。有效單檔上限 = `min(格式上限, 類型上限, 系統硬上限)`。

| 項目 | 限制 | 備註 |
|------|------|------|
| 教材單檔大小 | 依格式與類型，最高 **1 GiB**（硬上限） | 每檔、每次上傳各自檢查 |
| 單次上傳檔數 | ≤ **5 份** | 僅限該次 HTTP 請求 |
| 單次上傳總量 | ≤ **1 GiB** | 僅限該次 HTTP 請求 |
| 單一計畫歷史累積 | **不設硬上限** | 支援分日分批累加 |
| 單檔下載 | 依 DB `file_size_bytes` | 檔名用 `original_filename` |
| 批次下載檔數 | ≤ **10 份** | |
| 批次下載總量 | ≤ **1 GiB** | 超限拒絕並提示；超大影片建議單檔下載 |
| ZIP 產生逾時 | ≤ **120 秒** | |

環境變數：

| 變數 | 預設值 | 說明 |
|------|--------|------|
| `TEACHING_MATERIAL_MAX_FILE_BYTES` | 1073741824 (1 GiB) | 全系統硬上限 |
| `TEACHING_MATERIAL_MAX_BATCH_UPLOAD_COUNT` | 5 | 單次上傳檔數 |
| `TEACHING_MATERIAL_MAX_BATCH_UPLOAD_BYTES` | 1073741824 (1 GiB) | 單次上傳總量 |
| `TEACHING_MATERIAL_MAX_BATCH_DOWNLOAD_COUNT` | 10 | 批次下載檔數 |
| `TEACHING_MATERIAL_MAX_BATCH_DOWNLOAD_BYTES` | 1073741824 (1 GiB) | 批次下載總量 |
| `EXAM_TXT_MAX_FILE_BYTES` | 5242880 (5MB) | 考卷 TXT（`exams/`）專用 |

技術連動：反向代理 body 上限、大檔串流寫入 NAS、上傳進度與逾時（proxy timeout 由部署層調整）。

### 5.4a 教材傳輸安全與 NAS 驗證

> 對應建議事項 PLAN §5.1；儲存層 **interactive** 模式見 NAS PLAN §5.3。

**上傳流程（修訂）**：

```
JWT 已登入 → 選檔／衝突處理完成
  → NAS 登入視窗（NAS 本地帳號 或 AD 網域帳號）
  → 後端驗證 SMB 並開始上傳
  → 傳輸視窗：進度 %、可取消
  → 寫 DB + NAS → Audit → Close Session → 成功提示
```

**下載流程（單檔／批次 ZIP）**：

```
選擇下載 → NAS 登入 → 傳輸視窗 → 完成 → Audit → Close Session
```

批次 ZIP：**一次** NAS 登入涵蓋整次 ZIP 產生與下載。

**傳輸視窗規格**：

| 項目 | 規格 |
|------|------|
| 進度 | 顯示傳輸 **%**（`onUploadProgress`／下載串流） |
| 取消 | 「取消傳輸」；`AbortController` |
| 關閉 | 僅右上角 **X** 或「取消傳輸」；禁用點遮罩關閉 |
| 防重整 | `beforeunload` 警告 |
| 完成 | 顯示成功後關閉視窗 |

**NAS 憑證傳遞**（技術設計二擇一）：

- **A**：每次 API 帶 `nas_username` + `nas_password`（HTTPS）
- **B**：先 `POST /nas-session/verify` 得短時 `nas_session_token`（建議 5～15 分鐘），同次傳輸複用

**Audit**：每次傳輸寫入 `file_transfer_audit_logs`（含 `nas_username`、`emp_id`、`client_ip`）；詳見建議事項 PLAN §7.1。

### 5.5 分批、分日、分檔上傳

- 教材與訓練計畫為 **1:N 累加**；不要求一次上傳完成。
- 每次上傳（單檔或多檔）各產生獨立記錄（衝突處理除外，見 5.6）。
- 限額只約束**單次請求**，不限制計畫歷史總檔數或總容量。
- 批次上傳採**部分成功**：成功項已寫 DB＋NAS；失敗項回傳原因，可單獨重傳。
- **僅「進行中」計畫可上傳**；已封存唯讀（可列表／下載使用中教材）。

### 5.6 同名衝突處理

**觸發條件**：同 `plan_id` 已有 `is_active=true` 且 `original_filename` 相同。

上傳前呼叫衝突預檢；若有衝突，跳窗**必選其一**：

#### 選項 A：停用舊教材 ＋ 上傳新版

| 項目 | 行為 |
|------|------|
| 舊筆錄 | `is_active=false`，填 `deactivated_at/by`，`replaced_by_id=新版.id` |
| 舊 NAS 檔 | **不更名、不移動、不刪除** |
| 新版 | 新 `id`、新路徑 `{new_id}.{ext}` |
| 對外 | 列表／搜尋／下載僅見新版 |

#### 選項 B：新檔取代舊檔

| 項目 | 行為 |
|------|------|
| DB | **沿用同一筆** `id` |
| NAS | **覆寫**原 `storage_path` |
| 中繼資料 | 更新 `original_filename`、`file_size_bytes`、`uploaded_at/by` 等 |
| 舊內容 | 實體覆寫，**無舊版可下載** |

上傳參數：

```
on_conflict: "deactivate_and_new" | "replace_in_place"
conflict_target_id: <既有使用中教材 id>   # 有衝突時必填
```

### 5.7 下載與檔名

| 場景 | 檔名規則 |
|------|----------|
| 單檔下載 | `Content-Disposition` 使用 `original_filename`（中文用 `filename*` RFC 5987） |
| 批次 ZIP | 每 entry 為 `original_filename` |
| ZIP 內重名 | 自動改為 `{id}_{original_filename}`；確認窗標示 |
| 可下載範圍 | **僅 `is_active=true`**；停用 id 拒絕下載 |

> **[後續再議]**：已停用教材的歷史版本下載／版本瀏覽；本期不提供。

### 5.8 教材庫（搜尋與批次下載）

**列表篩選**：`keyword`（title、description、tags、original_filename）、`material_type_id`、`sub_category_id`、`year`、`plan_id`、`file_format`、`uploaded_at` 區間；預設 `is_active=true`。

**跨頁勾選**：

- 以 `Set<materialId>` 保存，換頁、排序、篩選不丟失已勾選項。
- **預覽不改變勾選狀態**。
- 工具列顯示「已選 N 項」。

**批次下載流程**：

1. 點「批次下載」。
2. 確認窗：標題、原檔名、類型、所屬計畫、檔案大小。
3. 確認後 `POST .../batch-download`，回傳 ZIP。
4. 解壓後檔名為原檔名（重名依 5.7）。

### 5.9 API 端點摘要

Router 前綴建議：`/admin/teaching-materials`（權限比照 `menu:exam` 或技術設計定案）。

| 端點 | 方法 | 說明 |
|------|------|------|
| `/upload` | POST | 上傳（含 `on_conflict`、`conflict_target_id`；須 NAS 憑證或 `nas_session_token`） |
| `/conflict-check` | GET | 同名衝突預檢 |
| `/` | GET | 教材庫總覽（跨計畫、分頁篩選） |
| `/by-plan/{plan_id}` | GET | 單一計畫使用中教材列表 |
| `/{id}/download` | GET | 單檔下載（須 NAS 憑證） |
| `/batch-download` | POST | 批次 ZIP（須 NAS 憑證） |
| `/nas-session/verify` | POST | （可選）驗證 NAS 帳密，回傳短時 `nas_session_token` |
| `/{id}` | PUT | 更新中繼資料（不換檔） |
| `/{id}` | DELETE | 軟刪除（`is_active=false`） |
| `/material-types` | GET/POST/PUT/DELETE | 類型主檔維護（已引用禁止改 `slug`） |
| `/material-file-formats` | GET/POST/PUT/DELETE | 允許格式主檔維護（見主檔維護 PLAN） |

查詢模式可參考 `backend/app/routers/question_bank.py`（分頁、`keyword`、`tags` 篩選）。

### 5.10 前端畫面

1. **訓練計畫編輯頁**：教材上傳區、已連結列表、衝突對話框；**NasLoginModal**、**FileTransferModal**（進度、取消）；允許格式由 API 動態載入。

## 7. 資料結構與 API 欄位

> 本節定義 API／表單欄位契約，供開發、測試、驗收對照；非紙本表單。

### 7.1 `TeachingMaterialUpload`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `file` / `files[]` | file | 是 | 白名單格式 |
| `plan_id` | int | 是 | 訓練計畫 |
| `title` | string | 否 | 未填則由檔名推導 |
| `material_type_id` | int | 是 | 教材類型 |
| `description` | string | 否 | |
| `tags` | string[] | 否 | |
| `on_conflict` | string | 條件 | 有衝突時必填：`deactivate_and_new` / `replace_in_place` |
| `conflict_target_id` | int | 條件 | 有衝突時必填 |
| `nas_username` | string | 條件 | 未使用 `nas_session_token` 時必填 |
| `nas_password` | string | 條件 | 未使用 `nas_session_token` 時必填 |
| `nas_session_token` | string | 條件 | 若已呼叫 `/nas-session/verify` 則帶此 token |

### 7.1b `NasSessionVerifyRequest`（可選）

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `nas_username` | string | 是 | NAS 本地或 `DOMAIN\user` |
| `nas_password` | string | 是 | 不落地儲存 |

### 7.2 `TeachingMaterial`（回應／列表）

含 5.2.2 欄位；列表 API 預設不含 `is_active=false`。

### 7.3 `MaterialType`

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | int | 主鍵 |
| `name` | string | 類型名稱 |
| `slug` | string | 目錄識別 |
| `sort_order` | int | 排序 |
| `max_file_bytes` | int nullable | 單檔上限 |
| `is_active` | boolean | 是否啟用 |

### 7.4 `BatchDownloadRequest`

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `ids` | int[] | 是 | 僅允許使用中且通過權限檢查者 |
| `nas_username` | string | 條件 | 未使用 `nas_session_token` 時必填 |

## 8. 風險與待確認

| 項目 | 說明 |
|------|------|
| NAS 儲存層未就緒 | 教材上傳依賴 NAS PLAN 儲存抽象層；可先用本機 stub 開發 API |
| 大檔上傳逾時 | 50MB 在慢網路需調整代理／前端逾時與進度顯示 |
| NAS 憑證安全 | interactive 模式須 HTTPS；密碼不寫 DB；見建議事項 PLAN §8 |
| 使用者無 NAS 權限 | 須有明確錯誤訊息 |
| 同名 ZIP entry | 批次下載含多計畫同名檔時以 `{id}_` 前綴處理 |
| 權限範圍 | 教材庫跨計畫查詢須套用 `resolve_data_scope`（技術設計定案） |
| 舊版下載 | 本期不做；停用資料僅供備份／稽核 |
