# 超管考試中心試考模式 — 實作計劃 (PLAN)

**文件類型**：棕地實作計劃  
**建立日期**：2026-07-23  
**狀態**：📋 **待審**（草案；待 Wilson 對齊 Q1～Q8 後再開 TASKS／實作）  
**前置規格**：[已完成/20260612_AD整合_系統管理者登入_PLAN.md](已完成/20260612_AD整合_系統管理者登入_PLAN.md) §5.3.2（`is_trainee` 訓練流程隔離）  
**衝突說明**：本 PLAN **放寬** AD W4「管理帳號完全不參與考試中心」之限制，改為「可瀏覽＋可選試考，但不污染訓練／成績／報到資料」。

> **背景**：目前 `GET /api/exam/my_exams` 對 `is_trainee=false`（break-glass `admin`、AD JIT 系統管理等）直接 `return []`，考試中心顯示「目前沒有考試任務」。產品期望：超管仍須能**檢視**所有訓練計畫；**不必**應考；若自願試考，交卷後**僅當場顯示分數**，**不寫入**成績中心；**免報到**。

---

## 1. 目的

1. **可視性**：超管／管理帳號進入考試中心時，可看到**全部未封存**訓練計畫（不受受課對象、本人部門限制），便於現場支援與驗收。
2. **非受訓者**：超管**不強制**應考；列表以「瀏覽／試考」為主，非「待辦任務」語意。
3. **試考可選**：若超管點「試考」，流程與學員類似（讀題、作答、交卷、看分數），但屬**試考模式**。
4. **資料隔離**：試考**不得**寫入 `exam_records`／`exam_history`／`exam_detail`；**不得**出現在成績中心（`report.py`）、個人歷史、部門統計、授權重考標的。
5. **免報到**：試考模式**跳過**報到檢查；**不得**寫入 `attendance_records`（避免虛增實到人數）。

## 4. 名詞解釋

| 名詞 | 定義 |
|------|------|
| **超管／管理帳號** | 本 PLAN 預設：`is_trainee=false`（`auth_source=local` 之 break-glass，或 AD JIT 管理員） |
| **試考模式 (practice mode)** | 可讀題、作答、看分；**不落庫**、免報到、不進成績中心 |
| **應考模式 (trainee mode)** | 現行學員流程：受課對象篩選、須報到、交卷寫入 DB |
| **成績中心** | `backend/app/routers/report.py` 及前端 `ReportDashboard`／`PersonalScoreHistory` 等依 `ExamRecord` 之報表 |
| **開考鐵律** | 一般學員：`start_exam`／`submit_exam` 前須有 `attendance_records`；試考模式**例外** |

## 5. 作業內容

### 5.1 現況（As-Is）

```344:346:backend/app/routers/exam_center.py
    # 管理帳號（is_trainee=False）不參與訓練考試流程
    if not current_user.is_trainee:
        return []
```

| 端點 | 現況 |
|------|------|
| `GET /exam/my_exams` | 管理帳號 → `[]` |
| `GET /exam/start/{plan_id}` | 未擋 `is_trainee`；但若無列表入口，實務上進不去 |
| `POST /exam/submit/{plan_id}` | 寫入 `ExamRecord`＋`ExamHistory`＋`ExamDetail`；須報到 |
| `report.py` | 查 `ExamRecord` **未**過濾 `is_trainee` → 若管理員曾交卷會進成績中心 |

### 5.2 目標行為（To-Be）

```text
超管登入 → 考試中心
  → my_exams：全部未封存計畫（is_practice_mode=true）
  → 卡片：無報到按鈕；「試考（不計成績）」
  → start_exam：回傳題目；不建 ExamRecord；不查報到
  → submit_exam：評分 → 回傳分數與詳情；不 commit DB
  → 前端顯示成績 → 返回列表
  → 成績中心／報到統計：無任何新增紀錄
```

### 5.3 後端設計

#### 5.3.1 試考用戶判斷

建議新增於 `backend/app/access_scope.py`（或 `exam_center.py` 區域 helper）：

```python
def is_exam_practice_user(user: models.User) -> bool:
    """管理帳號：考試中心試考模式，不污染訓練資料。"""
    return user is not None and user.is_trainee is False
```

> 若 Q1 定案為「僅 SUPER_ADMIN 角色」，改為 `is_super_admin_role(user.role.name)`。

#### 5.3.2 Schema 擴充

`ExamListItem` 新增：

| 欄位 | 型別 | 說明 |
|------|------|------|
| `is_practice_mode` | `bool` | `true` 表示試考列表項 |

試考模式下：

- `can_start_exam`：依計畫日期計算（允許試考進行中計畫）
- `status`／`score`／`attempts`：無 DB 紀錄時固定為未交卷狀態（`active`／`pending` 等，**不**顯示「已完成」除非未來另有需求）

`ExamResultResponse` 可選增 `is_practice_mode: true`（交卷回應供前端顯示提示）。

#### 5.3.3 `get_my_exams` 邏輯

```text
if is_exam_practice_user(current_user):
    plans = 全部未封存 TrainingPlan（order by training_date desc）
    record_by_plan_id = {}   # 不查該員 ExamRecord（或查了也不使用）
    每筆 ExamListItem：is_practice_mode=True, can_start_exam=compute_can_start_exam(...)
else:
    維持現行受課對象篩選邏輯
```

#### 5.3.4 `start_exam`

```text
if is_exam_practice_user:
    跳過 _require_attendance_record
    不建立／更新 ExamRecord
    僅回傳 ExamStartResponse（題目、限時）
else:
    維持現行邏輯
```

#### 5.3.5 `submit_exam`

```text
if is_exam_practice_user:
    執行評分迴圈（與現行相同）
    return ExamResultResponse(..., is_practice_mode=True)
    # 不 add ExamRecord / ExamHistory / ExamDetail
    # 不 db.commit() 考試資料
else:
    維持現行邏輯（含報到、落庫、授權重考消耗）
```

#### 5.3.6 報到相關 API（可選，依 Q6）

| 端點 | 試考建議 |
|------|----------|
| `GET /exam/plan/{id}/attendance/status` | 回 `{ is_checked_in: true }` 或前端不呼叫 |
| `POST /exam/plan/{id}/attendance/checkin` | 403「試考模式不需報到」或前端不顯示按鈕 |

### 5.4 前端設計

| 元件 | 變更 |
|------|------|
| `ExamDashboard.tsx` | `is_practice_mode` 時隱藏 `CheckInButton`、報到彈窗；按鈕改「試考（不計成績）」；空狀態文案區分 |
| `ExamRunner.tsx` | 交卷成功且 `is_practice_mode`：成績頁加提示「試考結束，成績未寫入系統」；返回 `/` |
| `types.ts` / `User` | 依 Q7 增 `is_trainee` 或列表項旗標 |
| `CheckInPage.tsx` | 不變（試考不走報到頁） |

### 5.5 與其他模組邊界

| 模組 | 關係 |
|------|------|
| **成績中心** (`report.py`) | 試考不落庫 → 無需改 report 查詢（防禦性可選：`JOIN User WHERE is_trainee=true` 另開） |
| **報到統計** (`training.py`) | 試考不寫 `attendance_records` → 不虛增實到 |
| **受課對象／應到** | 仍只計 `is_trainee=true`；不變 |
| **Owner 寫入權** | 與試考無關；訓練計畫編輯仍依 `can_modify_owned_resource` |
| **角色部門權限（可視範圍）** | 試考列表**不**依可視範圍；與成績中心「看資料」分離 |
