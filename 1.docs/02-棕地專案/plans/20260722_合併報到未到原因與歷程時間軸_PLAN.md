## 1. 目的

1. **課後未到原因不必靠記憶**：課程結束後填未到原因時，承辦人從「合併報到紀錄」進入；系統依 `batch → plans` 自動知道要套用到哪幾場，**不要求記住計畫名稱**。
2. **合併 N 場、未到原因填一次**：同一人因故未到，在批次入口填一次原因，自動套用到該 batch 內所有「應到且未到」的計畫；單一計畫可覆寫例外。
3. **報到歷程＝時間軸**：主列表「報到時間」顯示**最後一次狀態時間**；展開歷程依**新→舊（3→2→1）**顯示；含公假／病假變更與最終準時報到。
4. **資料可稽核**：未到原因變更改為 append-only 寫入歷程事件，不再只覆蓋現況一筆而失去歷史。
5. **合併現場可看真實名單**：合併報到 Modal 左側為合併 QR；右側以總覽同款列表顯示本 batch 各計畫摘要；點「報到統計」鑽入該計畫人員明細（已到／未到），**不必離開合併現場去總覽另開**。
6. **列表可讀、搜尋可驅動批次操作**：報到總覽上下半與相關 Modal 表格，筆數 **>5** 顯示分頁；合併未到名單可搜尋；「一鍵填寫多人請假原因」名單＝**目前卡片條件 ∩ 搜尋結果**（非僅卡片全量）。

## 4. 名詞解釋

| 名詞 | 定義 |
|------|------|
| **合併報到批次 (batch)** | 一次 QR／一個標籤對應多個 `TrainingPlan`；關閉後 QR 不可報到，但紀錄仍可查 |
| **合併報到紀錄** | 報到總覽上可搜尋、可重開的 batch 列表（解決「Modal 關了就找不到」） |
| **合併報到 Modal** | 現場／紀錄開啟的批次詳情：左 QR、右各場摘要列表；可關閉／重開、填未到原因 |
| **單計畫報到統計 Modal** | 既有「報到統計 - {計畫名}」：人員已到／未到、未到原因、列印；可內嵌單一計畫 QR（本波從合併入口開時不顯示 QR 鈕） |
| **批次層未到原因** | 以 `batch_id + emp_id` 表達「這次合併場次此人未到的原因」 |
| **計畫層未到原因** | 既有 `attendance_absence_reasons`（`plan_id + emp_id`）；可作現況與 override |
| **歷程事件 (event)** | append-only；含報到成功／已報到過／未到原因變更等 |
| **時間軸** | 依 `event_time` 新→舊列出該員在該 plan（或 batch 視角）之狀態變更 |

## 5. 作業內容

### 5.1 使用者流程

#### 5.1.1 課後填未到原因

```text
報到總覽
  →「合併報到紀錄」
  → 依場次日／標籤找到「0721 上午場」
  → 開啟批次詳情（顯示綁定的 A、B 計畫名稱）
  → 未到人員清單 → 選原因（病假／公假…）填一次
  → 系統套用到 A、B 中應到且未到者
  → 各計畫報到統計與歷程皆可見
```

**失敗情境（本 PLAN 要消除）**：關掉合併 Modal → 忘記是哪兩場 → 在單一計畫亂填或漏填。

#### 5.1.2 合併現場查看真實報到狀況（Q10）

```text
報到總覽 →「多場訓練合併報到」→ 建立／開啟合併 Modal
  → 左側：合併 QR（掃碼報到用）
  → 右側：僅本 batch 計畫之摘要表（樣式同報到總覽主表）
       訓練計畫｜日期｜應到｜實到｜未到｜出席率｜操作[報到統計]
  → 點某一計畫「報到統計」
  → 疊開單計畫報到統計 Modal（Title 無「顯示 QRCode」）
  → 可見誰已報到、誰未到；可「更新」刷新該計畫
  → 關閉子 Modal → 回到合併 Modal（QR 與右側摘要仍在）
  → 合併 Modal「更新」刷新右側各場摘要數字
```

**現況缺口**：右側僅有摘要數字、無「報到統計」鈕，無法在合併現場看人員名單。

### 5.2 資料設計（提案）

#### G2 方案 B（推薦）

| 表／欄位 | 用途 |
|----------|------|
| **新表** `attendance_batch_absence_reasons` | `batch_id`, `emp_id`, `reason_code`, `reason_text`, `recorded_by`, `recorded_at`（現況／預設） |
| **既有** `attendance_absence_reasons` | 套用時同步 upsert；plan 個別修改＝override |
| **擴充** `attendance_checkin_events` | 新增可選 `reason_code` / `reason_text` / `operator_emp_id`；`event_type` 含 `absence_reason_updated` / `absence_reason_cleared` |

#### 寫入規則（套用）

對 `batch` 內每個 `plan_id`：

1. 員工為該 plan 應到對象  
2. 無 `attendance_records`（未報到）  
3. → upsert plan-level reason  
4. → append event（同一操作時間可相同；**實作時採「每 plan 一筆 event」**以利單一計畫歷程一致）

#### G1b 資料／API 依賴

- **不需新 API**：右側摘要沿用既有 `GET /training/attendance/batches/{id}/stats`；鑽入明細沿用既有 `GET /training/plans/{plan_id}/attendance/stats`。
- 若 stats 尚無 `training_date`，前端可用 `batchModal.plans`／總覽已載入之 plan 對照補日期欄。

### 5.3 API（草案）

| Method | Path | 說明 |
|--------|------|------|
| GET | `/training/attendance/batches` | 列表：`training_date`、`keyword`（標籤）、`status`；回傳 plans 摘要、未到人數 |
| GET | `/training/attendance/batches/{id}` | 既有；詳情含 plans |
| GET | `/training/attendance/batches/{id}/stats` | **既有**；G1b 右側摘要來源 |
| GET | `/training/plans/{plan_id}/attendance/batches` | 該計畫曾參與的 batch（給單一 Modal 提示） |
| PUT | `/training/attendance/batches/{id}/absence-reason` | 單人批次套用 |
| PUT | `/training/attendance/batches/{id}/absence-reason/bulk` | 多人批次套用 |
| GET | `/training/plans/{plan_id}/attendance/events` | **改降冪**；回傳中文友善欄位／`batch_label`／`reason_*` |
| GET | `/training/plans/{plan_id}/attendance/stats` | `checked_in_users`／未到列補 `latest_event_time`（及可選 label）；**G1b 鑽入明細亦用此** |

權限：沿用 `menu:attendance-overview`／`menu:plan`；未到原因編輯規則沿用現行 `_can_edit_absence_reason`。

### 5.4 前端（草案）

| 畫面 | 變更 |
|------|------|
| 報到總覽 | 新增「合併報到紀錄」區塊（日期＋標籤篩選、狀態預設進行中、計畫名稱、開啟詳情）；上半訓練清單顯示開始～結束日；**上下半筆數 >5 分頁** |
| 合併報到 Modal（左 QR／右列表） | 右側改為**對齊總覽主表**之欄位與樣式；僅列 batch 內計畫；每列「報到統計」鈕；各場／未到 **>5 分頁**；未到可搜尋 |
| 單計畫報到統計 Modal（從合併開） | 複用既有 Modal；加旗標（如 `hideQrToggle`）：**不渲染「顯示 QRCode」**；`z-index` 高於合併 Modal；關閉不影響合併 Modal |
| 批次詳情（課後） | 除 QR／統計外，增加「未到人員／填寫原因」；已關閉仍可填原因（Q4） |
| 單計畫報到統計（一般入口） | 頂部提示所屬 batch；歷程新→舊；主列最後事件時間；從總覽開時仍可顯示 QR 鈕；人員名單 **>5 分頁** |
| 一鍵填寫多人請假原因 | 「批次填寫請假原因」Modal 的可勾選名單＝**目前卡片條件 ∩ 搜尋結果**（報到統計與合併未到皆同） |
| 歷程文案 | 例：`準時報到`、`病假`、`公假`；合併來源可作次要括號 |

**G1b 實作備註**：

- 現況列表與 Modal 皆在 `AttendanceOverviewPage.tsx` 內聯，**無獨立共用套件**；本波可：(a) 同檔複用 `openAttendanceModal` + `openedFromBatch` 旗標，或 (b) 抽小型表格列元件（選用，非必須）。
- 子 Modal 開啟時合併 Modal 可保持掛載（遮罩疊層）；避免關閉合併導致 QR／狀態丟失。
