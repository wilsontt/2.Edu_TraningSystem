# 系統啟動與連線檢修指南 (Q&A)

本文件總結了開發過程中常見的啟動錯誤、連線問題及其解決方案，供開發者快速排查。

---

## Q1: 啟動後端時出現 `[Errno 48] Address already in use`

*   **問題描述**：嘗試啟動 FastAPI 服務時報錯，提示 8000 埠號已被佔用。
*   **原因方案**：上一個後端進程未正確終止，或有其他服務佔用了該埠號。
*   **解決步驟**：
    1.  檢查佔用 8000 埠號的進程 PID：
        ```bash
        lsof -i :8000
        ```
    2.  強制終止該進程（假設 PID 為 12345）：
        ```bash
        kill -9 12345
        ```
    3.  重新啟動後端服務。

---

## Q2: 後端報錯 `ImportError: cannot import name 'xxx' from 'app.routers'`

*   **問題描述**：啟動時出現模組匯入錯誤，通常與自定義工具（如 `auth_utils`）相關。
*   **原因方案**：
    1.  **路徑層級錯誤**：在 `app/routers` 目錄下的檔案中使用 `from . import xxx` 會查找同目錄，應改用 `from .. import xxx` 查找父目錄。
    2.  **PYTHONPATH 未設置**：執行指令時未將當前目錄目錄加入 Python 路徑。
*   **正確啟動指令**：
    ```bash
    export PYTHONPATH=$PYTHONPATH:. && .venv/bin/python3 -m uvicorn app.main:app --reload
    ```

---

## Q3: 前端提示「無法取得驗證碼，請檢查伺服器狀態」

*   **問題描述**：登入頁面加載後，驗證碼區塊顯示轉圈或報錯，無法顯示圖片。
*   **檢查清單**：
    1.  **後端是否啟動**：確認 `http://localhost:8000` 是否可存取。
    2.  **API 測試**：使用瀏覽器或 cURL 訪問 `http://localhost:8000/api/auth/captcha`，確認是否返回 JSON 資料。
    3.  **CORS 限制**：確認 `backend/app/main.py` 中已正確配置 `CORSMiddleware`。

---

## Q4: 前端樣式（Tailwind CSS）未正確套用，畫面佈局混亂

*   **問題描述**：頁面顯示純文字或基本 HTML 樣式，缺少 Tailwind 的排版效果。
*   **原因方案**：Tailwind CSS 4 需配合 Vite 插件編譯。
*   **修正步驟**：
    1.  確認 `frontend/vite.config.ts` 已引入 `@tailwindcss/vite`。
    2.  確認 `frontend/src/index.css` 第一行包含 `@import "tailwindcss";`。
    3.  重啟 Vite 開發伺服器。

---

## Q5: 如何恢復意外丟失的登入狀態？

*   **問題描述**：重新整理頁面後需要重新登入。
*   **原因方案**：JWT Token 存儲或驗證邏輯問題。
*   **解決機制**：
    1.  系統會將 Token 存於 `localStorage.getItem('token')`。
    2.  `App.tsx` 在掛載時會調用 `GET /api/auth/me`。
    3.  若 Token 效期內且伺服器正常，系統會自動恢復 `user` 狀態。

---

## Q6: 導覽列只顯示首頁，其他選單不見了？

*   **問題描述**：登入完成後，導覽列只看到「首頁」，缺乏「訓練計畫」、「系統管理」等項目。
*   **原因方案**：
    1.  **角色權限限制**：系統實施 RBAC，新註冊的帳號預設角色為 `User`，僅能看到「首頁」與「成績中心」。
    2.  **管理員權限**：若需看到完整功能，該帳號必須具備 `Admin` 角色。
*   **如何提升為 Admin (方法 A：使用 VS Code SQLite 擴充功能)**：
    1.  按下 `Cmd+Shift+P` (Mac) 或 `Ctrl+Shift+P` (Win)，輸入 `SQLite: Open Database`。
    2.  **關鍵點**：請務必選擇專案目錄下的 `data/education_training.db` 檔案。
    3.  開啟後在左側 Explorer 的 SQLITE EXPLORER 視窗中應可看到 `users` 與 `roles` 等資料表。
    4.  右鍵點擊資料庫選擇 `New Query`，執行以下 SQL (請替換為您的工號)：
        ```sql
        UPDATE users SET role_id = (SELECT id FROM roles WHERE name = 'Admin') WHERE emp_id = '您的工號';
        ```

*   **如何提升為 Admin (方法 B：終端機一鍵指令)**：
    在專案根目錄終端機直接貼上此指令 (請替換 `您的工號`)：
    ```bash
    sqlite3 data/education_training.db "UPDATE users SET role_id = (SELECT id FROM roles WHERE name = 'Admin') WHERE emp_id = '您的工號';"
    ```

> [!IMPORTANT]
> 若出現 `no such table: roles`，代表您連線到了錯誤的 `.db` 檔 (通常是自動產生的空檔案)。請確認路徑絕對是 `data/education_training.db`。

---

> [!TIP]
> 建議在開發時保持兩個終端機視窗，分別專門運行前端 (`npm run dev`) 與後端 (`uvicorn`)，以便隨時觀察 Log 輸出。
