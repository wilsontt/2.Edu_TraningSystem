# 單位管理錯誤回饋優化與防禦性修正報告

## 1. 發現的問題
- **初步問題**：使用者回報刪除單位時，錯誤訊息一閃即逝，無法閱讀。
- **根本原因**：前端按鈕預設為 `submit` 類型，觸發了瀏覽器的表單提交行為，導致頁面在顯示錯誤後立即刷新。

## 2. 修正內容
### 介面優化
- **替換 Alert**：將原生的 `window.alert` 替換為自定義的 React Modal 元件，提供更明確的視覺回饋。
- **持久化顯示**：Modal 設計為必須由使用者手動點擊「關閉」才會消失。

### 防禦性程式碼修正
- **強制按鈕類型**：為所有相關的 `<button>` 標籤顯式添加 `type="button"` 屬性，防止預設提交。
- **阻止事件冒泡**：在點擊處理函式中加入 `e.preventDefault()`，確保不會觸發非預期的瀏覽器行為。

## 3. 驗證結果
- **測試情境**：嘗試刪除含有使用者的單位 (`NonEmptyDept`)。
- **測試過程**：
    1. 點擊刪除 -> 確認對話框 -> 彈出錯誤 Modal。
    2. 等待 3 秒 -> Modal **持續顯示，未消失**。
    3. 點擊關閉 -> Modal 正常關閉。
- **結論**：問題已完全解決，錯誤回饋機制現在穩定且可靠。
