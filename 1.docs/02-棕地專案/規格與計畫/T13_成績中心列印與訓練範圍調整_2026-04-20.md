# 成績中心列印與訓練範圍調整（T13 #3 / 2026-04-20）

## 範圍對照（來源文件）

需求摘錄自 [T13 增修功能實作PLAN_測試問題.md](T13%20增修功能實作PLAN_測試問題.md) 約 174–192 行：

|------|------|
| 個人成績歷史 | 「選擇計畫」區塊增加**全選／不全選（清除）** |

## 1. 個人成績歷史：選擇計畫全選／清除

**檔案**：[`PersonalScoreHistory.tsx`](frontend/src/components/personal/PersonalScoreHistory.tsx)

- 在「選擇計畫」標題列加兩個按鈕（或連結）：**全選**、**清除**（或「不全選」）。
- 行為：`selectedPlanIds` 設為所有 `plansData` 的 `plan_id`，或 `new Set()`。
- 不需抽成跨專案共用元件（文件亦表示可不必）；若日後重複再抽即可。

---

## 2. `ScorePrintFlow`：支援「歷程 Modal 專用」版面

**檔案**：[`ScorePrintFlow.tsx`](frontend/src/components/common/ScorePrintFlow.tsx)

- 新增可選 props（命名示例，實作時可微調）：
  - `variant?: 'full' | 'planHistoryFooter'`（預設 `full`）
  - 或布林：`showPlanQuestion?: boolean`、`showSignatureHistoryRow?: boolean`、`showPreviewPdfButtons?: boolean`
- **`planHistoryFooter`（或等價組合）**：
  - 保留：**詢問1**（訓練計畫；在 [`PlanHistoryModal`](frontend/src/components/personal/PlanHistoryModal.tsx) 情境下可後續與第 198 行「disable 詢問1」對齊—本次文件 188 行先要求拿掉簽名/歷程與兩按鈕，**詢問1 是否隱藏**若與 198 行衝突，以**最新**文件為準：若需同步 disable 計畫勾選，則在 Modal 傳入 `planOptions` 單一計畫且 `readOnlySelectedPlan`）
  - **詢問2** 文案：無「個人」前綴的選項改為文件用語：**列印個人成績清單**、**列印個人考卷成績**（僅在 `variant=planHistoryFooter` 或 `context=personalHistory` 時替換文案，避免後台報表用語被改錯）。
  - **隱藏**：`列印員工簽名`、`列印考試歷程` 兩個 checkbox。
  - **隱藏**：`載入預覽`、`列印 PDF（已選 N 人）` 兩按鈕。
- **列印實際行為**：文件寫「不需要載入預覽／按列印後再預覽」—若底部按鈕移除，需補**單一主按鈕**（例如「產生 PDF」或「列印」）改呼叫既有 `exportModalPrintPdf`／或新開瀏覽器預覽；此細節在實作時與現有 [`PlanHistoryModal`](frontend/src/components/personal/PlanHistoryModal.tsx) 的 `exportModalPrintPdf` 對齊，避免無法列印。

**檔案**：[`PlanHistoryModal.tsx`](frontend/src/components/personal/PlanHistoryModal.tsx)

- 移除或取代下方 `printPreview` 表格區塊（若已「不需要載入預覽」）：改為單一路徑直接下載／開啟 PDF，或僅保留關閉；與第 2 節按鈕一致。

## 4. `ScoreDetailModal`：歷程進入時的「預覽成績單」互動

**檔案**：[`ScoreDetailModal.tsx`](frontend/src/components/personal/ScoreDetailModal.tsx)

- 新增 props（示例）：`printGate?: 'none' | 'requireSignatureCheckbox'`，或由 [`PlanHistoryModal`](frontend/src/components/personal/PlanHistoryModal.tsx) 傳入 `historyId` 時自動啟用 `requireSignatureCheckbox`。
- UI：
  - Header 右側：**Checkbox「列印員工簽名」**（預設 `false`）在 **「預覽成績單」左側**。
  - **預覽按鈕**：預設 `disabled` + 灰階樣式；**僅当 checkbox 為 `true`** 時 `disabled` 解除並恢復藍色。
- `ScoreCardPreview`／列印參數：將 `include_employee_signature` 與 checkbox 綁定（預覽內容需反映勾選）。
- [`PlanHistoryModal`](frontend/src/components/personal/PlanHistoryModal.tsx) 呼叫 `<ScoreDetailModal historyId={...} />` 時傳入上述模式。

**注意**：若產品意圖是「未勾簽名也要能預覽」，與文件「勾選才可按」矛盾；屆時需你一句話確認再改規則。

## 5. 成績總覽與部門統計：訓練計畫狀態篩選（同訓練計畫管理）

**參考實作**：[`TrainingPlanManager.tsx`](frontend/src/components/admin/TrainingPlanManager.tsx) 使用 `activeTab: 'active' | 'expired' | 'archived'`，並以 `GET /training/plans?status=active|expired|archived` 取得對應清單。

**需求修正**（對應測試文件「續 1，加上可篩選」）：

- 不再只做「僅進行中、其餘永遠不顯示」，而是：**使用者可切換**與訓練計畫管理相同的三態，**預設**建議與訓練計畫管理一致（**進行中**）。
- **UI**：在**個人成績總覽**與**部門成績／統計報表**（[`ReportDashboard.tsx`](frontend/src/components/admin/ReportDashboard.tsx) 等）區塊上方或側邊，加入與訓練計畫管理相同體驗的分頁／切換（文案：進行中／已過期／已封存，或與現有後台用語一致）。

**後端**：

- [`report.py`](backend/app/routers/report.py)：依 `TrainingPlan` 狀態過濾（與 [`training.py`](backend/app/routers/training.py) 既有 `status` 語意一致，或共用輔助函式），讓 `overview`、`/plan`、`/department` 等「依計畫維度」的統計支援 `plan_status` 或 `training_plan_status` query（命名於實作時統一）。
- [`exam_center.py`](backend/app/routers/exam_center.py) `personal/overview`（若總覽含計畫列表或依計畫聚合）：同樣支援篩選參數。

**前端**：

- [`ReportDashboard.tsx`](frontend/src/components/admin/ReportDashboard.tsx) 對應的 fetch 帶 `status`／新參數；篩選狀態變更時重新拉取。
- [`PersonalScoreOverview.tsx`](frontend/src/components/personal/PersonalScoreOverview.tsx)：若該頁僅 KPI、無計畫維度，則篩選套用於**有列出訓練計畫或依計畫統計**的區塊；若目前無計畫列表，則與 API 擴充一併補上「可篩選的計畫區塊」或明確僅篩選後端聚合維度（實作時依現有版面定案）。

**驗證**：切換「已過期／已封存」時，報表與總覽數字／列表與訓練計畫管理同一批計畫定義一致。

- 個人成績歷史：全選／清除後，趨勢圖與列表聯動正確。
