## 二、資料庫結構 (Database Schema)

### 核心實體模型 (Core Entity Models)

#### 1. 組織架構模組 (Organization Module)

**departments（部門/單位）**
- `id` (Integer, PK)：部門 ID
- `name` (String, Unique)：部門名稱
- 關聯：`users` (One-to-Many)

**users（使用者）**
- `emp_id` (String, PK)：員工編號（主鍵）
- `name` (String)：姓名
- `dept_id` (Integer, FK → departments.id)：所屬部門
- `role_id` (Integer, FK → roles.id)：角色
- `status` (String, default="active")：帳號狀態
- 關聯：`department`, `role`, `exam_records`

**roles（角色）**
- `id` (Integer, PK)：角色 ID
- `name` (String, Unique)：角色名稱（如 "Admin", "User"）
- 關聯：`users` (One-to-Many), `functions` (Many-to-Many via role_functions)

**system_functions（系統功能）**
- `id` (Integer, PK)：功能 ID
- `name` (String)：功能名稱
- `code` (String, Unique)：功能代碼（如 "menu:exam", "btn:ai_gen"）
- `parent_id` (Integer, FK → system_functions.id, Nullable)：父節點 ID（支援樹狀結構）
- `path` (String, Nullable)：路由路徑
- 關聯：`children` (Self-referential), `roles` (Many-to-Many)

**role_functions（角色功能關聯表）**
- `role_id` (Integer, FK → roles.id)
- `function_id` (Integer, FK → system_functions.id)
- 用途：Many-to-Many 關聯，定義角色可存取的功能

#### 2. 訓練管理模組 (Training Management Module)

**main_categories（訓練課程大項目）**
- `id` (Integer, PK)
- `name` (String, Unique)：大項目名稱
- 關聯：`sub_categories` (One-to-Many)

**sub_categories（訓練課程細項目）**
- `id` (Integer, PK)
- `main_id` (Integer, FK → main_categories.id)
- `name` (String)：細項目名稱
- 關聯：`main_category`, `training_plans`

**training_plans（訓練計畫）**
- `id` (Integer, PK)
- `sub_category_id` (Integer, FK → sub_categories.id)
- `dept_id` (Integer, FK → departments.id)：開課/上課地點
- `title` (String)：計畫標題
- `training_date` (Date)：開始日期
- `end_date` (Date, Nullable)：結束日期
- `year` (String)：年度（從 training_date 提取）
- `timer_enabled` (Boolean, default=False)：是否啟用計時
- `time_limit` (Integer, default=0)：時間限制（分鐘）
- `passing_score` (Integer, default=60)：及格分數
- 關聯：`sub_category`, `location` (Department), `target_departments` (Many-to-Many), `questions`, `exam_records`

**plan_target_departments（訓練計畫受課單位關聯表）**
- `plan_id` (Integer, FK → training_plans.id)
- `dept_id` (Integer, FK → departments.id)
- 用途：Many-to-Many 關聯，定義哪些部門可參與此訓練計畫

**questions（考卷題目）**
- `id` (Integer, PK)
- `plan_id` (Integer, FK → training_plans.id)
- `content` (Text)：題目內容
- `question_type` (String)：題型（"single", "multiple", "true_false"）
- `options` (Text)：選項（JSON 字串）
- `answer` (String)：正確答案
- `points` (Integer, default=10)：配分
- 關聯：`training_plan`

**question_bank（題庫）**
- `id` (Integer, PK)
- `content` (Text)：題目內容
- `question_type` (String)：題型
- `options` (Text, Nullable)：選項（JSON 字串）
- `answer` (String)：正確答案
- `tags` (Text, Nullable)：標籤（JSON 字串陣列）
- `created_by` (String, Nullable)：建立者員工編號
- `created_at` (DateTime)：建立時間
- 用途：獨立題庫，可重複使用於多個訓練計畫

#### 3. 考試記錄模組 (Exam Records Module)

**exam_records（考試記錄）**
- `id` (Integer, PK)
- `emp_id` (String, FK → users.emp_id)
- `plan_id` (Integer, FK → training_plans.id)
- `total_score` (Integer)：總分
- `is_passed` (Boolean)：是否通過
- `start_time` (DateTime)：開始時間
- `submit_time` (DateTime)：提交時間
- `attempts` (Integer, default=1)：作答次數
- 關聯：`user`, `training_plan`, `details`

**exam_details（考試詳情）**
- `id` (Integer, PK)
- `record_id` (Integer, FK → exam_records.id)
- `question_id` (Integer, FK → questions.id)
- `user_answer` (String)：使用者答案
- `is_correct` (Boolean)：是否正確
- 關聯：`record`

## 三、API 路由結構 (API Routes Structure)

### 1. 認證模組 (`/api/auth`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/auth/captcha` | GET | 生成 4 位數字圖形驗證碼 | 公開 |
| `/auth/register` | POST | 員工編號註冊（需選擇部門） | 公開 |
| `/auth/login` | POST | 登入（員工編號 + 驗證碼） | 公開 |
| `/auth/me` | GET | 取得當前使用者資訊（含權限清單） | 需認證 |
| `/auth/departments` | GET | 公開部門列表（供註冊使用） | 公開 |

**認證機制**：
- 免密碼登入：員工編號 + 4 位數字圖形驗證碼
- JWT Token：8 小時有效期（`ACCESS_TOKEN_EXPIRE_MINUTES = 480`）
- Token 儲存：前端 localStorage
- 驗證碼儲存：後端記憶體字典（生產環境建議改用 Redis）

### 2. 系統管理模組 (`/api/admin`)

#### 單位管理 (`/admin/departments`)
- `GET /admin/departments`：取得所有部門（含使用者數量）
- `POST /admin/departments`：新增部門
- `PUT /admin/departments/{id}`：更新部門
- `DELETE /admin/departments/{id}`：刪除部門（需無使用者）
- `GET /admin/departments/{id}/users`：取得部門使用者清單
- **權限要求**：`menu:admin:dept`

#### 分類管理 (`/admin/categories`)
- `GET /admin/categories/main`：取得所有大項目（含細項目）
- `POST /admin/categories/main`：新增大項目
- `PUT /admin/categories/main/{id}`：更新大項目
- `DELETE /admin/categories/main/{id}`：刪除大項目（需無細項目）
- `GET /admin/categories/sub/{main_id}`：取得特定大項目下的細項目
- `POST /admin/categories/sub`：新增細項目
- `PUT /admin/categories/sub/{id}`：更新細項目
- `DELETE /admin/categories/sub/{id}`：刪除細項目（需無計畫使用）
- **權限要求**：`menu:admin` 或 `menu:plan`

#### 人員管理 (`/admin/users`)
- `GET /admin/users`：取得所有使用者（含部門、角色資訊）
- `PUT /admin/users/{emp_id}`：更新使用者（角色、單位、狀態）
- **權限要求**：`menu:admin:user`
- **保護機制**：Admin 帳號的角色與單位不可變更

#### 角色管理 (`/admin/roles`)
- `GET /admin/roles`：取得所有角色（含使用者數量、功能數量）
- `POST /admin/roles`：新增角色
- `PUT /admin/roles/{role_id}`：更新角色名稱
- `DELETE /admin/roles/{role_id}`：刪除角色（需無成員且無權限）
- **權限要求**：`menu:admin:role`
- **保護機制**：Admin 角色不可更名、不可刪除

#### 權限管理 (`/admin/roles/{role_id}/permissions`)
- `GET /admin/roles/{role_id}/permissions`：取得角色的功能 ID 列表
- `PUT /admin/roles/{role_id}/permissions`：更新角色權限（指派功能）
- **權限要求**：`menu:admin:perm`
- **保護機制**：Admin 角色的權限不可變更

#### 功能清單管理 (`/admin/functions`)
- `GET /admin/functions`：取得所有系統功能（樹狀結構，僅回傳根節點）
- `POST /admin/functions`：新增系統功能
- `PUT /admin/functions/{id}`：更新系統功能
- `DELETE /admin/functions/{id}`：刪除系統功能（需無子節點且未被角色引用）
- **權限要求**：`menu:admin:func`

### 3. 訓練計畫模組 (`/api/training`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/training/plans` | GET | 取得所有訓練計畫 | `menu:plan` |
| `/training/plans` | POST | 建立訓練計畫（含受課單位） | `menu:plan` |
| `/training/plans/{plan_id}` | PUT | 更新訓練計畫 | `menu:plan` |

**訓練計畫建立流程**：
1. 選擇細項目（sub_category_id）
2. 選擇開課單位（dept_id）
3. 設定訓練日期、結束日期
4. 設定計時器與時間限制
5. 設定及格分數
6. 選擇受課單位（target_dept_ids，支援多選）

### 4. 考卷工坊模組 (`/api/admin/exams`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/admin/exams/upload` | POST | 上傳 TXT 檔案並解析題目 | `menu:exam` |
| `/admin/exams/questions/{plan_id}` | GET | 列出計畫的所有題目 | `menu:exam` |
| `/admin/exams/questions/{question_id}` | PUT | 更新題目 | `menu:exam` |
| `/admin/exams/questions/{question_id}` | DELETE | 刪除題目 | `menu:exam` |
| `/admin/exams/materials/{plan_id}` | GET | 列出已上傳的教材檔案 | `menu:exam` |
| `/admin/exams/materials/preview/{year}/{plan_id}/{filename}` | GET | 預覽教材內容 | `menu:exam` |
| `/admin/exams/materials/{plan_id}/{filename}` | DELETE | 刪除教材檔案 | `menu:exam` |

**題目上傳流程**：
1. 上傳 TXT 檔案（格式：`Q:`, `A:`, `B:`, `ANS:`, `SCORE:`）
2. 解析器自動解析題目（`services/parser.py`）
3. 存入 `questions` 表（綁定訓練計畫）
4. 同步存入 `question_bank` 題庫（避免重複）
5. 備份原始檔案至 `data/materials/{year}/{plan_id}/`

**題目解析器支援格式**：
- **單選題**：`Q: 題目`, `A: 選項1`, `B: 選項2`, `ANS: A`, `SCORE: 10`
- **多選題**：答案為 `A,C` 或 `AC`
- **是非題**：答案為 `Y/N`（自動判斷）

### 5. 題庫管理模組 (`/api/admin/question-bank`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/admin/question-bank/` | GET | 查詢題庫（支援分頁、關鍵字、標籤、題型篩選） | `menu:exam_studio` |
| `/admin/question-bank/{id}` | PUT | 更新題目 | `menu:exam_studio` |
| `/admin/question-bank/{id}` | DELETE | 刪除題目 | `menu:exam_studio` |
| `/admin/question-bank/import` | POST | 從題庫匯入題目至訓練計畫 | `menu:exam_studio` |

**題庫功能**：
- 獨立題目庫，可重複使用於多個訓練計畫
- 支援標籤系統（JSON 字串陣列）
- 上傳題目時自動同步至題庫
- 可從題庫匯入題目至指定訓練計畫

### 6. 考試中心模組 (`/api/exam`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/exam/my_exams` | GET | 取得使用者的考試列表（依狀態分類） | 需認證 |
| `/exam/start/{plan_id}` | GET | 開始考試（取得題目，不含答案） | 需認證 |
| `/exam/submit/{plan_id}` | POST | 提交答案並自動評分 | 需認證 |

**考試狀態**：
- `pending`：尚未開始（today < training_date）
- `active`：可進行考試（today >= training_date 且 <= end_date）
- `completed`：已完成（有 ExamRecord）
- `expired`：已過期（today > end_date）

**考試流程**：
1. 使用者選擇訓練計畫
2. 系統驗證考試資格（日期、是否已通過）
3. 取得題目（不含答案）
4. 使用者作答（前端本地儲存進度）
5. 提交答案
6. 後端自動評分並計算通過狀態
7. 儲存考試記錄與詳情

**重考機制**：
- 未通過（分數 < passing_score）可重新作答
- 記錄 `attempts` 次數
- 更新既有 ExamRecord 而非建立新記錄

### 7. 報表模組 (`/api/admin/reports`)

| 端點 | 方法 | 說明 | 權限 |
|------|------|------|------|
| `/admin/reports/overview` | GET | 總體統計（總考試場次、應考人次、平均分數、及格率） | `menu:report` |
| `/admin/reports/department` | GET | 各部門統計（應考人次、平均分數、及格率） | `menu:report` |
| `/admin/reports/plan` | GET | 各訓練計畫統計 | `menu:report` |
| `/admin/reports/export/pdf` | GET | 導出 PDF 成績單 | `menu:report` |

## 四、核心功能實作 (Core Functionality Implementation)

### 1. 認證與授權機制 (Authentication & Authorization)

**免密碼登入流程**：
```
使用者輸入員工編號 → 取得驗證碼圖片 → 輸入驗證碼 → 後端驗證 → 簽發 JWT Token → 前端儲存 Token
```

**RBAC 權限控制**：
- **功能導向權限**：每個功能都有唯一代碼（如 `menu:exam`, `menu:admin:dept`）
- **角色權限指派**：透過 `role_functions` 關聯表定義角色可存取的功能
- **權限檢查**：使用 `check_permission(required_func_code)` dependency
- **Admin 特權**：Admin 角色自動擁有所有權限（bypass 檢查）

**JWT Token 機制**：
- 簽發：`auth_utils.create_access_token(data={"sub": emp_id, "role": role_name})`
- 驗證：`auth_utils.verify_token(token)`
- 有效期：8 小時
- 前端自動附加：Axios interceptor 自動在請求標頭加入 `Authorization: Bearer {token}`

### 2. 題目解析器 (`services/parser.py`)

**TXTParser 類別**：
- `parse_content(content: str)`：解析文字內容
- `parse_file(file_path: str)`：解析檔案

**支援格式**：
1. **單選題**：
   ```
   Q: 這是單選題目內容？
   A: 選項 1
   B: 選項 2
   ANS: A
   SCORE: 10
   ```

2. **多選題**：
   ```
   Q: 這是多選題目內容？
   A: 選項 1
   B: 選項 2
   C: 選項 3
   ANS: A,C
   SCORE: 10
   ```

3. **是非題**：
   ```
   Q: 這是是非題內容？
   ANS: Y
   SCORE: 10
   ```

**自動識別邏輯**：
- 無選項且答案為 `Y/N/T/F/YES/NO` → 判斷為是非題
- 答案包含逗號或長度 > 1 且為字母組合 → 判斷為多選題
- 其他情況 → 判斷為單選題

### 3. 考試流程 (Exam Flow)

**前端考試介面** (`ExamRunner.tsx`)：
- **題目切換動畫**：使用 Framer Motion 實現滑動效果
- **進度條顯示**：即時顯示作答進度
- **計時器**：可選的時間限制（`ExamTimer.tsx`）
- **作答進度本地儲存**：使用 `useExamProgress` Hook 儲存至 localStorage
- **離開確認**：如有作答紀錄，離開時顯示確認對話框

**後端評分邏輯** (`exam_center.py` - `submit_exam`)：
1. 取得所有題目
2. 比對使用者答案與正確答案（嚴格比對）
3. 計算總分與獲得分數
4. 判斷是否通過（`earned_score >= passing_score`）
5. 儲存 ExamRecord 與 ExamDetail
6. 回傳結果（含每題詳情）

### 4. 前端特色功能 (Frontend Features)

**動態日曆圖示** (`App.tsx` - `CalendarIcon`)：
- SVG 生成，顯示當前日期時間
- 格式：`yyyy-MM-dd HH:MM:SS`
- 即時更新（每秒）

**響應式設計**：
- 支援手機、平板、桌面
- 使用 Tailwind CSS 響應式類別（`sm:`, `md:`, `lg:`）
- 行動端漢堡選單

**權限控制選單** (`App.tsx` - `Navbar`)：
- 依使用者權限動態顯示選單項目
- Admin 角色顯示所有功能
- 一般使用者僅顯示有權限的功能

```
backend/
├── app/
│   ├── __init__.py
│   ├── main.py                    # FastAPI 應用入口，註冊所有路由
│   ├── database.py                # SQLAlchemy 資料庫連線設定

│   ├── schemas.py                 # Pydantic 資料結構（請求/回應）
│   ├── auth_utils.py              # JWT Token 生成與驗證工具
│   ├── init_db.py                 # 資料庫初始化腳本
│   ├── routers/                   # API 路由模組
│   │   ├── __init__.py
│   │   ├── auth.py                # 認證路由（登入、註冊、驗證碼）
│   │   ├── admin.py               # 系統管理路由（單位、分類、人員、角色、權限、功能）
│   │   ├── training.py            # 訓練計畫路由
│   │   ├── exam.py                # 考卷上傳路由（TXT 解析、題目管理）
```

## 六、安全性與權限 (Security & Permissions)

### 安全性機制

1. **驗證碼機制**：
   - 4 位數字，使用 Pillow 生成
   - 暫存於記憶體字典（生產環境建議改用 Redis）
   - 驗證成功後立即刪除，避免重複使用

2. **JWT Token**：
   - 8 小時有效期
   - 自動附加到請求標頭（Axios interceptor）
   - Token 過期後需重新登入

3. **權限檢查**：
   - 每個管理功能都有 `check_permission()` 保護
   - 未授權存取回傳 403 Forbidden

4. **Admin 保護**：
   - 系統預設 Admin 角色不可刪除
   - Admin 角色的權限不可變更
   - Admin 帳號的角色與單位不可變更

### 權限代碼規範

| 代碼格式 | 說明 | 範例 |
|---------|------|------|
| `menu:*` | 選單項目 | `menu:exam`, `menu:plan`, `menu:report` |
| `menu:admin:*` | 管理後台子選單 | `menu:admin:dept`, `menu:admin:user`, `menu:admin:role` |
| `btn:*` | 按鈕操作 | `btn:ai_gen`, `btn:export` |
| `menu:exam_studio` | 考卷工坊功能 | 題庫管理、題目匯入 |

## 七、資料流程 (Data Flow)

### 1. 訓練計畫建立流程

```
系統管理 → 建立分類（大項目/細項目）
    ↓
建立訓練計畫（選擇細項目、開課單位、受課單位、設定日期與分數）
    ↓
考卷工坊 → 上傳題目（TXT 檔案或線上輸入）
    ↓
題目存入 questions 表（綁定訓練計畫）
    ↓
同步存入 question_bank 題庫（避免重複）
    ↓
考試中心 → 使用者選擇計畫開始考試
    ↓
提交答案 → 自動評分 → 儲存考試記錄
    ↓
成績中心 → 查看統計與報表
```

### 2. 題目匯入流程

```
上傳 TXT 檔案
    ↓
TXTParser.parse_content() 解析
    ↓
判斷題型（單選/多選/是非）
    ↓
存入 questions 表（plan_id 綁定）
    ↓
檢查 question_bank 是否已存在（以 content 判斷）
    ↓
不存在則新增至 question_bank（含標籤）
    ↓
備份原始檔案至 data/materials/{year}/{plan_id}/
```

### 3. 考試流程

```
使用者登入 → 取得考試列表（/exam/my_exams）
    ↓
選擇訓練計畫 → 開始考試（/exam/start/{plan_id}）
    ↓
取得題目（不含答案）→ 前端顯示
    ↓
使用者作答 → 本地儲存進度（localStorage）
    ↓
提交答案（/exam/submit/{plan_id}）
    ↓
後端自動評分 → 計算通過狀態
    ↓
儲存 ExamRecord 與 ExamDetail
    ↓
回傳結果 → 前端顯示成績卡
```
