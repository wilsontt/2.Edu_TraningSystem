| **不是** | 產品規格／長青維運準則（那些在 `00-專案總覽/`、`plans/已完成/`） |

## 慣例

- **新產生的** Superpowers 計畫：建議直接寫入本目錄（避免再占根目錄 `docs/`）。
- 功能結案後：在檔頭加歸檔說明，並以 `plans/已完成/` 與 [棕地功能總覽](../棕地功能總覽.md) 為準。
- 可執行測試維持在專案根目錄 **`tests/`**（不放入 `1.docs/`）。
