# 部門成員成績批次列印：考卷成績單與考試成績清單字型不一致（新版任務單）

日期：2026/05/02

目標檔案（依實際調整為準，至少涵蓋）：

- `Dockerfile.backend`
- `backend/app/routers/report.py`（`register_chinese_fonts` 內 Linux `font_paths`）
- `backend/requirements.txt`（若需補 `fonttools`）
- `README.md`（專案根目錄）
- `CLAUDE.md`

---

## 1. 目的

修正（並於文件說明）Docker 部署時，**考試成績清單**（後端 ReportLab PDF）與**考卷成績單**（瀏覽器 HTML 列印）**視覺字型差異過大、PDF 中文異常或欄位對不齊**的問題，達成：

- 後端容器內能穩定註冊**繁體中文** PDF 用字型，與 `register_chinese_fonts()` 掃描路徑一致。
- 主控文件清楚說明：兩種輸出**本來就不是同一條產線**，但容器**不得**因缺字型而讓清單 PDF 無法閱讀。

---

## 2. 兩種輸出與字型來源（須依此對齊實作與文件敘述）

| 輸出 | 觸發路徑（摘要） | 字型來源 |
|------|------------------|----------|
| **考卷成績單** | `DeptMemberScoreModal` → `POST /admin/reports/dept-plan/individual-print-data` → `scoreCardPrintHtml.buildBatchPrintHtml` + `printHtmlInIframe` | 使用者**瀏覽器／OS**；`CHINESE_FONT_STACK` 見 `frontend/src/components/personal/scoreCardPrintHtml.ts` |
| **考試成績清單** | `POST /admin/reports/dept-plan/pdf` → ReportLab | **後端容器**內字型檔；`register_chinese_fonts()` 見 `backend/app/routers/report.py` |

---

## 3. 根因與現況（重要）

1. **`Dockerfile.backend`** 使用 `python:3.12-slim-bookworm`，目前 `apt-get` 僅安裝 `fonts-dejavu-core` 與 `build-essential`，**未**提供 `report.py` 在 Linux 分支所列之中文字型路徑（例如 Noto CJK、文泉驛等）。
2. 當註冊失敗時，`register_chinese_fonts()` 會退到內建 CID 或 **Helvetica**，導致 PDF 中文顯示或字寬異常、表頭與資料**對齊跑掉**。
3. **考卷成績單**不依賴後端容器字型，故本機或瀏覽器環境正常時，與 PDF 清單**視覺不一致屬預期**；任務重點是**修正容器內 PDF**，並在 README／CLAUDE **寫清楚差異**，避免誤判為「同一套輸出卻壞掉」。

---

## 4. 實作要求（必做）

### 4.1 `Dockerfile.backend`

- 在既有 `apt-get install` 中增加套件，使容器內**至少一條**路徑與 `register_chinese_fonts()` 的 Linux `font_paths` 相符（例如 Debian bookworm 的 `fonts-noto-cjk` 或 `fonts-wqy-zenhei` 等；以實際安裝後檔案路徑為準）。
- 若套件實際安裝路徑與程式內常數**不一致**，請**同步**更新 `backend/app/routers/report.py` 的 Linux `font_paths`（擇一：改 Dockerfile 套件組合或改程式路徑，以可重現、可維護為準）。

### 4.2 TTC 與 `fonttools`

- 若註冊邏輯依賴從 `.ttc` 抽取 TTF，確認 `backend/requirements.txt` 是否已含 `fonttools`；若執行時需要且未列出，請補上。

### 4.3 主控文件 `README.md`

- 於「重要注意事項」或獨立小節（建議標題：**Docker 與成績 PDF 字型**）說明：
  - ReportLab 產出之 PDF 字型來自**後端映像**；
  - 瀏覽器列印之成績單字型來自**客戶端**；
  - Docker 部署後端須安裝 CJK 字型，並與 `register_chinese_fonts()` 路徑一致。

### 4.4 `CLAUDE.md`

- 於「環境與配置」或等價區塊新增一句：Docker 後端產 PDF 需映像內繁中字型，避免僅本機正常、容器內 PDF 異常。

### 4.5 程式風格與範圍

- 僅修改與本任務相關之檔案；勿無關重構。
- 新增或修改之註解維持**繁體中文**。
