`個人成績歷史 -> 查看詳情 -> 考試歷程記錄Modal -> 查看詳情 -> 成績詳情Modal -> 右上預覽成績單 -> 右上藍色列印按鈕（小印表機） -> 瀏覽器列印預覽`

### 必做
1. 部門 individual 請直接共用上述「個人成績預覽列印」同一套渲染來源（同函式/同模板/同資料格式）。
2. 禁止維持或新增獨立的「部門自製 individual 版型」。
3. 已移除的導流文案維持不出現（不要再新增替代句）。

### 驗收標準
- 同一位成員、同一計畫、同一筆考試資料，比對：
  - 個人路徑列印預覽
  - 部門 individual 列印預覽  
  需達成版型與內容一致（抬頭、資訊區、題目/作答結果、結果章、簽名區）。

## B) 成績清單（list）列印排版修正
### 需求定義
1. 表頭與列表文字需「垂直置中」。
2. 列高不足要修正，避免出席狀態時間被下一列遮到。
3. 出席狀態欄改為**同一列左右排列**：  
   `狀態文字` + 空一格 + `日期時間`  
   （禁止上下兩行顯示）

### 必做
- 調整 list PDF 表格繪製邏輯：字級、row height、baseline 計算。
- 出席狀態欄位繪製為單行排版，不得再畫第二行時間。
- 時間格式維持：`YYYY/MM/DD HH:mm`。

### 驗收標準
- 20+ 筆資料時，任何列都不會壓行或被遮蓋。
- 出席狀態欄每列都呈現「文字 + 時間」單行。
- 表頭與資料列肉眼可見垂直置中。

### 2. A 的共用來源說明

- **共用函式**：`buildBatchPrintHtml(members, includeSignature)`，路徑：`frontend/src/components/personal/scoreCardPrintHtml.ts`
- `ScoreCardPreview.handlePrint()` 傳入 `[{ has_exam: true, detail }]` 呼叫此函式
- `DeptMemberScoreModal.handlePrintPdf()` 呼叫 `POST /dept-plan/individual-print-data` 取得成員詳情，再以相同函式產生 HTML，以 `printHtmlInIframe()` 觸發瀏覽器列印
- 結果：兩者使用**完全相同的 TypeScript 函式**產生同一套 HTML 模板，達成「同源」

### 3. B 的表格繪製前後差異

| 項目 | 舊 | 新 |
|------|----|----|
| `row_h` | 18 | 26 |
| 文字基線偏移 | 0（直接 y） | `y + text_vcenter`（text_vcenter=6） |
| 出席狀態 | 兩行（狀態、時間分行，y-9 第二行溢出 rect） | 單行：`att_status + " " + att_time_str` |
| 分頁判斷閾值 | `y < 70` | `y < 80` |

### 4. C 的容器對齊策略

- **原來**：`<div className="space-y-4">` 為 block 100% 寬，靠左
- **修改**：外層加 `<div className="flex justify-end">` wrapper，內層加 `w-full max-w-3xl`
- 效果：整個成員成績列表（含成績分布、Top 10）整體偏右，最大寬度 48rem（768px）
