# 成員成績列表欄寬調整（新版任務單）
日期：2026/05/01
目標檔案：`frontend/src/components/admin/ReportDashboard.tsx`

## 1. 目的
修正「部門績效表現 > 展開部門 > 成員成績列表」的寬度使用問題，達成：
- 盡量利用可用水平空間
- 避免右側大塊無效留白
- 避免不必要水平捲軸
- 保持整個展開內容靠右對齊的視覺需求

---

## 2. 本次欄位寬度規格（字元寬度基準）
以 `ch` 為主要寬度單位，固定如下：

- 姓名：15
- 計劃：30
- 分數：10
- 是否通過：15
- 提交時間：20（數字寬）
- 操作：8（中文寬）

---

## 3. 關鍵調整策略（重要）
### 3.1 外層不要再用固定 `max-w-3xl` 卡死整塊
目前外層 `max-w-3xl` 會同時限制表格/圖表/Top10，造成：
- 表格可用寬度不足
- 右側留白過大
- 更容易出水平捲軸

請改為：
- 外層維持 `w-full` + `flex justify-end`
- 移除展開內容主容器的固定 `max-w-3xl`

### 3.2 只對「成員成績列表」區塊做寬度控制
在「成員成績列表」表格外層新增專屬 wrapper（僅作用於這一塊）：
- `w-full`
- `ml-auto`（保持靠右）
- `max-w-[calc(100%-...)]` 或適當 `max-w-[XXrem]`
- `min-w-[欄位總寬 + padding + 操作按鈕寬度]`

> 重點：寬度控制要局部化，不要影響同層的成績分布圖與 Top10 版面。

---

## 4. 程式碼可維護性要求（必做）
### 4.1 欄寬常數集中管理
請在 `ReportDashboard.tsx` 建立清楚註解的欄寬常數，例如：

- `MEMBER_TABLE_COL_WIDTH_CH`
  - `name: 15`
  - `plan: 30`
  - `score: 10`
  - `isPassed: 15`
  - `submittedAt: 20`
  - `actions: 8`

### 4.2 註解文字（請保留）
在常數上方加註解：
- 「成員成績列表欄寬設定（單位：ch）。後續若需調整欄寬，直接修改此處數值。」

---

## 5. 表格行為要求
1. 表格維持 `table-fixed`。
2. 儲存格統一：
   - `whitespace-nowrap`
   - `overflow-hidden`
   - `text-ellipsis`
3. 提交時間欄保持 `tabular-nums`。
4. 禁止用 `pr-[86px]`、`pr-[30px]` 之類硬編碼 padding 作為主要對齊手段。

---

## 6. 驗收標準
- [ ] 展開區塊右側無明顯大塊無效空白
- [ ] 成員成績列表盡量吃滿可用寬度（但不貼邊壓迫）
- [ ] 一般資料情境不出現水平捲軸
- [ ] 長文字時以省略號處理，不折行
- [ ] 調整欄寬只需改常數，不需改多處 class
