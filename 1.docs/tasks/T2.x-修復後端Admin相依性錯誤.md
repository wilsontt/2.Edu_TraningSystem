# 後端啟動錯誤修復報告

## 發現的問題
後端伺服器啟動失敗，出現 `TypeError: Depends(...) is not a callable object` 錯誤。這導致使用者無法進入登入畫面及取得驗證碼。

## 根本原因
在 `backend/app/routers/admin.py` 中，`check_permission` 函式被錯誤地包裹在 `Depends(...)` 中（例如：`Depends(check_permission(...))`）。
然而，`check_permission` 本身的定義已經**回傳**了一個 `Depends` 物件（例如：`return Depends(...)`）。
再次包裹會導致 `Depends(Depends(...))` 的情況，外層的 `Depends` 預期接收一個可呼叫物件（callable），但卻收到了一個 `Depends` 實例。

## 修正內容
- 修改 `backend/app/routers/admin.py`，直接呼叫 `check_permission(...)`，不再將其包裹於 `Depends()` 中。

## 驗證
- 透過終端機輸出確認 `uvicorn` 已偵測到變更並成功重新載入：
  ```
  INFO:     Started server process [39693]
  INFO:     Waiting for application startup.
  INFO:     Application startup complete.
  ```
- API 現在應可存取，允許登入頁面順利取得驗證碼。
