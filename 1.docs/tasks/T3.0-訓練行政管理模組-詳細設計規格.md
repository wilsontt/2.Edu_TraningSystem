# T3: 訓練行政管理模組 - 詳細設計規格

本文件定義了 T3 模組的技術實作細節，包括 API 契約、數據模型與前端設計。

## 1. 數據模型 (Pydantic Schemas)

### 單位 (Department)
```python
class DepartmentBase(BaseModel):
    name: str

class DepartmentCreate(DepartmentBase):
    pass

class Department(DepartmentBase):
    id: int
    class Config:
        from_attributes = True
```

### 分類 (Category)
```python
class MainCategoryBase(BaseModel):
    name: str

class SubCategoryBase(BaseModel):
    name: str
    main_id: int

class SubCategory(SubCategoryBase):
    id: int
    class Config:
        from_attributes = True

class MainCategory(MainCategoryBase):
    id: int
    sub_categories: List[SubCategory] = []
    class Config:
        from_attributes = True
```

### 訓練計畫 (Training Plan)
```python
class TrainingPlanCreate(BaseModel):
    title: str
    sub_category_id: int
    dept_id: int
    training_date: date
    timer_enabled: bool = False
    time_limit: int = 0

class TrainingPlan(TrainingPlanCreate):
    id: int
    year: str
    class Config:
        from_attributes = True
```

## 2. API 路由規劃 (`backend/app/routers/admin.py`)

| 屬性 | 路徑 | 方法 | 說明 | 權限要求 |
| :--- | :--- | :--- | :--- | :--- |
| 單位 | `/api/admin/departments` | GET | 獲取所有單位 | menu:admin:dept |
| | `/api/admin/departments` | POST | 新增單位 | menu:admin:dept |
| | `/api/admin/departments/{id}` | PUT | 修改單位 | menu:admin:dept |
| 分類 | `/api/admin/categories/main` | GET | 獲取大項目清單 | menu:plan (或是 admin) |
| | `/api/admin/categories/sub/{main_id}` | GET | 獲取大項下的細項 | menu:plan |
| 計畫 | `/api/admin/training-plans` | POST | 建立新的訓練計畫 | menu:plan |
| | `/api/admin/training-plans` | GET | 獲取計畫清單 | menu:plan |

## 3. 前端介面設計

### A. 單位管理頁面 (`/admin/departments`)
- **佈局**: 使用 Shadcn-like Table。
- **功能**: 
    - 點擊「新增」彈出 Modal。
    - 列表顯示「修改/刪除」按鈕。

### B. 訓練計畫建立頁面 (`/plans`)
- **佈局**: 多步驟表單或分類 Select 選單。
- **互動**:
    - 第一層下拉：選擇大項目。
    - 第二層下拉：依據大項連動顯示細項目。
    - 第三層下拉：選擇開課地點 (Departments)。
    - 日期選擇器：選擇訓練日期。

## 4. 驗證計畫

### 自動化測試
- 使用 `pytest` 測試 Admin CRUD API。
- 驗證非 Admin 角色請求 `PUT/POST/DELETE` 應返回 403。

### 手動測試
- 建立一個新的單位，觀察 Login 頁面的部門下拉選單是否同步。
- 建立訓練計畫，確認資料庫中 `year` 欄位是否正確提取 `yyyy`。
