# 考試系統與後台管理改進說明 (Archived)

## 主要變更

### 1. 考試介面優化 (Exam Runner UI Adjustment)
- **問題**：在較小的螢幕上，「下一題」按鈕有時會被遮擋。
- **解決方案**：重構了 `ExamRunner.tsx`，使用固定佈局 (`h-screen`, `flex-col`)，並將中間的題目區域設為可捲動 (`flex-1 overflow-y-auto`)。這確保了導航按鈕始終固定在底部可見。

### 2. 考試重考機制 (Exam Retake Mechanism)
- **後端**：
    - 更新 `ExamRecord` 模型，新增 `attempts` (挑戰次數) 欄位。
    - 修改 `submit_exam` API 以允許重複提交，每次提交都會增加嘗試次數並更新分數/狀態，同時保持 `plan_id` 和 `emp_id` 的唯一性 (使用更新邏輯)。
- **前端 (ExamDashboard)**：
    - 更新儀表板，針對未通過 (分數 < 60) 的考試顯示「Retry」按鈕。
    - 顯示「挑戰次數」供使用者查看。
    - 允許點擊未通過的考試以重新開始測驗。

### 3. 列表項目編號 ("項次")
- **實作**：在主要的管理列表中，將原本顯示的資料庫 ID 替換為流水號 (`index + 1`)：
    - **TrainingPlanManager.tsx** (訓練計畫管理)
    - **UserManager.tsx** (用戶管理)
    - **RoleManager.tsx** (角色管理)
- **效益**：提供更整潔、更符合使用者預期的列表編號方式。

### 4. 權限管理體驗優化 (Permission Manager UX)
- **問題**：原本的瀏覽器預設確認視窗會顯示 IP 位址，觀感不佳。
- **解決方案**：實作了自定義的 `ConfirmModal` 組件。
- **結果**：在有未儲存變更的情況下切換角色時，會觸發美觀的確認對話框，且不再顯示 IP 位址等技術細節。

### 5. 角色刪除功能 (Role Deletion)
- **後端**：新增 `DELETE /roles/{role_id}` API 端點，並包含嚴格驗證：
    - 不可刪除 "Admin" (系統管理員) 角色。
    - 不可刪除尚有成員的角色。
    - 不可刪除尚有權限設定的角色。
- **前端**：在 `RoleManager` 的角色卡片上新增了 **刪除** 按鈕 (垃圾桶圖示，僅對非 Admin 角色顯示)。點擊後會透過 `ConfirmModal` 確認再呼叫 API 進行刪除。

### 6. 考試離開確認 (Exit Confirmation)
- **實作**：在 `ExamRunner` 中攔截返回按鈕事件。
- **邏輯**：
    - 若使用者尚未作答：允許直接離開。
    - 若使用者已作答至少一題：顯示 `ConfirmModal` 警告，提示離開將不會保存進度。

### 7. 考試中心增強功能 (Phase 4)
- **受課對象 (Target Audience)**：
  - 新增多選功能，允許訓練計畫指定多個部門。
  - 預設提供「同開課單位」快速按鈕。
  - 考試中心會根據使用者的部門是否在受課對象清單中，來決定是否顯示該考試。
- **及格分數 (Passing Score)**：
  - 在新增/編輯計畫時可設定及格分數 (絕對分數，預設 60)。
  - 系統判斷通過與否時，將使用此設定值 (大於等於即通過)。

所有需求皆已實作完成。
