# 人員管理排序功能實作計劃

**建立日期**: 2026-01-13  
**狀態**: 待實作

## 目標

在人員管理頁面中，實作可點擊表頭欄位進行排序的功能，同時確保 admin 帳號永遠固定在第一順位。

## 需求分析

### 核心需求

1. **Admin 固定第一順位**：admin 帳號永遠排在第一個，不受任何排序影響
2. **預設排序**：其餘帳號預設依員工編號排序（遞增）
3. **可點擊排序**：可以點擊表頭欄位來決定排序方式
4. **排序切換**：點擊同一欄位可以循環切換：無排序 → 遞增 → 遞減 → 無排序

### 可排序欄位

| 欄位 | 排序依據 | 資料類型 | 備註 |
|------|---------|---------|------|
| 員工編號 | `user.emp_id` | 字串 | 預設排序欄位 |
| 姓名 | `user.name` | 字串 | - |
| 部門 | `user.department?.name` | 字串 | 可能為 null |
| 角色 | `user.role?.name` | 字串 | 可能為 null |
| 狀態 | `user.status` | 字串 | 'active' 或 'inactive' |

## 實作方案

### 1. 狀態管理設計

```typescript
// 排序狀態
const [sortConfig, setSortConfig] = useState<{
  field: 'emp_id' | 'name' | 'dept_id' | 'role_id' | 'status' | null;
  direction: 'asc' | 'desc' | null; // null 表示無排序（回到預設）
}>({
  field: 'emp_id', // 預設按員工編號
  direction: 'asc'  // 預設遞增
});
```

**說明**：
- `field: null` 且 `direction: null` 表示無自訂排序，使用預設排序（員工編號遞增）
- 預設狀態為 `{ field: 'emp_id', direction: 'asc' }`

### 2. 排序邏輯設計

```typescript
const getSortedUsers = (usersToSort: User[]) => {
  // 1. 分離 admin 和其他用戶
  const adminUser = usersToSort.find(u => u.emp_id.toLowerCase() === 'admin');
  const otherUsers = usersToSort.filter(u => u.emp_id.toLowerCase() !== 'admin');
  
  // 2. 對其他用戶進行排序
  let sortedOthers = [...otherUsers];
  
  // 如果沒有排序設定，使用預設排序（員工編號遞增）
  const effectiveField = sortConfig.field || 'emp_id';
  const effectiveDirection = sortConfig.direction || 'asc';
  
  sortedOthers.sort((a, b) => {
    let aValue: string | number;
    let bValue: string | number;
    
    switch (effectiveField) {
      case 'emp_id':
        aValue = a.emp_id;
        bValue = b.emp_id;
        break;
      case 'name':
        aValue = a.name;
        bValue = b.name;
        break;
      case 'dept_id':
        aValue = a.department?.name || '';
        bValue = b.department?.name || '';
        break;
      case 'role_id':
        aValue = a.role?.name || '';
        bValue = b.role?.name || '';
        break;
      case 'status':
        aValue = a.status;
        bValue = b.status;
        break;
      default:
        return 0;
    }
    
    // 字串比較（使用 localeCompare 支援中文）
    const comparison = String(aValue).localeCompare(String(bValue), 'zh-TW');
    return effectiveDirection === 'asc' ? comparison : -comparison;
  });
  
  // 3. 合併：admin 在前，其他在後
  return adminUser ? [adminUser, ...sortedOthers] : sortedOthers;
};
```

### 3. 表頭點擊處理

```typescript
const handleSort = (field: 'emp_id' | 'name' | 'dept_id' | 'role_id' | 'status') => {
  setSortConfig(prev => {
    // 如果點擊的是當前欄位，切換排序方向
    if (prev.field === field) {
      if (prev.direction === 'asc') {
        // 遞增 → 遞減
        return { field, direction: 'desc' };
      } else if (prev.direction === 'desc') {
        // 遞減 → 無排序（回到預設）
        return { field: null, direction: null };
      }
    }
    // 點擊新欄位，設為遞增
    return { field, direction: 'asc' };
  });
};
```

**排序切換邏輯**：
- 第一次點擊欄位：設為遞增（asc）
- 第二次點擊同一欄位：切換為遞減（desc）
- 第三次點擊同一欄位：取消排序，回到預設（null）

### 4. UI 設計

#### 表頭設計

```typescript
// 可排序欄位的表頭
<th 
  onClick={() => handleSort('emp_id')}
  className="cursor-pointer hover:bg-gray-100 transition-colors px-6 py-4 text-left text-sm font-black text-gray-400 uppercase tracking-wider"
>
  <div className="flex items-center gap-2">
    <span>員工編號</span>
    {sortConfig.field === 'emp_id' ? (
      sortConfig.direction === 'asc' ? (
        <ArrowUp className="w-4 h-4 text-blue-600" />
      ) : (
        <ArrowDown className="w-4 h-4 text-blue-600" />
      )
    ) : (
      <ArrowUpDown className="w-4 h-4 text-gray-300 opacity-0 group-hover:opacity-100" />

  </div>
</th>
```

#### 視覺指示器

- **無排序**：顯示半透明的 `ArrowUpDown` 圖示（hover 時顯示）
- **遞增排序**：顯示藍色的 `ArrowUp` 圖示（↑）
- **遞減排序**：顯示藍色的 `ArrowDown` 圖示（↓）
- **當前排序欄位**：可選的背景色高亮

### 5. 整合搜尋與排序

```typescript
// 完整的資料處理流程
const processedUsers = useMemo(() => {
  // 1. 搜尋過濾
  const filtered = users.filter(user => 
    user.name.toLowerCase().includes(searchTerm.toLowerCase()) || 
    user.emp_id.includes(searchTerm)
  );
  
  // 2. 排序（包含 admin 固定第一）
  return getSortedUsers(filtered);
}, [users, searchTerm, sortConfig]);
```

## 實作細節

### 排序欄位對應表

| 表頭欄位 | 排序欄位 | 排序依據 | 空值處理 |
|---------|---------|---------|---------|
| 員工編號 | `emp_id` | `user.emp_id` | 不會為空 |
| 姓名 | `name` | `user.name` | 不會為空 |
| 部門 | `dept_id` | `user.department?.name` | 空值視為空字串 |
| 角色 | `role_id` | `user.role?.name` | 空值視為空字串 |
| 狀態 | `status` | `user.status` | 不會為空 |

### 特殊處理

1. **Admin 固定第一**：
   - 排序前先分離 admin
   - 排序後再合併到最前面
   - 無論如何排序，admin 永遠在第一個

2. **空值處理**：
   - 部門/角色為 null 時，排序時視為空字串 `''`
   - 空值在遞增排序時通常排在最後
   - 空值在遞減排序時通常排在最前

3. **中文排序**：
   - 使用 `localeCompare` 並指定 `'zh-TW'` 區域設定
   - 確保中文姓名和部門名稱正確排序

4. **項次重新計算**：
   - 排序後，項次需要重新計算
   - Admin 永遠是第 1 項
   - 其他用戶依排序後的順序編號

## 完整資料流程

```
1. 載入用戶列表（users）
   ↓
2. 搜尋過濾（filteredUsers）
   - 根據 searchTerm 過濾姓名或員工編號
   ↓
3. 分離 admin 和其他用戶
   - adminUser = users.find(u => u.emp_id === 'admin')
   - otherUsers = users.filter(u => u.emp_id !== 'admin')
   ↓
4. 對其他用戶進行排序（根據 sortConfig）
   - 如果 sortConfig 為 null，使用預設（員工編號遞增）
   - 根據欄位類型進行比較
   ↓
5. 合併：admin + 排序後的其他用戶
   ↓
6. 渲染列表（項次重新計算）
```

## 測試要點

1. **Admin 固定第一**：
   - 測試各種排序情況下，admin 是否永遠在第一個
   - 測試搜尋後，admin 是否仍然在第一個（如果符合搜尋條件）

2. **預設排序**：
   - 初始載入時，確認其他用戶按員工編號遞增排序
   - Admin 在第一個

3. **排序切換**：
   - 點擊員工編號欄位：遞增 → 遞減 → 無排序（回到預設）
   - 點擊其他欄位：直接設為遞增
   - 切換欄位時，確認排序正確

4. **空值處理**：
   - 測試部門為 null 的用戶排序
   - 測試角色為 null 的用戶排序
   - 確認空值排序位置正確

5. **搜尋與排序整合**：
   - 搜尋後排序，確認結果正確
   - 排序後搜尋，確認結果正確
   - 搜尋結果中，admin 仍然在第一個（如果符合搜尋條件）

6. **項次顯示**：
   - 確認排序後項次正確
   - Admin 永遠是第 1 項
