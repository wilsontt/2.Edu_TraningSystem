# 修復單位與角色管理功能問題

**建立日期**: 2026-01-13  
**狀態**: 待實作

## 問題分析

### 問題1：單位設定，按編輯並沒有可以看到有「新增成員」按鈕

**分析**：
- 用戶點擊「編輯」按鈕編輯單位名稱時，期望能看到「新增成員」按鈕
- 目前編輯模式下只有輸入框和確認/取消按鈕，沒有成員管理入口
- 「新增成員」按鈕目前只在「查看成員」模態框中（需要點擊單位名稱才能看到）

**解決方案**：在編輯單位名稱時，也要在操作欄位中提供「查看成員」按鈕，讓用戶可以在編輯模式下也能管理成員

### 問題2：單位管理清單列，ID欄改為項次，使用流水號，按單位名稱排序

**需求**：
- 將「ID」欄位名稱改為「項次」
- 使用索引+1作為流水號（而非資料庫ID）
- 列表按單位名稱排序

### 問題3：角色管理，當user的角色變更後，該角色卡片上的成員數也要重新計算

**分析**：
- 當用戶角色變更後，需要重新載入角色列表以更新成員數統計
- 目前可能只在查看成員時更新，但角色卡片上的數字未更新

### 問題4：其他管理頁面缺少雙擊編輯功能

**需求**：
- 人員管理已有雙擊編輯功能（`onDoubleClick={() => handleEdit(user)}`）
- 需要在其他管理頁面（單位管理、角色管理、分類管理等）也加入雙擊編輯功能
- 訓練計劃、考卷工坊等也需要雙擊編輯

2. 在 `handleRemoveMemberFromRole` 完成後，重新載入角色列表：

3. 確保 `fetchRoles` 會重新計算每個角色的 `user_count`

**實作範例**：
