# 測試文件與審查報告目錄

本目錄存放所有測試相關的文件和審查報告，包括：
- API 測試手冊
- 測試報告
- 修復記錄
- 已知問題清單
- 其他驗證測試產生的文件

## 文件組織規範

### 測試文件位置
- **所有測試文件/報告** → `1.docs/reviews/` 目錄
- **所有測試程式** → `tests/` 目錄

### 檔案命名規範

建議使用以下命名格式：
- `test_*.md` - 測試手冊或測試說明
- `*測試報告*.md` - 測試結果報告
- `*修復記錄*.md` - 修復問題的記錄
- `*已知問題*.md` - 已知問題清單

## 現有文件

- `test_api_manual.md` - API 測試手冊，包含所有 API 端點的測試方法和範例

## 注意事項

1. 所有測試相關的文件都應放在此目錄
2. 測試程式應放在 `tests/` 目錄
3. 測試產生的臨時文件（如測試資料庫、測試上傳檔案）應在 `.gitignore` 中排除
