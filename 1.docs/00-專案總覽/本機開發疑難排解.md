## 一、目的

彙整本機開發時，**前端（npm / Vite）**與**後端（Python / `.venv` / Uvicorn）**常見異常、故障與無法重新安裝等問題之**判斷方式與修復步驟**，供開發者與維運人員快速排除，無須翻查零散對話紀錄。

## 四、名詞解釋

| 名詞 | 說明 |
|------|------|
| **`uv_cwd`** | Node.js / npm 讀取「目前工作目錄」失敗；通常代表 **terminal 的 cwd 已失效**（目錄被刪除、移動或 session 過期） |
| **幽靈 cwd** | Terminal 提示字仍顯示某路徑，但 OS 已無法解析該目錄 |
| **損壞的 `.venv`** | 虛擬環境內 Python 的 `sys.prefix` 異常（如 `/install`），或找不到 `encodings` 等標準函式庫 |
| **uv standalone Python** | 由 [uv](https://github.com/astral-sh/uv) 管理的 Python，prefix 常為 `/install`；**不建議**直接 `python3 -m venv` 建 venv |
| **proxy ECONNREFUSED** | Vite 將 `/training/api/*` 轉發至 `localhost:8000` 時，**後端未啟動**或埠不符 |

## 五、故障對照與修復步驟

### 5.1 前端：npm 無法執行（`uv_cwd`）

#### 症狀

```text
Error: ENOENT: no such file or directory, uv_cwd
  syscall: 'uv_cwd'
```

`npm install`、`npm run dev` 在**同一個已開很久的 terminal** 突然全部失敗；專案目錄其實仍存在。

#### 原因

Terminal 的工作目錄（cwd）已失效。常見觸發：

- 在 terminal 仍開啟時刪除／重建 `frontend/` 或 `node_modules/`
- 專案資料夾被移動或重新命名
- Git 操作清掉目錄後還原
- Cursor / VS Code **舊 terminal session** 未關閉即重開工作區

#### 修復

1. **關閉**出錯的 terminal，**開新 terminal**（不要沿用舊 session）
2. 以**絕對路徑**進入目錄：

```bash
cd "/Users/wilson/5.Projects/3.企業入口網站/2.教育訓練教材及線上考卷/frontend"
pwd    # 必須印出路徑，不可報錯
npm install
npm run dev
```

或在專案根目錄：

```bash
cd "/Users/wilson/5.Projects/3.企業入口網站/2.教育訓練教材及線上考卷"
npm run dev   # 根 package.json 會轉發至 frontend
```

3. 驗證：Vite 顯示 `ready`，網址含 `/training/`（預設 `http://localhost:5173/training/`）

> **注意**：在幽靈 cwd 的 terminal 內執行 `cd frontend` 通常**無法**修復，請開新 terminal。

---

### 5.2 前端：頁面可開但 API 全失敗（`proxy error ECONNREFUSED`）

#### 症狀

Vite 正常，但 terminal 或瀏覽器出現：

```text
[vite] http proxy error: /api/auth/me
AggregateError [ECONNREFUSED]
```

登入、驗證碼、部門列表等請求失敗。

#### 原因

後端 FastAPI **未在 port 8000 運行**。前端 dev proxy 設定見 `frontend/vite.config.ts`：`/training/api/*` → `http://localhost:8000/api/*`。

#### 修復

1. 另開 terminal，依 [§5.3](#53-後端虛擬環境損壞無法啟動) 或 [§5.4](#54-後端正常啟動檢查清單) 啟動後端
2. 確認 `http://localhost:8000/docs` 可開啟
3. 重新整理前端頁面

---

### 5.3 後端：虛擬環境損壞、無法啟動

#### 症狀 A — 找不到 Python

```text
zsh: no such file or directory: .venv/bin/python3
```

#### 症狀 B — 標準函式庫缺失

```text
ModuleNotFoundError: No module named 'encodings'
Fatal Python error: init_fs_encoding: failed to get the Python codec of the filesystem encoding
```

錯誤詳情中常見：

```text
sys.prefix = '/install'
stdlib dir = '/install/lib/python3.12'
sys._base_executable = '.../uv/python/cpython-3.12.12-macos-x86_64-none/...'
```

#### 症狀 C — 在損壞 venv 上重建失敗

```text
python3 -m venv .venv
Error: Command '... ensurepip ...' returned non-zero exit status 1.
```

#### 原因

1. `backend/.venv` **不在版控內**，可能從未建立、被刪除或半殘留
2. 預設 `python3` 指向 **uv standalone Python**（`~/.local/bin/python3`）時，用 `python3 -m venv` 常建出 **prefix 為 `/install` 的損壞 venv**
3. 在**未刪除**舊 `.venv` 的情況下再次 `python3 -m venv .venv`，會疊加更亂的狀態

#### 修復（macOS / Linux，已安裝 uv — 建議）

```bash
cd "/Users/wilson/5.Projects/3.企業入口網站/2.教育訓練教材及線上考卷/backend"

# 1. 必須整個刪除損壞的 venv
rm -rf .venv

# 2. 用 uv 建立（勿用 python3 -m venv）
uv venv .venv --python 3.12
uv pip install -r requirements.txt

# 3. 驗證
.venv/bin/python3 -c "import encodings, uvicorn; print('OK')"

# 4. 啟動
export PYTHONPATH=$PYTHONPATH:.
.venv/bin/python3 -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 修復（macOS / Linux，Homebrew Python）

若未安裝 uv，或 uv 建 venv 失敗：

```bash
cd backend
rm -rf .venv

brew install python@3.12   # 若尚未安裝
/opt/homebrew/bin/python3.12 -m venv .venv   # Apple Silicon
# Intel Mac 可能為：/usr/local/bin/python3.12 -m venv .venv

.venv/bin/python3 -m pip install -r requirements.txt
export PYTHONPATH=$PYTHONPATH:.
.venv/bin/python3 -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 修復（Windows PowerShell）

```powershell
cd backend
Remove-Item -Recurse -Force .venv -ErrorAction SilentlyContinue

python -m venv .venv
# 若 python 找不到：py -3 -m venv .venv

.\.venv\Scripts\python.exe -m pip install -r requirements.txt
$env:PYTHONPATH = "$PWD;$env:PYTHONPATH"
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

> Windows 請使用 `.\.venv\Scripts\python.exe`，**不可**使用 `.venv/bin/python3`。

### 5.4 後端：正常啟動檢查清單

| 步驟 | 指令 | 預期 |
|------|------|------|
| 1 | `test -x .venv/bin/python3 && echo OK`（Windows：`Test-Path .\.venv\Scripts\python.exe`） | `OK` / `True` |
| 2 | `.venv/bin/python3 -c "import encodings; print('OK')"` | `OK` |
| 3 | `export PYTHONPATH=$PYTHONPATH:.` 後啟動 uvicorn | `Uvicorn running on http://0.0.0.0:8000` |
| 4 | 瀏覽器開 `http://localhost:8000/docs` | Swagger UI 正常 |

---

### 5.5 前端：依賴異常或需乾淨重裝

#### 症狀

- `npm install` 報錯、套件版本衝突
- `npm run dev` 找不到模組
- 換 Node 版本後行為異常

#### 修復

```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

建議 Node.js **20+**（與 `Dockerfile.frontend` 一致時用 **22**）。以 `node --version` 確認。

---

### 5.6 埠號被占用

| 服務 | 預設埠 | 現象 | 處理 |
|------|--------|------|------|
| Vite | 5173 | 自動改用 5174… | 以 terminal 顯示網址為準 |
| FastAPI | 8000 | `Address already in use` | 關閉占用 8000 的舊 uvicorn，或 `lsof -i :8000` 查 PID 後結束 |

---

### 5.7 快速決策樹

```text
npm 報 uv_cwd？
  └─ 是 → 新 terminal + 絕對路徑 cd → 再 npm

npm / vite 正常但 API 失敗（ECONNREFUSED）？
  └─ 是 → 啟動後端 port 8000

.venv/bin/python3 不存在或 encodings 錯誤？
  └─ 是 → rm -rf .venv → uv venv（或 Homebrew python -m venv）→ pip install → uvicorn

python3 -m venv 的 ensurepip 失敗？
  └─ 是 → 先 rm -rf .venv，改用 uv venv 或 Homebrew Python
```
