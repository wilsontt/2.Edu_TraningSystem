# 生產部署指南（Docker）

**文件類型**：長青維運文件  
**最後更新**：2026-07-09  
**適用**：Linux Docker 生產環境（企業入口 `enterprise-portal` 整合部署）

---

## 1. 目的

集中說明「線上教育訓練系統」於 **Linux Docker** 之建置、環境變數、資料庫遷移、NAS／AD 設定與驗證步驟，避免僅修改本機 `backend/.env` 卻期望容器生效的常見錯誤。

## 2. 範圍

| 涵蓋 | 不涵蓋 |
|------|--------|
| `training-backend`／`training-frontend` 容器 | Windows／macOS 本機開發（見 [本機開發疑難排解](本機開發疑難排解.md)） |
| `deploy/.env` 之 `TRAINING_*` 變數 | 企業入口其他子系統之 compose 服務 |
| SQLite volume、DB 遷移、SMB／AD | NAS 管理員 ACL 設定細節（僅說明責任分界） |
| PDF 繁中字型、Fernet 密文 | 本機 `python -m venv` 流程 |

## 3. 權責

| 角色 | 責任 |
|------|------|
| 部署／維運 | 維護 `deploy/.env`、volume 備份、容器 recreate、執行遷移腳本 |
| 後端開發 | 提供 `Dockerfile.backend`、遷移腳本；**不**將含密 `.env` 打入映像 |
| IT／AD | DC 連線、IT Admins 群組、break-glass 初始密碼政策 |
| NAS 管理 | SMB 共享、service／backup 帳號、materials／backups ACL |

## 4. 名詞解釋

| 名詞 | 說明 |
|------|------|
| **`deploy/.env`** | 企業入口部署目錄下的環境變數；`TRAINING_*` 前綴映射至容器內無前綴變數 |
| **`DATA_ROOT`** | 主機上掛載至容器 `/data` 的目錄；含 `education_training.db` |
| **`force-recreate`** | `docker compose up -d --force-recreate training-backend`；改 `.env` 後必做 |
| **邏輯 SMB 路徑** | `MATERIALS_ROOT` 等以 `/` 分段，非本機 `D:\` 或 `/mnt/` |
| **`enc:` 密文** | Fernet 加密字串；解密金鑰為 `TRAINING_CREDENTIAL_SECRET` |

## 5. 作業內容

### 5.1 部署架構概觀

```
瀏覽器 → Nginx（training-frontend）→ 靜態 SPA /training/
              ↓ /training/api/*
         training-backend:8000（FastAPI）
              ↓ SQLAlchemy
         volume: /data/education_training.db
              ↓ smbprotocol（短連線）
         NAS（考卷 TXT、教材、排程備份 ZIP）
```

- **映像建置**：本子專案根目錄之 `Dockerfile.backend`、`Dockerfile.frontend`（前端建置需同層 `0.shared-ui`）。
- **Compose 定義**：位於企業入口 **`deploy/docker-compose.yml`**（路徑範例：`/opt/apps/enterprise-portal/deploy/`）。
- **環境變數範本**：`deploy/.env.example`（含 `TRAINING_*` 註解區塊）。

> **重要**：容器映像**不含** `backend/.env`。本機開發設定與生產設定**分離**。

### 5.2 首次部署檢查清單

1. **建置映像**（於企業入口 CI 或手動；依實際 pipeline 為準）。
2. **準備 volume**：`${DATA_ROOT}/training/education_training.db`（可自本機備份複製或執行 `init_db`）。
3. **編輯 `deploy/.env`**：至少設定下列區塊（見 §5.3）。
4. **啟動容器**：

```bash
cd /opt/apps/enterprise-portal/deploy
docker compose up -d training-backend training-frontend
```

5. **執行 DB 遷移**（依目前 DB 版本；見 [MIGRATION_GUIDE](資料庫遷移/MIGRATION_GUIDE.md)）：

```bash
# 範例：AD 欄位
docker compose exec training-backend python migrations/add_ad_auth_user_fields.py

# 範例：教材表
docker compose exec training-backend python migrations/add_teaching_materials.py

# 範例：報到 backfill（上線強制報到檢查前）
docker compose exec training-backend python migrations/backfill_attendance_from_exam.py

# 範例：已通過授權重考（exam_records.retake_authorized）
docker compose exec training-backend python migrations/add_exam_retake_authorization.py

# 範例：授權重考歷程對應（consumed_history_id）
docker compose exec training-backend python migrations/add_retake_auth_consumed_history_id.py
```

6. **驗證健康狀態**：

```bash
curl -s http://localhost:8000/docs   # 或經反向代理之路徑
docker compose exec training-backend python -c \
  "from app.config import get_settings; s=get_settings(); \
   print('smb=', s.smb_configured, 'ad=', s.ad_configured)"
```

### 5.3 環境變數（`deploy/.env`）

#### 5.3.1 對照表（`TRAINING_*` → 容器內）

| `deploy/.env` | 容器內 | 用途 |
|---------------|--------|------|
| `TRAINING_JWT_SECRET_KEY` | `JWT_SECRET_KEY` | JWT 簽章（生產必設） |
| `TRAINING_SMB_SERVER` | `SMB_SERVER` | NAS 主機 |
| `TRAINING_SMB_SHARE` | `SMB_SHARE` | SMB 共享名 |
| `TRAINING_SMB_AUTH_DOMAIN` | `SMB_AUTH_DOMAIN` | 教材 NAS 登入自動補網域 |
| `TRAINING_MATERIALS_ROOT` | `MATERIALS_ROOT` | 教材根（共享內邏輯路徑） |
| `TRAINING_BACKUP_ROOT` | `BACKUP_ROOT` | 備份根 |
| `TRAINING_EXAM_SMB_USERNAME` | `EXAM_SMB_USERNAME` | 考卷 service 帳號 |
| `TRAINING_EXAM_SMB_PASSWORD` | `EXAM_SMB_PASSWORD` | 考卷 service 密碼 |
| `TRAINING_CREDENTIAL_SECRET` | `CREDENTIAL_SECRET` | Fernet 金鑰 |
| `TRAINING_SMTP_*` | `SMTP_*` | Email OTP；密碼建議 `enc:` |
| `TRAINING_AD_ENABLED` | `AD_ENABLED` | AD 管理登入開關 |
| `TRAINING_AD_SERVER_URI` | `AD_SERVER_URI` | 例 `ldap://10.9.82.28:389` |
| `TRAINING_AD_BASE_DN` | `AD_BASE_DN` | 例 `DC=yourco,DC=com` |
| `TRAINING_AD_DOMAIN` | `AD_DOMAIN` | 例 `yourco.com` |
| `TRAINING_AD_ADMIN_GROUP` | `AD_ADMIN_GROUP` | 預設 `IT Admins` |

完整列表與範例值見 `deploy/.env.example` 與 [NAS與路徑跨平台慣例](NAS與路徑跨平台慣例.md) §5.4.1。

#### 5.3.2 NAS 最小範例

```env
TRAINING_SMB_SERVER=10.9.82.22
TRAINING_SMB_SHARE=CrownWork
TRAINING_SMB_AUTH_DOMAIN=yourco.com
TRAINING_MATERIALS_ROOT=教育訓練教材及考卷/materials
TRAINING_BACKUP_ROOT=教育訓練教材及考卷/backups
TRAINING_EXAM_SMB_USERNAME=exam_service
TRAINING_EXAM_SMB_PASSWORD=your_password_or_enc
```

#### 5.3.3 AD 最小範例

```env
TRAINING_JWT_SECRET_KEY=<openssl rand -hex 32>
TRAINING_AD_ENABLED=true
TRAINING_AD_SERVER_URI=ldap://10.9.82.28:389
TRAINING_AD_USE_SSL=false
TRAINING_AD_BASE_DN=DC=yourco,DC=com
TRAINING_AD_DOMAIN=yourco.com
TRAINING_AD_ADMIN_GROUP=IT Admins
```

### 5.4 變更設定後必做：recreate

```bash
cd /opt/apps/enterprise-portal/deploy
docker compose up -d --force-recreate training-backend
```

僅 `docker compose restart` **不會**重新載入已變更的 `deploy/.env`（若 compose 未變更）。

### 5.5 SMTP／Fernet 密文（Email OTP）

```bash
cd /opt/apps/enterprise-portal/deploy

# 1. 產生金鑰 → 寫入 TRAINING_CREDENTIAL_SECRET
docker compose exec training-backend python -c \
  "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# 2. recreate 後加密 SMTP 密碼
docker compose up -d --force-recreate training-backend
docker compose exec training-backend python scripts/encrypt_env_secret.py "SMTP明文密碼"
# → 將輸出貼至 TRAINING_SMTP_PASSWORD=enc:...

# 3. 再次 recreate
docker compose up -d --force-recreate training-backend
```

詳見 [MIGRATION_GUIDE](資料庫遷移/MIGRATION_GUIDE.md)「SMTP 密碼密件化」小節。

### 5.6 PDF 繁中字型

`Dockerfile.backend` 已安裝：

- `fonts-wqy-microhei` → `/usr/share/fonts/truetype/wqy/wqy-microhei.ttc`
- `fonts-noto-cjk` → `/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc`

對應 `register_chinese_fonts()`。若 PDF 中文亂碼，確認映像未刪除字型套件並重建後端映像。

| 輸出類型 | 字型來源 |
|----------|----------|
| list PDF（後端 ReportLab） | **容器內**字型 |
| individual HTML 列印 | **使用者瀏覽器** OS 字型 |

### 5.7 常見故障對照

| 現象 | 根因 | 處理 |
|------|------|------|
| NAS 共享尚未設定 | 只改 `backend/.env` 或未設 `TRAINING_SMB_*` | 改 `deploy/.env` + recreate |
| AD 整合未啟用 | 未設 `TRAINING_AD_*` | 補 AD 區塊 + recreate |
| break-glass 403 未設定密碼 | 未跑 AD 遷移／未設 `INITIAL_ADMIN_PASSWORD` | 見 MIGRATION_GUIDE §AD |
| `encrypt_env_secret` 無金鑰 | 金鑰只在主機 `backend/.env` | 改 `TRAINING_CREDENTIAL_SECRET` |
| NAS `0xc0000234` | AD 帳號鎖定 | 請 IT 解鎖 |
| 改 `.env` 無效 | 未 force-recreate | §5.4 |
| 考試中心／成績中心空白 | 未跑 `add_exam_retake_authorization.py`；`retake_authorized` 欄位缺失 | 見 MIGRATION_GUIDE「已通過授權重考」 |
| 遷移報 `.venv/bin/python3` 不存在 | 誤用本機指令於 Docker 主機 | `docker compose exec training-backend python migrations/...` |
| 已考過仍要求報到 | `attendance_records` 缺列（舊資料） | 個人點「立即報到」；維運跑 `backfill_attendance_from_exam.py` |

本機開發類似問題見 [本機開發疑難排解](本機開發疑難排解.md) §5.10。
