## 一、目的與適用對象

本文件說明「線上教育訓練系統」的**使用情境、操作步驟與注意事項**，供下列對象使用：

- **系統管理員**：單位、分類、人員、角色、權限、功能清單設定
- **訓練／計畫承辦**：訓練計畫建立與維護、考卷出題、題庫匯入、報到與 QRcode
- **一般使用者**：登入、報到、應考、查詢個人成績與學習分析

詳細架構與技術說明請見 [專案系統架構分析](專案系統架構分析.md)；快速啟動與目錄說明請見 [文件中心 README](../README.md)。

## 二、系統入口與登入

### 2.1 存取網址

- **本機開發**：`http://localhost:5173`（或 Vite 顯示之位址）
- **區網**：`http://<伺服器IP>:5173`（依實際部署設定）

### 2.2 登入方式

| 方式 | 說明 |
|------|------|
| **一般登入** | 輸入員工編號 → 取得圖形驗證碼 → 輸入驗證碼 → 登入 |
| **QRcode 登入** | 由管理員產生登入 QRcode（固定登入頁 URL），掃描後仍須輸入員工編號與驗證碼 |
| **AD 管理登入** | IT Admins 群組成員於登入頁「管理登入」分頁，以 AD 帳密登入（JIT 自動建檔） |
| **緊急登入（break-glass）** | AD 不可用時，本地 `admin` 帳號於「緊急登入」分頁；須先完成 DB 遷移設定密碼 |
| **Email OTP 備援** | AD 全斷且曾 AD 登入並有 email 時，可請求 Email OTP（見 AD 技術設計） |

**注意**：  
- 員工編號需已於「人員管理」中建立；若為首次使用，可能需透過「註冊」或由管理員先行建檔。  
- 驗證碼為 4 位數字，逾時或錯誤需重新取得。

### 2.3 註冊（若開放）

於登入頁選擇「註冊」→ 選擇部門 → 輸入員工編號與姓名等必填欄位 → 送出。註冊後通常需由管理員指派角色與單位，方能使用完整功能。

## 三、管理後台使用情境

### 3.1 系統管理

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **單位管理** | 部門/單位的新增、修改、刪除、成員檢視與調整 | 新增單位、將使用者指派至單位、查看成員 |
| **分類管理** | 訓練課程「大項目／細項目」階層 | 新增大項目、在項目下新增細項目、供訓練計畫選用 |
| **教材主檔** | 教材類型、允許副檔名（pdf、mp4 等）集中維護 | 系統管理 → 教材主檔；上傳驗證讀取 DB 主檔 |
| **人員管理** | 使用者之單位、角色、職務、狀態 | 查詢、編輯單位/角色/職務、停用/啟用帳號、分頁與排序 |
| **職務管理** | 職務名稱維護，供人員指派 | 新增、編輯、刪除職務 |
| **角色管理** | 角色名稱、成員、權限；卡片人數僅計**在職** | 新增角色、為角色指派功能權限；成員清單可依**員工編號／姓名／部門**搜尋並分頁 |
| **權限管理** | 角色可存取的「功能清單」 | 依角色勾選/取消功能，控制選單與按鈕顯示 |
| **角色部門權限** | 角色可查視的部門資料範圍 | 設定全域／部門／個人範圍及部門明細 |
| **QRcode 管理** | 產生登入用 QRcode（方案 A） | 產生、複製登入頁連結 |
| **排程備份** | SQLite 資料庫排程 ZIP 備份至 NAS | 設定頻率、保留份數、備份 NAS 帳密；可立即備份 |
| **功能清單** | 系統功能樹（選單與按鈕） | 維護功能代碼、名稱、父子關係、路由路徑（通常由開發/管理員操作） |

**權限**：以上功能依角色權限顯示；未具權限之選單或按鈕不會出現。

### 3.2 訓練計畫管理

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **列表與頁籤** | 「正在進行中」「已過期」「已封存」三個頁籤 | 切換頁籤、依年份/開課單位/分類篩選、關鍵字搜尋 |
| **新增／編輯計畫** | 細項目、開課單位、訓練日期、結束日期、及格分數、計時與時間限制、應到人數 | 選擇受課單位（部門）、選擇個人授課對象、儲存 |
| **封存／取消封存** | 將計畫標記為已封存或恢復 | 在列表中依計畫操作「…」→「封存計畫」或「取消封存」，需確認 |
| **刪除計畫** | 刪除訓練計畫（慎用） | 「…」→「刪除計畫」→ 確認 |
| **報到統計** | 應到/實到、報到名單、未報到名單；名單可搜尋與分頁 | 點選「報到統計」→ 以應到／實到等卡片篩選；名單可依**員工編號／姓名／部門**搜尋；長名單分頁瀏覽 |
| **產生 QRcode** | 產生本計畫報到用 QRcode 與連結；同一組 QR 供上課前報到與考試時報到重複使用 | 列表操作欄點「產生 QRcode」→ Modal 顯示 QR 圖與可複製連結 |
| **計畫教材區** | 編輯計畫 Modal 右側（桌面雙欄）；管理本計畫綁定之教材**套組** | 見 §3.6 |

**注意**：  
- 封存後，該計畫不會出現在「正在進行中」「已過期」，且考試中心、考卷工坊等僅會顯示未封存計畫。  
- 個人授課對象可在編輯計畫時以「員工編號、姓名、部門」搜尋並多選。
- **擁有權（Owner）**：新增計畫時所選「開課單位」即為該計畫的 Owner；僅開課單位或超管／系統管理角色可**編輯、刪除、封存／取消封存、產生報到 QR**；其餘角色僅能檢視（編輯開啟為檢視模式、變更按鈕 disabled、後端亦拒絕 403）。既有資料若無開課單位（`dept_id` 為 NULL）不受此限制。報到統計填寫未到原因／列印清冊不套用 Owner（有對應選單權限即可）。

### 3.3 考卷工坊

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **選擇訓練計畫** | 先選定一筆訓練計畫，再管理其題目與教材；**僅開課單位或超管可寫入**，其餘為檢視模式 | 由下拉或列表選擇計畫（僅顯示未封存） |
| **上傳 TXT 題目** | 依固定格式上傳文字檔，系統解析為單選/多選/是非；實體檔寫入 NAS | 上傳符合格式之 .txt、確認解析結果、必要時在題目列表中修正（非 Owner 停用） |
| **編輯／刪除題目** | 修改題目內容、選項、答案、配分、提示 | 在題目列表點選編輯或刪除、存檔（非 Owner／已封存／已過期按鈕 disabled） |
| **從題庫匯入** | 自題庫挑題加入當前計畫 | 開啟「從題庫匯入」→ 用關鍵字/題型/標籤篩選 → 勾選題目 → 確認匯入（非 Owner 停用） |
| **已匯入考卷檔** | 列出該計畫於 NAS 的 TXT；**NAS 無法連線時仍可讀取題目**，清單區顯示琥珀色提示（非整頁錯誤） | 檢查 SMB／`MATERIALS_ROOT`；預覽仍可用；刪除檔僅 Owner |
| **教材上傳與預覽** | 考卷工坊內舊版區塊；**新教材請用訓練計畫編輯頁教材區或教材庫**（§3.6） | 見 §3.6 |

**題目格式概要**：  
- 單選：`Q:` 題目、`A:`/`B:`… 選項、`ANS: A`、`SCORE: 10`  
- 多選：`ANS: A,C` 或 `ANS: AC`  
- 是非：`ANS: Y` 或 `ANS: N`  

詳細格式與範例請參考後端或專案內題目解析說明。

### 3.4 題庫維護

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **題庫列表** | 查詢題庫內所有題目；列表含**開課單位**欄 | 依關鍵字、題型、標籤、**開課單位**篩選、分頁瀏覽 |
| **編輯／刪除** | 修改題庫題目內容或刪除 | 僅影響題庫本身，不直接刪除已匯入考卷的題目；**編輯與刪除僅開課單位或超管可執行**，其餘角色按鈕 disabled |
| **題型與標籤** | 題型為單選/多選/是非；標籤供分類與篩選 | 篩選時可選題型、輸入標籤關鍵字 |

題庫與考卷題目為不同來源：考卷題目綁定計畫，題庫為共用池；自題庫匯入會將題目複製到該計畫的考卷中。上傳考卷 TXT 匯入題庫的題目，其**開課單位**沿用該考卷所屬訓練計畫的開課單位。

> **部署注意**：若題庫維護頁出現 **HTTP 500** 而本機正常，多半是正式 DB 尚未執行 `add_owner_dept_fields.py`（缺 `question_bank.dept_id`）。見 [MIGRATION_GUIDE](資料庫遷移/MIGRATION_GUIDE.md)「開課單位擁有權」症狀表與 [生產部署指南](生產部署指南.md) §5.7。

### 3.5 成績中心（報表）

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **總覽** | 總考試場次、應考人次、平均分數、及格率等 | 查看數字與圖表 |
| **部門統計** | 各部門應考人次、平均分、及格率、完成率 | 依部門篩選、下鑽至部門成員明細 |
| **計畫統計** | 各訓練計畫之應考與成績摘要 | 依計畫篩選、下鑽至該計畫考生明細 |
| **個人成績查詢** | 查詢指定員工之成績（需權限） | 輸入員工編號或從報表連結進入個人成績 |
| **批次列印** | 跨訓練計畫、跨部門、多員工批次列印（個人成績頁頂層頁籤「批次列印」，與部門績效內層「成績列印」共用同一套邏輯） | 四步精靈：①篩選（計畫狀態／訓練計畫多選／部門多選）②列印類型（「最後一次成績」或「考試歷程」二選一；「成績清單(list)」或「考卷成績單(individual)」二選一，選歷程時 individual 會停用）③載入預覽並勾選人員 ④設定簽名選項後產出——list 模式下載 PDF（單一部門＋計畫）或 ZIP（跨部門或跨計畫，內含各組 PDF）；individual 模式開啟瀏覽器列印視窗，版型與個人「預覽成績單」相同 |
| **PDF 導出** | 產出成績相關 PDF | 依畫面上之導出或列印功能操作 |

**權限**：成績中心相關選單與 API 皆受「報表／成績」權限控制；批次列印之部門選項依使用者權限範圍（全域／部門／個人）過濾，僅能選取範圍內部門。

### 3.6 教材庫與 NAS 傳輸（套組模式）

現行教材以**套組（Set）**為單位：一標題可含多檔，並可綁定 0～N 個訓練計畫；未綁任何可勾選計畫＝**通用教材**。資料表為 `teaching_material_sets`／`files`／`set_plans`（舊單檔表已遷移，見 [MIGRATION_GUIDE](資料庫遷移/MIGRATION_GUIDE.md)）。

| 功能 | 說明 | 常見操作 |
|------|------|----------|
| **教材庫總覽** | 跨計畫搜尋、類型／格式／**開課單位**篩選，套組檢視與檔案檢視皆含**開課單位**欄，分頁 | 導覽列「教材庫」（需 `menu:exam`） |
| **新增／編輯互斥** | 編輯中或新增面板開啟時，鎖定所有列的編輯／刪除與「新增教材套組」按鈕；下載／預覽不受影響 | 開新增會關閉編輯（反之亦然）；取消編輯或關閉新增後自動恢復 |
| **開課單位（Owner）** | 新增套組時**必選**開課單位；由訓練計畫編輯頁新增時自動帶入該計畫開課單位並鎖定唯讀 | 獨立於教材庫新增時可自由選擇；僅開課單位或超管可變更既有套組（含開課單位、綁定、上傳檔） |
| **刪除／編輯限制** | 套組與檔案皆僅開課單位或超管可編輯或刪除 | 非開課單位之編輯／刪除按鈕 disabled；後端亦拒絕（403）；下載／預覽仍可用 |
| **綁定訓練計畫** | 勾選進行中／已過期計畫；不列封存供新勾 | 教材庫寬欄可多欄；計畫側欄多為一列一標題。既有封存綁定以灰標**唯讀保留** |
| **選檔與儲存** | 選檔後可見檔名＋大小；按儲存會一併上傳已選檔並寫中繼資料 | 未選檔僅改標題 → 只更新中繼資料 |
| **大小上限** | 有效上限＝`min(類型、格式、系統硬上限)`；超限須顯示觸發層級，禁止假成功 | 傳輸視窗顯示失敗原因；列表／NAS 不應出現該檔 |
| **NAS 登入** | 每次上傳／下載前驗證個人 NAS／AD 帳密 | 於傳輸對話框輸入帳密 |
| **移除套組內檔案** | 須先經應用內確認窗；檔案檢視頁籤亦可直接刪除單一檔案 | 取消則不刪；確定後軟刪、列表減少 |
| **全選本頁／勾選顯示** | 表頭或「全選本頁」可全選與取消；勾選後標題左藍邊與「已選 N 個檔案」一致 | 批次下載前勾選套組／檔案 |
| **批次下載** | 跨頁勾選後 ZIP | 確認後下載；ZIP 內檔名為原始檔名 |
| **教材主檔** | 類型與允許副檔名、單檔限額 | 系統管理 → 教材主檔 |

## 四、考試中心（應考者）

### 4.1 考試列表

登入後進入「考試中心」或「我的考試」：  
- 列出**尚未開始、進行中、已完成、已過期**之訓練計畫（依系統設計分類）。  
- 僅顯示**未封存**且使用者有權應考之計畫（依受課單位/個人判斷）。

### 4.2 報到（必須先完成才能開考）

業務鐵律：**登入 → 考試中心 → 先報到 → 再開始考試**。未報到時按「開始考試」會被要求先完成報到；系統於後端強制檢查，無法繞過。

**報到雙時機 SOP**（上課前／考試時，共用同一組 QRcode，僅單一報到記錄，不分報到管道）：
1. **上課前**：訓練計畫承辦人於課程開始前，在「訓練計畫管理」列表操作欄點「產生 QRcode」，開放供學員報到。
2. 學員掃描 QRcode 或開啟連結：
   - **未登入**：導向登入頁（`returnTo` 帶原報到連結，僅允許 `/checkin` 開頭路徑），登入成功後**自動導回報到頁並自動完成報到**，無需再手動點按。
   - **已登入**：直接進入報到頁並自動完成報到。
   - 同一計畫同一人僅一筆 `attendance_records`（`UNIQUE(emp_id, plan_id)`）；重掃 QR 為冪等成功，不會重複寫入。
3. **當天請假補報到**：若學員上課當天請假、遲到（QR 已關閉）或當下未報到成功，不需另外處理——待該計畫**考試時**再次使用**同一組 QRcode**（或於考試中心直接開始考試觸發報到）即可補上報到記錄，不影響應考資格。
4. **已報到者**免重複報到：已於上課前完成報到者，進入考試中心開始考試時**不會再要求報到**（同一計畫、同一使用者僅需一次報到記錄）。
5. 亦可在考試中心對進行中之計畫點「**立即報到**」，效果與掃碼報到相同。
6. 已報到者按「開始考試」即可進入作答；**重考時不需再次報到**（沿用第一次報到紀錄）。若該計畫**無考題**（例如僅政策宣導），報到成功頁**不顯示「開始考試」**，僅提供返回首頁。
7. 已通過的學員**不可**自行重考；需由具授權重考權限的主管或稽核人員開放後方可再次作答。
8. 各訓練計畫的及格分由管理員在計畫設定中指定（預設 60 分），考試中心依計畫實際及格分判斷通過與否。

**歷史資料**：T10 報到功能上線前已交卷但無報到列者，由維運執行 `backfill_attendance_from_exam.py` 補登；`checkin_time` 取該員工該計畫之**第一次考試時間**（見 `MIGRATION_GUIDE.md`）。若報到統計出現同一人兩列，執行 `add_attendance_emp_plan_unique.py`（見遷移指南）。

### 4.3 作答與提交

1. 選擇一筆計畫 → 點「開始考試」。  
2. 依題目順序作答（單選/多選/是非），可依介面設計前後翻頁或跳題。  
3. 若有計時，請在時間內完成。  
4. 作答中離開再回來，若系統有「暫存」設計，會還原至上次進度（依實際實作為準）。  
5. 確認無誤後點「提交」→ 系統即時評分 → 顯示成績、通過／未通過印章（PASS/FAIL）與是否及格。  
6. 若**未通過**，系統允許直接重考（無需授權）；若**已通過**，需由主管或具授權重考權限者在成績中心開放後，考試中心才會再度出現「開始考試」入口。

## 五、個人成績與學習分析

詳細 KPI／口徑定義見 [個人成績總覽與學習分析說明](個人成績總覽與學習分析說明.md)。

### 5.1 個人成績總覽

- 路徑通常為「成績中心」→「個人」或側欄「個人成績」。  
- 三頁籤（總覽／歷史記錄／學習分析）共用 **訓練計畫狀態**篩選：`進行中`／`已過期`／`已封存`／`全部`（預設進行中）。  
- 可查看：已完成考試場次、平均分、通過率、最佳/最差成績、總學習時數等。

### 5.2 歷史紀錄與詳情

- **歷史紀錄**：依計畫或時間列出歷次考試與分數；**不含**成績趨勢圖（趨勢見學習分析）。歷程列可顯示授權重考之**授權者**。  
- **成績詳情**：點選單筆紀錄可查看該次考試每題對錯、正確答案與使用者答案；若為授權重考可見授權者／時間／原因。  
- **開放重考**：具成績中心範圍權限者，可對已通過學員授權再次應考（不可對自己授權）。  
- **成績單預覽／列印**：可預覽或列印該次成績單；考試歷程另有精靈式 PDF 列印。

### 5.3 學習分析

- 包含：學習進度、擅長／待加強領域、分類成績、成績趨勢（3／6／12 個月）。  
- 與總覽／歷史相同 `plan_status`；自部門成績「查看個人」進入時會帶入部門頁當下狀態。

## 六、常見問題與注意事項

### 6.1 登入與權限

| 問題 | 建議處理 |
|------|----------|
| 驗證碼一直錯誤 | 確認輸入與圖片一致、英文/數字勿混淆；必要時重新整理取得新驗證碼 |
| 登入後沒有選單或功能很少 | 可能角色權限不足，請管理員在「權限管理」中為該角色勾選所需功能 |
| 忘記員工編號 | 由管理員在「人員管理」中查詢或重設（若支援） |

### 6.2 訓練計畫與考卷

| 問題 | 建議處理 |
|------|----------|
| 找不到剛建的訓練計畫 | 確認頁籤（進行中/已過期/已封存）與篩選條件；確認未遭封存 |
| 題目上傳失敗或解析錯誤 | 檢查 TXT 格式是否符合（Q:/A:/B:/ANS:/SCORE: 等）；編碼建議 UTF-8 |
| 從題庫匯入找不到題目 | 用關鍵字、題型、標籤縮小範圍；確認題庫中已有該題 |

### 6.3 考試與成績

| 問題 | 建議處理 |
|------|----------|
| 考試中心沒有可考的計畫 | 確認登入身分是否在被指派之受課單位或個人名單中；確認計畫未封存、且處於「進行中」日期區間 |
| 提交後看不到成績 | 重新進入「個人成績」或該次考試詳情；若仍無，可能為網路或權限問題，需由管理員/維運檢視紀錄 |
| PDF 或列印異常 | 確認瀏覽器允許彈跳視窗與下載；改用 Chrome/Edge 等常見瀏覽器再試 |

### 6.4 本機開發環境

| 問題 | 建議處理 |
|------|----------|
| `npm install` / `npm run dev` 報 `ENOENT … uv_cwd` | **新開 terminal**，以絕對路徑 `cd` 至 `frontend/` 再執行；勿沿用舊 session |
| 前端可開但登入／API 全失敗，terminal 有 `proxy error ECONNREFUSED` | 後端未啟動；另開 terminal 啟動 Uvicorn（port 8000），確認 `http://localhost:8000/docs` |
| 後端 `No module named 'encodings'` 或 `sys.prefix = '/install'` | `.venv` 損壞；`rm -rf backend/.venv` 後以 `uv venv .venv --python 3.12` 重建並重裝依賴 |
| `python3 -m venv` 的 `ensurepip` 失敗 | 先刪除舊 `.venv`，勿在損壞 venv 上覆蓋重建 |
| `.venv/bin/python3` 不存在 | 尚未建立 venv 或 symlink 斷裂；執行 README「0. 建立虛擬環境」 |

詳見 [本機開發疑難排解](本機開發疑難排解.md)。

### 6.5 資料與系統

- **資料備份**：除手動複製 `education_training.db` 外，具權限管理員可使用「系統管理 → 排程備份」設定 NAS 自動備份。主機搬遷／災備完整還原（預設共用同一 NAS）見 [主機搬遷與災備還原SOP](主機搬遷與災備還原SOP.md)；Docker 遷移見 [MIGRATION_GUIDE](資料庫遷移/MIGRATION_GUIDE.md)。  
- **瀏覽器**：建議使用 Chrome、Edge、Safari、Firefox 等最新版本，並允許 JavaScript 與 Cookie。  
- **螢幕與裝置**：支援手機、平板、桌機；若為行動裝置，請在穩定網路下操作以利提交與載入。
