# 教育訓練系統問題修正計劃（2026-03-16）
## 一、目的
整理並修正目前教育訓練系統在訓練計劃、考試中心、考卷工坊與成績中心的 1–11 項問題。  
先在本機開發環境完成行為釐清與修正，再將穩定版本部署至 Docker 正式環境。
## 二、問題清單與釐清
### 類別 A：後台功能與權限
- **問題 1：停用「功能選單管理」功能**
  - 最終決策：這個功能 **不再開發**，僅保留既有程式碼與資料做備查。
  - 預期行為：
    - 前端：在所有後台選單與入口中，**完全隱藏「功能選單管理」**。
    - 後端：若仍有遺留 API 被呼叫，一律回傳「功能停用」相關錯誤（例如 HTTP 403 + 統一錯誤碼）。
  - 影響範圍：
    - 後台選單組件（admin 區域）。
    - 相關後端 router / 權限設定（僅需封鎖，不再擴充）。
### 類別 B：考試中心與訓練計劃可見性（問題 2, 3, 4, 9）
- **問題 2：訓練計劃中有 4 個正在進行中的計劃，但考試中心看不到**
  - 狀況：訓練計畫管理頁可以看到 4 筆「進行中」，考試中心列表完全沒有。
  - 影響範圍：
    - 後端 `backend/app/routers/exam_center.py` 中，取得「可應考訓練計劃」的查詢邏輯。
    - 前端考試中心頁面 API 呼叫與列表綁定。
  - 可能問題：
    - 狀態欄位或日期區間（開始/結束日）判斷不正確。
    - 受課對象條件過度嚴格，導致符合條件的訓練計劃被排除。
- **問題 3：2026 老人計劃係全員訓練，但任何帳號在考試中心都看不到**
  - 狀況：此訓練計畫設計上應是對「所有人」開放。
  - 影響範圍：
    - DB 中表示「全員適用」的欄位或資料結構（例如 `is_global` 或特定受課單位設定）。
    - `exam_center.py` 內查詢是否有處理「全員適用」的特殊條件。
  - 可能問題：
    - 目前程式把「全員適用」解讀成「沒有指派受課對象」，反而視為沒人可看。
- **問題 4：Admin 帳號在考試中心應可看到所有訓練計劃**
  - 狀況：目前 Admin 帳號只看到部分計劃，或行為與一般使用者一樣。
  - 預期待定：
    - 一般使用者：只看到自己相關的訓練計劃（包含全員適用）。
    - Admin：看到 **所有訓練計劃**，不受受課對象限制。
  - 影響範圍：
    - 使用者角色/權限判斷。
    - `exam_center.py` 查詢邏輯是否有「Admin 模式」分支。
- **問題 9：在「編輯訓練計畫」中，2026 老人計劃受課單位顯示為全部，理論上應全部可見但考試中心看不到**
  - 這是問題 3 的具體化情境：
    - 在「編輯訓練計畫」畫面，**受課單位全選**，代表所有部門均為受課對象。
    - 但實際上任何帳號都看不到該訓練計劃。
  - 重點釐清：
    - 受課單位與實際查詢條件是否一致。
    - 是否存在「資料有打勾，但 mapping table / join 條件漏寫或寫錯」的情況。
### 類別 C：考卷工坊與題庫（問題 5, 6）
- **問題 5：考卷工坊左側訓練計劃列表缺少狀態篩選**
  - 預期行為：
    - 左側訓練計劃列表應提供與「訓練計畫管理」相同的狀態篩選：
      - 「正在進行中」
      - 「已過期」
      - 「已封存」
  - 影響範圍：
    - 前端考卷工坊左欄清單元件（訓練計劃列表）。
    - 後端提供訓練計劃列表的 API 是否接受 `status` 參數，並使用與管理頁一致的定義。
- **問題 6：考卷工坊中所有訓練計劃的題目與歷史題庫都不見了**
  - 狀況：在 DB 中的題目與歷史題庫資料是存在的（你已可手動查表），但畫面顯示空。
  - 影響範圍：
    - 題目與訓練計劃的關聯 table（例如 exam_paper / question_mapping 類型）。
    - 後端考卷工坊題目列表 API 的 join 與 where 條件。
    - 前端題目列表資料綁定與渲染。
  - 可能問題：
    - schema 或欄位名稱修改，查詢仍用舊欄位。
    - 是否有狀態、版本號等額外過濾導致無資料。
### 類別 D：成績中心（問題 7, 8）
- **問題 7：成績中心「部門績效表現」只能點左邊 `>` 展開**
  - 預期 UX：
    - 點擊整列（任何文字區域／空白處）都能展開/收合該部門。
  - 影響範圍：
    - 前端成績中心頁面，「部門績效表現」的列表元件。
    - 目前 click handler 只綁在 `>` 圖示。
- **問題 8：進入個人成績中心後，「查看詳情」無法載入**
  - 狀況：
    - 列表中已有每一筆成績概要（時間、分數等），但按「查看詳情」後，詳細內容無法載入或持續 loading。
  - 影響範圍：
    - 成績詳情 API（後端 router）。
    - 前端詳情 dialog / 頁面呼叫 API 傳遞的參數（成績 id、user id、training_plan id）。
  - 可能問題：
    - 參數錯誤或缺漏。
    - 後端 join / 過濾條件過嚴或 field 名稱不一致，導致查詢結果為空或 500。
### 類別 E：編輯訓練計畫畫面互動（問題 10, 11）
- **問題 10：受課單位勾選與「個人受課對象」要連動**
  - 預期行為：
    - 勾選某個「受課單位」（部門）時，該部門底下的員工在「個人受課對象」區塊中 **自動勾選**。
    - 取消勾選某個「受課單位」時，該部門底下的員工在「個人受課對象」中 **自動取消勾選**。
  - 目標：避免同一訓練計劃在部門與個人兩邊各勾一次，造成認知落差。
- **問題 11：全選／取消全選也要連動個人受課對象**
  - 預期行為：
    - 若在「受課單位」使用「全選」：
      - 所有部門被勾選。
      - 所有部門底下的員工在「個人受課對象」一併勾選。
    - 若使用「取消全選」：
      - 所有部門取消勾選。
      - 「個人受課對象」全數取消。
  - 影響範圍：
    - 前端「編輯訓練計畫」表單的狀態管理與勾選邏輯。
    - 後端對勾選結果的解析（部門／個人對應表）需與前端一致。

  - 前端：`frontend/src/components/admin/ExamStudio.tsx`（狀態篩選 + 鎖定 disable）

     - **正在進行中**：只出現未封存且未過期的計劃
