# 個人成績總覽與學習分析說明

## 1. 目的

說明「成績中心 → 個人成績」底下兩個子頁面——**個人成績總覽**與**個人學習分析**——其畫面上各區塊所顯示的**資訊定義**與**統計計算方式**，供產品、測試與後續維護參考。

## 2. 範圍

- **個人成績總覽**：6 張 KPI 卡片、底下的「成績分析」區塊。
- **個人學習分析**：學習進度、擅長領域、需要加強的領域、各分類成績分析圖、過去 6 個月成績趨勢圖。

## 3. 個人成績總覽

### 3.1 資料來源與 API

- **API**：`GET /api/exam/personal/overview`  
  - 可選查詢參數：`emp_id`（僅 Admin 可傳，用於查看他人成績）。
- **後端邏輯**：`backend/app/routers/exam_center.py` 之 `get_personal_overview`。
- **統計對象**：該使用者所有 **已提交** 的考試記錄（`ExamRecord`，且 `submit_time IS NOT NULL`）。  
  - 同一訓練計畫多次應考會有多筆 `ExamRecord`，每一筆皆納入統計。

### 3.2 六張 KPI 卡片

| 卡片名稱 | 顯示內容 | 統計方式 |
|----------|----------|----------|
| **已完成考試** | 數字 +「場次」 | `completed_count`：符合上述條件的 `ExamRecord` 筆數。每一筆記錄 = 一場已提交的考試。 |
| **平均分數** | 數值（小數 1 位）+「分」 | `average_score` = 所有記錄的 `total_score` 總和 ÷ `completed_count`。若無記錄則為 0。 |
| **通過率** | 百分比（小數 1 位）+「總體」 | `pass_rate` = （`is_passed == True` 的筆數 ÷ `completed_count`）× 100。若無記錄則為 0。 |
| **最佳成績** | 整數 +「分」 | `best_score` = 所有記錄中 `total_score` 的**最大值**。若無記錄則為 0。 |
| **最差成績** | 整數 +「分」 | `worst_score` = 所有記錄中 `total_score` 的**最小值**。若無記錄則為 0。 |
| **總學習時數** | 時/分/秒 +「累積時間」 | `total_study_time`（秒）：對每筆記錄計算 `submit_time - start_time` 的秒數後加總。前端再換算為「X 小時 Y 分」或「X 分 Y 秒」等格式顯示。<br><br>**注意**：此為**實際考試時間**（考生從開始作答到提交的實際耗時），**非**訓練計畫設定的預設時限（`time_limit`）。若考生提前交卷，實際時間會小於預設時間；若考生用滿時限，實際時間會接近或等於預設時間。 |

### 3.3 成績分析（藍色提示區塊）

- **顯示時機**：僅在 `completed_count > 0` 時顯示。
- **內容**：一段文字，彙總：
  - 平均分數、最佳成績、通過率（與上方卡片數值一致，非另外計算）。
  - 依通過率給予一句提示：
    - `pass_rate >= 80`：「表現優異！繼續保持！」
    - `60 <= pass_rate < 80`：「表現良好，還有進步空間！」
    - `pass_rate < 60`：「需要加強學習，加油！」

因此，「成績分析」**沒有額外公式**，純粹是上述總覽數字的文字化與條件文案。

## 4. 個人學習分析

### 4.1 資料來源與 API

- **API**：`GET /api/exam/personal/analysis`  
  - 可選查詢參數：`emp_id`（僅 Admin 可傳）。
- **後端邏輯**：`backend/app/routers/exam_center.py` 之 `get_personal_analysis`。
- **統計對象**：同上，該使用者所有已提交的 `ExamRecord`，並透過 `TrainingPlan` 關聯到 `SubCategory`（訓練計畫所屬**子分類**）。

### 4.2 學習進度

| 項目 | 說明 |
|------|------|
| **已完成** | 該使用者曾經**至少有一筆**已提交考試記錄的**不重複訓練計畫數**（`plan_id` 的 distinct 數）。 |
| **總計畫數** | 受課對象包含該使用者所屬部門的訓練計畫數。即 `TrainingPlan.target_departments` 中有該使用者 `dept_id` 的計畫總數。 |
| **完成率** | （已完成 ÷ 總計畫數）× 100，小數 1 位。若總計畫數為 0 則為 0。 |

前端以進度條與「完成率：X%」呈現。

### 4.3 擅長領域

- **定義**：某個 **SubCategory**（子分類）底下，該使用者所有相關考試的**平均分數 ≥ 80**。
- **計算**：  
  - 依 `SubCategory` 分組，同一分類底下的所有 `ExamRecord.total_score` 取平均。  
  - 若該分類平均 ≥ 80，則列入「擅長領域」。
- **顯示**：分類名稱、該分類下的考試次數、平均分數。  
- 若無任何分類達標，則顯示：「目前沒有擅長的領域（平均分數 ≥ 80）」。

### 4.4 需要加強的領域

- **定義**：某個 **SubCategory** 底下，該使用者所有相關考試的**平均分數 < 60**。
- **計算**：同擅長領域之分組與平均，門檻改為 < 60。
- **顯示**：分類名稱、該分類下的考試次數、平均分數。  
- 若無，則顯示：「目前沒有需要特別加強的領域（平均分數 < 60）」。

### 4.5 各分類成績分析（長條圖）

- **資料**：`category_analysis`。  
  - 每個 **SubCategory** 一筆：`category_id`、`category_name`、`avg_score`（該分類下該使用者所有考試的平均分）、`count`（該分類下的考試次數）。
  - 依 `avg_score` 由高到低排序。
- **圖表**：BarChart，X 軸 = 分類名稱，Y 軸 = 平均分數；一檔「平均分數」長條。  
- 僅在有 `category_analysis` 資料時顯示。

### 4.6 成績趨勢圖（長條圖，支援多種時間範圍）

- **資料**：`trend_data`。  
  - 依使用者選擇的時間範圍（3 個月、6 個月、1 年），以**當月為基準**往前推對應的**曆月**（含當月）。  
  - 每個月一筆：`month`（如 `"2026-01"`）、`year`、`month_num`、`avg_score`、`count`。
- **計算方式**：  
  - `avg_score`：該月內 `submit_time` 落在該月之所有 `ExamRecord` 的 `total_score` 平均。  
  - `count`：該月內上述記錄筆數。  
  - 若該月無考試，則 `avg_score = 0`，`count = 0`。
- **圖表**：BarChart，X 軸 = 月份，Y 軸 對應「平均分數」與「考試次數」兩組長條。  
- **前端呈現**：以**頁籤（Tabs）**方式讓使用者選擇「3 個月」、「6 個月」、「1 年」三種時間範圍，切換時重新呼叫 API 取得對應資料。  
- 僅在 `trend_data` 有資料時顯示。

## 5. 名詞對照

| 名詞 | 說明 |
|------|------|
| ExamRecord | 單次考試紀錄；同一計畫多次應考會有多筆。 |
| TrainingPlan | 訓練計畫；可設定受課單位（target_departments）。 |
| SubCategory | 訓練計畫所屬子分類；用於「依分類」的成績與擅長/弱項統計。 |
| 已完成考試 | 已提交的考試「場次」數，即 `ExamRecord` 筆數（含同計畫多次應考）。 |
| 學習進度之「已完成」 | 至少考過一次的**不重複**訓練計畫數。 |
