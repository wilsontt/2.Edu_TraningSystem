## 專案概述

### 專案名稱
教育訓練教材及線上考卷系統 (Educational Training & Online Exam System)

### 專案類型
企業內部教育訓練管理系統，支援線上考試、成績管理、AI 自動出題等功能。

### 核心特性
- 行動優先設計（響應式介面）
- AI 智慧出題（PDF/PPT 自動解析）
- 免密碼登入（員工編號 + 圖形驗證碼）
- QRcode 快速登入與報到
- RBAC 權限管理
- PDF 成績單導出

### 架構特點

1. **前後端分離**: 前端 React SPA，後端 FastAPI RESTful API
2. **無狀態設計**: 使用 JWT Token 進行認證，無需 Session
3. **單機部署**: SQLite 資料庫，適合中小型企業內部使用
4. **響應式設計**: 前端支援桌面、平板、手機多種裝置
5. **模組化設計**: 後端路由模組化，前端元件化

## 資料庫結構

### 資料庫概覽

- **資料庫類型**: SQLite 3
- **資料庫檔案**: `data/education_training.db`
- **ORM 框架**: SQLAlchemy
- **總表數**: 15 個資料表

#### 1. 基礎資料表

| 表名 | 說明 | 主要欄位 |
|------|------|----------|
| `departments` | 部門表 | id, name |

| `system_functions` | 系統功能表 | id, name, code, path, parent_id |
| `role_functions` | 角色功能關聯表 | role_id, function_id |

| 表名 | 說明 | 主要欄位 |

#### 3. 分類與訓練表

| 表名 | 說明 | 主要欄位 |

### 核心資料模型

#### User (用戶)
```python
class User(Base):
    emp_id: str (PK)          # 員工編號
    name: str                 # 姓名
    dept_id: int (FK)         # 部門 ID
    role_id: int (FK)         # 角色 ID
    status: str               # 狀態 (active/inactive)
    
    # 關聯
    department: Department
    role: Role
    exam_records: List[ExamRecord]
    attendance_records: List[AttendanceRecord]
```

#### TrainingPlan (訓練計畫)
```python
class TrainingPlan(Base):
    id: int (PK)
    sub_category_id: int (FK)    # 子分類 ID
    dept_id: int (FK)             # 開課單位 ID
    title: str                    # 標題
    training_date: date           # 開始日期
    end_date: date (nullable)      # 結束日期
    year: str                     # 年度
    timer_enabled: bool           # 是否啟用計時
    time_limit: int               # 時間限制（分鐘）
    passing_score: int           # 及格分數
    expected_attendance: int (nullable) # 應到人數
    
    # 關聯
    sub_category: SubCategory
    location: Department
    target_departments: List[Department] (M2M)
    questions: List[Question]
    exam_records: List[ExamRecord]
    attendance_records: List[AttendanceRecord]
```

#### ExamRecord (考試記錄)
```python
class ExamRecord(Base):
    id: int (PK)
    emp_id: str (FK)             # 員工編號
    plan_id: int (FK)            # 訓練計畫 ID
    total_score: int             # 總分
    is_passed: bool              # 是否及格
    start_time: datetime         # 開始時間
    submit_time: datetime        # 提交時間
    attempts: int                # 嘗試次數
    
    # 關聯
    user: User
    training_plan: TrainingPlan
    details: List[ExamDetail]
```

## API 設計

### API 基礎資訊

- **Base URL**: `http://<host>:8000/api`
- **認證方式**: JWT Bearer Token
- **資料格式**: JSON
- **CORS**: 開發環境允許所有來源

### API 路由模組

#### 1. 認證模組 (`/api/auth`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/auth/login` | POST | 用戶登入 |
| `/auth/register` | POST | 用戶註冊 |
| `/auth/captcha` | GET | 獲取圖形驗證碼 |
| `/auth/me` | GET | 獲取當前用戶資訊 |
| `/auth/departments` | GET | 獲取部門列表（公開） |
| `/auth/login/qrcode/{token}` | GET | 驗證 QRcode Token |
| `/auth/login/qrcode/{token}` | POST | QRcode 登入 |

#### 2. 管理模組 (`/api/admin`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/admin/departments` | GET/POST/PUT/DELETE | 部門管理 |
| `/admin/users` | GET/PUT | 用戶管理 |
| `/admin/roles` | GET/POST/PUT/DELETE | 角色管理 |
| `/admin/functions` | GET/POST/PUT/DELETE | 功能管理 |
| `/admin/permissions` | GET/PUT | 權限管理 |
| `/admin/categories` | GET/POST/PUT/DELETE | 分類管理 |

#### 3. 訓練計畫模組 (`/api/training`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/training/plans` | GET/POST/PUT/DELETE | 訓練計畫管理 |
| `/training/plans/{id}` | GET | 獲取訓練計畫詳情 |

#### 4. 考試模組 (`/api/exam`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/exam/upload` | POST | 上傳教材並 AI 解析 |
| `/exam/materials/{plan_id}` | GET | 獲取教材列表 |
| `/exam/start/{plan_id}` | POST | 開始考試 |
| `/exam/submit` | POST | 提交答案 |

#### 5. 考試中心模組 (`/api/exam-center`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/exam-center/available` | GET | 獲取可用的考試列表 |
| `/exam-center/checkin/{plan_id}` | POST | 報到 |

#### 6. 題庫模組 (`/api/question-bank`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/question-bank` | GET/POST/PUT/DELETE | 題庫管理 |
| `/question-bank/import` | POST | 匯入題目 |

#### 7. 報表模組 (`/api/report`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/report/personal` | GET | 個人成績報表 |
| `/report/plan/{plan_id}` | GET | 訓練計畫成績報表 |
| `/report/plan/{plan_id}/pdf` | GET | 生成 PDF 成績單 |

#### 8. QRcode 模組 (`/api/qrcode`)

| 端點 | 方法 | 說明 |
|------|------|------|
| `/qrcode/login` | POST | 生成登入 QRcode |
| `/qrcode/checkin/{plan_id}` | POST | 生成報到 QRcode |
| `/qrcode/list` | GET | 獲取 QRcode 列表 |

### API 認證流程

```
1. 用戶登入
   POST /api/auth/login
   Body: { emp_id, captcha_id, answer }
   Response: { access_token, token_type, user }

2. 後續請求
   Header: Authorization: Bearer <access_token>
   
3. Token 驗證
   - 後端使用 get_current_user dependency
   - 驗證 JWT Token
   - 載入用戶資訊與權限
```

## 安全機制

### 認證機制

1. **JWT Token 認證**
   - Token 有效期: 8 小時
   - 演算法: HS256
   - 儲存位置: localStorage

2. **圖形驗證碼**
   - 4 位數字驗證碼
   - 使用 PIL 生成
   - 記憶體暫存（開發環境）

3. **QRcode 登入**
   - Token 有效期: 可設定（預設 24 小時）
   - 多人可使用（不限制使用次數）

### 授權機制

1. **RBAC 權限控制**
   - 角色: Admin, System Admin, User
   - 功能碼: `menu:xxx`, `btn:xxx`
   - 權限檢查: Dependency Injection

2. **路由保護**
   - 前端: React Router 條件渲染
   - 後端: `check_permission` Dependency

### 資料安全

1. **輸入驗證**: Pydantic 模型驗證
2. **SQL 注入防護**: SQLAlchemy ORM
3. **XSS 防護**: React 自動轉義
4. **CORS 設定**: 開發環境允許所有來源（生產環境需限制）
