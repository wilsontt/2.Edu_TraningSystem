> 我是一家公司的IT工程師，我有一個新的小專案要開發，就是我們會有不定期的教育訓練，比如年度的綜合教育訓練，新進員工的教育訓練，以及一些專案的教育訓練...等等，教育訓練完成後，會有課後的考卷，考卷的題目可以用上傳的方式（TXT文字檔），或直接在線上輸入方式，然後同仁就可以在線上直接作答，並且能夠直接在線上完成考卷的評分，日後也可以直接在線上查看成績，有需要時也可以直接列印成績單。

## 🚀 User Stories
整個系統分為5大塊，管理者先在`系統管理`中進行基本的設定：**單位**、**角色**及**權限**管理的設定，然後在`訓練計畫`中新增需要的年度教育訓練、不定期的訓練、新進人員訓練、資訊安全...等訓練的課程計畫。
再到`考卷工坊`，指定訓練計畫進行考題的上傳來產生題目，然後就可以到`考試中心`來進行考試，最後可以到`成績中心`查看個人或單位或全部的成績。
網站登入以4位數字的圖型驗證碼進行安全驗證，第一次登入時以5碼的員工編號進行註冊新增個人帳號。

### User Story 1 系統管理（角色為 Admin 才有的）
- 建立每個部門單位、訓練課程的大項目及細項目分類及角色分類。
- 設計每個角色可存取的的系統功能與選單。
- 人員管理：掃描QRcode，連至網站，登入時以5碼的員工編號進行註冊時新增帳號，選擇自己的單位，預設角色為一般user。
### User Story 2 訓練計劃、考卷工坊
- 建立每一次需要的教育訓練，並設定訓練計劃的類別、時程、受課對象，並設定及格分數等。
- 針對每一個教育訓練項目，上傳受課的題目，以便生成考卷。
### User Story 3 考試中心
- 登入網站進入後預設的考試中心，選擇你需要的訓練課程（考卷），開始進行課程的考試。
- 作答完畢後，送交考卷系統會自動完成考卷批改。
### User Story 4 成績中心
- 具有管理身份的人登入後，可以查看各部門或個人的成績績效，及每個人的教育訓練歷程。
- 個人身份登入，僅可以查詢個人的成績及訓練歷程。
- 可輸出PDF檔。

## 🛠 專案結構與技術堆疊

### 1.1 資料夾結構
採用前後端分離的「模組化單體」架構：
- `backend/`: FastAPI 原始碼、SQLite 資料庫、AI 模組。
- `frontend/`: React + Vite + TypeScript 原始碼。
- `data/materials/{year}/{plan_id}/`: 按年份與計畫分開存放的教材。

### 1.2 技術堆疊
- **Frontend**: React 18, Tailwind CSS, Framer Motion (紅字特效), Lucide React。
- **SVG Engine**: 實作動態日曆產生組件 (藍/奶油色配色)。
- **Backend**: FastAPI, SQLAlchemy (ORM), Pydantic。
- **Database**: SQLite (單機開發與部署首選)。
- **PDF Export**: `reportlab` (Python) 根據 PDF 範本動態生成。


## 功能清單
```
導覽列：
├── 左邊：公司Logo＋系統名稱
├── 中間：
│    ├── 考試中心(查看您的考試任務)
│    ├── 訓練計畫(管理年度訓練課程計畫與開課資訊)
│    ├── 考卷工坊(上傳 TXT 考卷題目，系統將自動解析並匯入題庫)
│    ├── 成績中心(統計各部門的成績績效表現及查看個人的教育訓練歷程)
│    └── 系統管理
│         ├── 單位管理(管理系統內的組織部門與課別清單)
│         ├── 分類管理(管理訓練課程的大項目與細項目分類)
│         ├── 人員管理(檢視與管理系統使用者、角色及單位分配)
│         ├── 角色管理(管理系統角色與權限群組)
│         ├── 權限管理(配置角色可存取的系統功能與選單)
│         └── 功能清單管理(配置系統功能節點與路由結構)
├── 右邊：
│    └── 當下的 日期-時間
│         ├── 使用 SVG 動態生成日曆圖示
│         ├── 上半部分：淺藍色背景，顯示月份英文縮寫（例如：JAN, FEB, MAR）
│         ├── 下半部分：淺黃色/奶油色背景，顯示日期數字（例如：12, 17）
│         ├── 黑色邊框，圓角設計
│         └── 根據傳入的日期自動更新月份和日期
├── 登入者姓名(放在日期-時間的下面)
└── 登出
```
