# 徽章使用規範 (Badge Usage Standard)

![版本](https://img.shields.io/badge/版本-v1.1.0-blue)
![狀態](https://img.shields.io/badge/狀態-穩定版-green)

**文件版本**: v1.1.0  
**建立日期**: 2025-11-28  
**最近修訂**: 2025-01-20  
**適用範圍**: 所有專案的 README.md 文件  
**依據**: 
- 專案憲章（Project Constitution）規範
- SDD（規格驅動開發）工作流程規範
- 各專案設計規範

---

## 1. 目的

本規範定義**所有專案**中 README.md 檔案使用徽章（Badges）的標準，確保：
- **一致性**: 所有專案的 README 使用統一的徽章風格與格式
- **資訊性**: 徽章能快速傳達專案關鍵資訊（版本、狀態、技術堆疊等）
- **可維護性**: 徽章格式易於更新與維護
- **可重用性**: 本規範可被任何專案引用，無需修改即可適用

---

## 2. 規範優先順序

本規範**必須**與各專案的相關規範一併遵循，優先順序如下：

1. **專案憲章** (最高優先)
   - 所有 README 必須使用**正體中文 (zh-TW)** 撰寫
   - 所有 README.md 必須遵循專案憲章的文件格式與結構要求

2. **SDD 工作流程**（若適用）
   - 文件結構需遵循 7 大項規範（目的、範圍、權責...）
   - 文件命名需符合規範

3. **專案設計規範**（若適用）
   - 技術堆疊說明需符合專案設計規範定義
   - UI/UX 相關說明需符合設計規範

4. **本規範 (徽章使用規範)**
   - 徽章格式、排列、顏色等格式規範

---

## 3. 適用範圍

### 3.1 專案根目錄 README.md
- **必須遵循**: 專案憲章規範（正體中文）
- **必須遵循**: 本規範的徽章格式
- **建議遵循**: SDD 工作流程的文件結構規範（若適用）
- **建議遵循**: 專案設計規範的技術堆疊說明（若適用）

### 3.2 子專案/子模組 README.md
- **必須遵循**: 專案憲章規範（正體中文）
- **必須遵循**: 本規範的徽章格式（僅顯示該子專案/子模組的技術）
- **建議遵循**: 專案設計規範的技術堆疊說明（若適用）
- **建議遵循**: SDD 工作流程的文件結構規範（若適用）

### 3.3 文件目錄 README.md (docs/specs/plans/...)
- **必須遵循**: 專案憲章規範（正體中文）
- **建議遵循**: SDD 工作流程的文件結構規範（若適用）
- **可選遵循**: 本規範的徽章格式（若需要）

## 4. 徽章分類

### 4.1 專案狀態徽章 (Project Status Badges)

**必備徽章**:
- 版本號 (Version)
- 專案狀態 (Status)

**格式範例**:
```markdown
![版本](https://img.shields.io/badge/版本-{版本號}-blue)
![狀態](https://img.shields.io/badge/狀態-{狀態}-{顏色})
```

**狀態值**:
- `開發中` (yellow)
- `測試中` (orange)
- `穩定版` (green)
- `維護中` (blue)
- `已棄用` (red)

### 4.2 技術堆疊徽章 (Technology Stack Badges)

**必備徽章**（根目錄 README）:
- 顯示專案使用的主要技術堆疊
- 建議包含：前端框架、後端框架、資料庫、主要程式語言等

**格式範例**:
```markdown
![技術名稱](https://img.shields.io/badge/{技術名稱}-{版本}+-{顏色代碼}?logo={logo名稱}&logoColor={logo顏色})
```

**常見技術徽章範例**:
- 前端：React, Vue, Angular, TypeScript, JavaScript, Tailwind CSS, Vite 等
- 後端：.NET, Node.js, Python, Java, Go 等
- 資料庫：SQL Server, PostgreSQL, MySQL, MongoDB 等
- 其他：Docker, Kubernetes, AWS, Azure 等

**子專案/子模組徽章**:
- 僅顯示該子專案/子模組實際使用的技術
- 例如：前端子專案僅顯示前端技術，後端子專案僅顯示後端技術

### 4.3 品質指標徽章 (Quality Metrics Badges)

**可選徽章**:
- 建置狀態 (Build Status)
- 測試覆蓋率 (Test Coverage)
- 程式碼品質 (Code Quality)
- 文件狀態 (Documentation Status)

**格式範例**:
```markdown
![建置狀態](https://img.shields.io/badge/建置-{狀態}-{顏色})
![測試覆蓋率](https://img.shields.io/badge/測試覆蓋率-{百分比}%25-{顏色})
```

### 4.4 授權與聯絡徽章 (License & Contact Badges)

**格式範例**:
```markdown
![授權](https://img.shields.io/badge/授權-{授權類型}-lightgrey)
```

## 5. 徽章排列規範

### 5.1 排列順序

徽章應按照以下順序排列：

1. **專案狀態徽章**（版本、狀態）
2. **授權徽章**（如有）
3. **技術堆疊徽章**（依重要性排序）
4. **品質指標徽章**（建置、測試、文件）

### 5.2 排列格式

**單行排列**（推薦，適合徽章數量 < 8）:
```markdown
![版本](...) ![狀態](...) ![React](...) ![TypeScript](...)
```

**多行排列**（適合徽章數量 ≥ 8）:
```markdown
![版本](...) ![狀態](...) ![授權](...)

![React](...) ![TypeScript](...) ![.NET](...) ![SQL Server](...)

![建置狀態](...) ![測試覆蓋率](...) ![文件狀態](...)
```

## 6. 顏色規範

### 6.1 狀態顏色

| 狀態 | 顏色代碼 | 說明 |
|------|---------|------|
| 成功/通過 | `success` 或 `green` | 建置成功、測試通過 |
| 警告/進行中 | `yellow` 或 `orange` | 開發中、測試中 |
| 錯誤/失敗 | `red` | 建置失敗、測試失敗 |
| 資訊 | `blue` | 版本號、文件狀態 |
| 中性 | `lightgrey` | 授權、待定項目 |

### 6.2 技術堆疊顏色

使用各技術的官方品牌色，常見技術顏色範例：
- React: `61DAFB`
- TypeScript: `3178C6`
- .NET: `512BD4`
- SQL Server: `CC2927`
- Tailwind CSS: `38B2AC`
- Vite: `646CFF`
- Vue.js: `4FC08D`
- Angular: `DD0031`
- Node.js: `339933`
- Python: `3776AB`
- Java: `ED8B00`
- PostgreSQL: `336791`
- MongoDB: `47A248`

**注意**: 若技術無官方品牌色，可使用 `blue`、`green` 等通用顏色，或參考 [Shields.io](https://shields.io/) 的建議顏色。

## 7. 動態徽章 (Dynamic Badges)

### 7.1 CI/CD 建置狀態

如果專案有 CI/CD 流程，可使用動態徽章：

```markdown
![建置狀態](https://github.com/{username}/{repo}/workflows/{workflow}/badge.svg)
```

### 7.2 測試覆蓋率

如果使用測試覆蓋率工具（如 Codecov）：

```markdown
![測試覆蓋率](https://codecov.io/gh/{username}/{repo}/branch/{branch}/graph/badge.svg)
```

## 8. 檔案位置規範

### 8.1 專案根目錄 README.md

**必備徽章**:
- 版本號
- 專案狀態
- 主要技術堆疊（依專案實際使用的技術）

**可選徽章**:
- 授權
- 建置狀態
- 測試覆蓋率
- 文件狀態

### 8.2 子專案/子模組 README.md

**原則**:
- 僅顯示該子專案/子模組實際使用的技術
- 版本號與狀態徽章可選（若與根目錄不同則建議顯示）

**範例**:
- 前端子專案：顯示前端相關技術（框架、語言、工具等）
- 後端子專案：顯示後端相關技術（框架、語言、資料庫等）
- 文件子專案：可選顯示文件工具相關技術

---

## 9. 更新規範

### 9.1 版本號更新

當專案版本更新時，**必須**同步更新：
1. README.md 中的版本徽章
2. `package.json` 或 `.csproj` 中的版本號
3. 相關規格文件中的版本號

### 9.2 狀態更新

專案狀態變更時（如：從「開發中」改為「穩定版」），**必須**更新狀態徽章。

## 12. 檢查清單

建立或更新 README.md 時，請確認：

- [ ] 版本號徽章已更新
- [ ] 專案狀態徽章正確
- [ ] 技術堆疊徽章與實際使用的版本一致
- [ ] 徽章排列符合規範（狀態 → 授權 → 技術 → 品質指標）
- [ ] 顏色使用符合規範（狀態顏色、技術品牌色）
- [ ] 所有徽章連結可正常顯示
- [ ] 文件內容使用正體中文 (zh-TW)
- [ ] 文件結構符合專案憲章規範（若適用）

## 13. 如何引用本規範

### 13.1 在其他專案中引用

在其他專案的 README.md 或文件開頭，可加入以下引用說明：

```markdown
**徽章使用規範**: 本專案遵循 [徽章使用規範](../0.standards/徽章使用規範.md) (v1.1.0)
```

或使用相對路徑引用：

```markdown
**徽章使用規範**: 本專案遵循徽章使用規範（請參考專案標準文件）
```

### 13.2 規範版本管理

- 本規範為通用規範，適用於所有專案
- 各專案可依需求調整技術堆疊徽章，但需遵循本規範的格式與排列原則
- 若規範有重大更新，建議同步更新所有專案的 README.md
