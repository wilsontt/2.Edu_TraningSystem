**問題**：SQLAlchemy 2.0 的 `case()` 語法錯誤

**修復方案**：
- ✅ 改用 Python 邏輯計算及格數，避免在 SQL 查詢中使用 `case()`
- ✅ 先查詢所有記錄，然後在 Python 中計算及格數（第 596-603 行）
- ✅ 添加註解說明修復原因

**修改檔案**：`backend/app/routers/report.py` (第 570-631 行)

**注意**：此修復方式會增加記憶體使用，但避免了 SQLAlchemy 2.0 的語法問題。如果資料量很大，後續可以考慮優化。

**修復狀態**：✅ **已完成並驗證**
