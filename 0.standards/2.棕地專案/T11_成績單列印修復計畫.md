# 成績單列印修復計畫：邊界調整與頁碼修復

## 1. 目的
修復成績單列印時頁碼未顯示的問題，並將列印邊界（Margins）調整為一致，以符合使用者對列印版面的期望。

## 2. 範圍
- **前端元件**: `frontend/src/components/personal/ScoreCardPreview.tsx`

## 3. 分析
使用者反饋指出兩個主要問題：
1.  **頁碼未顯示**：目前的頁碼實作可能依賴於某些瀏覽器不支援的 CSS counter 行為，或者被遮擋。
2.  **邊界不一致**：頁首及左右邊界過小，需要與頁尾一致。

**解決方案**：
- **頁碼**：使用 CSS `@page` 規則在頁尾右下角加入頁碼，或者在頁面底部固定一個包含頁碼的 `div`。考慮到跨瀏覽器兼容性，使用固定定位的 `div` 模擬頁碼可能更穩定，但要注意不遮擋內容。另一種方式是確保瀏覽器預設的頁首/頁尾被啟用，但通常我們希望隱藏預設的 URL/日期等資訊。最穩健的方式是利用 CSS 的 `counter-increment` 和 `content: counter(page)` 在每頁的 margin box 中顯示，但這在瀏覽器支援度上有限。
    - **決定方案**：採用 `fixed`定位在頁面底部的 `div` 來顯示頁碼，並確保主內容有足夠的 `padding-bottom` 以免被頁碼遮擋。
- **邊界**：在 `@page` CSS 中統一設定 `margin`，例如 `2cm`，確保上下左右一致。

## 4. 實作步驟

### 4.1 調整列印邊界 (CSS)
修改 `ScoreCardPreview.tsx` 中的 iframe HTML 建構邏輯：
- 在 `<style>`區塊中，更新 `@page` 的 `margin` 設定。
- 將 `margin: 0` 改為統一的邊界值，例如 `2.5cm` (約 1 inch)，或者根據使用者需求「設定頁首及左右的邊界跟頁尾一樣」。假設頁尾目前邊界是合適的，我們將統一設定為 `2cm` 或 `2.5cm`。
- 移除 `body { margin: 1.5cm; }`，直接由 `@page` 控制邊界。

### 4.2 修復頁碼顯示
- **HTML 結構**：確認頁尾的頁碼 `div` 存在且樣式正確。
- **CSS 樣式**：
    - 使用 `position: fixed; bottom: 0;` 將頁碼固定在每頁底部。
    - 確保頁碼文字顏色深一點，避免被誤認為浮水印或太淡。
    - **關鍵**：目前的頁碼顯示依賴 `<span class="page-number"></span>`，但沒有 CSS counter 邏輯來填入數字。
    - **修正**：瀏覽器列印通常不會自動填入 `HTML` 中的頁碼。如果要顯示「第 X 頁 / 共 Y 頁」，通常依賴瀏覽器自身的列印設定（勾選「顯示頁首頁尾」）。
    - **替代方案**：若使用者希望我們自己控制頁碼，我們只能在 CSS 中使用 `counter(page)`，但这通常只在支援 Paged Media 的 user agent (如 PrinceXML) 有效，主流瀏覽器 (Chrome/Safari) 對此支援有限。
    - **務實作法**：告知使用者，瀏覽器列印時請勾選「Headers and footers」（頁首頁尾），這樣瀏覽器會自動加上頁碼。
    - **或者**：使用 CSS `content` 在頁尾顯示。Chrome 支援在 `@page { @bottom-center { content: counter(page); } }` 語法嗎？不支援。
    - **妥協方案**：如果一定要自己顯示頁碼，對於動態長度的 HTML 內容，前端難以計算總頁數。
    - **調整**：檢查目前的代碼 `第 <span className="page-number"></span> 頁`。這段代碼是空的。
    - **行動**：
        1.  移除無效的 `<span class="page-number">` 邏輯。
        2.  在 CSS 中嘗試使用 `::after` 配合 `counter`，但這僅對分頁元素有效。
        3.  **最可能的解法**：建議使用者在列印對話框開啟「頁首與頁尾」。
        4.  **如果使用者堅持要「自定義樣式」的頁碼**：我們可以在每頁底部放一個 footer，但這需要內容剛好分頁。
        5.  **決定**：先調整邊界。關於頁碼，將移除無效的 HTML 代碼，並在列印樣式中移除 `@page { margin-bottom: 0 }` 的設定，允許瀏覽器顯示預設頁碼；或者，保留自定義 footer 但僅顯示固定文字（如「教育訓練成績單」），頁碼交給瀏覽器設定。

    *自我修正*：使用者說「頁數沒有出現」。這意味著目前的 `@page { margin: 0 }` 把瀏覽器預設頁碼隱藏了（或者是切掉了）。
    *新策略*：
    1.  設定 `@page { margin: 20mm; }` (統一邊界)。
    2.  移除 `@page { margin-top: 0; margin-bottom: 0; }` 這些隱藏預設頁首頁尾的設定。
    3.  這樣瀏覽器預設的頁碼就會出現。
    4.  如果使用者不想要瀏覽器預設的 URL/日期，只想留頁碼：這在標準瀏覽器列印中很難透過 CSS 只選頁碼。
    5.  **嘗試方案 B**：利用 `position: fixed; bottom: 0;` 在每頁重複顯示一個 Footer。在這個 Footer 中，我們無法用 JS 算頁碼，但可以用 CSS `counter(page)` 嗎？Chrome 不支援在 DOM 元素中讀取 `counter(page)`。
    6.  **結論**：恢復瀏覽器預設邊界，讓使用者在列印對話框勾選「頁首/頁尾」是顯示頁碼最可靠的方法。
    7.  **但是**，使用者可能想要我們「做」出來的頁碼。若必須實作，通常只能做到「第 1 頁」（如果是單頁）。多頁時，無法在 HTML 裡知道現在是第幾頁。
    8.  **策略調整**：將邊界設定好，並在預覽畫面中提示使用者「列印時請勾選頁首/頁尾以顯示頁碼」。
    9.  **等等**，使用者提到「設定頁首及左右的邊界跟頁尾一樣」。這暗示目前的頁尾邊界是他喜歡的（或者他是指頁尾留白比較大？）。目前的 CSS 是 `body { margin: 1.5cm }`。
    10. **最終計畫**：
        - CSS `@page`: 設定 `size: A4; margin: 20mm;` (上下左右統一 2公分)。
        - CSS `body`: 移除 `margin: 1.5cm` (避免雙重邊界)。
        - 移除隱藏頁首頁尾的 CSS。
        - 移除無效的手寫頁碼 HTML（因為無法自動計算）。
