# T6.1-T6.3: 系統管理與報表實作計畫

## 目標
1.  **T6.3 系統功能管理**: 允許 Admin 維護系統選單結構。
2.  **T6.1 統計報表**: 顯示各部門/計畫的考試成績統計與及格率。
3.  **T6.2 PDF 導出**: 產生成績單 PDF 文件。

## 1. 後端實作 (Backend)

### A. 系統功能管理 (System Function Management)
- **Endpoints**:
    - `POST /admin/functions`: 新增功能節點
    - `PUT /admin/functions/{id}`: 修改功能 (名稱, code, path)
    - `DELETE /admin/functions/{id}`: 刪除功能 (需檢查是否有子節點或已被角色引用)
- **Logic**: 維護 `SystemFunction` table。

### B. 測試資料生成 (Data Seeding)
- **Script**: `seed_exam_data.py`
- **Purpose**: 生成一些假的 `ExamRecord` 與 `ExamDetail`，以便開發報表功能。
- **Content**: 隨機為現有 Users 生成針對現有 TrainingPlans 的考試紀錄。

### C. 統計報表 API (Statistics)
- **Endpoints**:
    - `GET /admin/reports/overview`: 全公司總體數據 (總考次, 平均分, 及格率)
    - `GET /admin/reports/department`: 各部門統計列表
    - `GET /admin/reports/plan`: 各訓練計畫統計列表
- **Query**: 使用 SQL aggregation (`func.avg`, `func.count`) 計算統計值。

### D. PDF 導出 (PDF Export)
- **Endpoints**:
    - `GET /admin/reports/export/pdf`: 導出總表或特定計畫的成績單
- **Library**: `reportlab` (需安裝)
- **Output**: StreamingResponse (application/pdf)

## 2. 前端實作 (Frontend)

### A. FunctionManager.tsx (T6.3)
- **UI**: 樹狀結構列表 (參考 PermissionManager 的呈現方式，但加上 CRUD 按鈕)。
- **Features**: 新增節點、編輯節點、刪除節點。

### B. ReportDashboard.tsx (T6.1, T6.2)
- **UI**: 
    - 頂部 KPIs 卡片 (總次數、及格率)。
    - Tabs 切換「部門統計」、「計畫統計」。
    - 漂亮的 Data Table 展示統計數據。
    - 「匯出 PDF」按鈕。

## 3. 驗證計畫
1.  執行 `seed_exam_data.py` 產生數據。
2.  驗證 `FunctionManager` CRUD 操作。
3.  驗證 `ReportDashboard` 數據是否正確顯示。
4.  點擊匯出 PDF，確認檔案下載且格式正確。
