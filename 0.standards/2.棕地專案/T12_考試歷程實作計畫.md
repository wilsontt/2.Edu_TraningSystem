# 成績單列印擴充計畫：加入考試歷程清單

## 1. 目的
在成績單列印的首頁（封面頁），新增「考試歷程」區塊，列出該名考生針對該訓練計畫的所有歷史考試紀錄（時間與成績）。這能完整呈現考生的學習軌跡，特別是對於多次重考的情況。

## 2. 範圍
- **後端 API**:
    - 確認是否有現成的 API 可取得單一訓練計畫的完整考試歷史。
    - 目前 `ExamRecord` 僅儲存「最後一次」的成績與次數 (`attempts`)，**沒有儲存每次嘗試的詳細歷史**。
    - **重大發現**：根據目前的 `models.py`，`ExamRecord` 是一對一（User-Plan）的，且重考時會覆蓋舊資料（Update existing record）。這意味著**歷史資料在資料庫層級並不存在**。
    - 解決方案：雖然無法回溯過去的歷史，但我們可以從現在開始記錄，或者修改需求。
    - **與使用者確認的假設**：使用者提到「每個考生，都至少會有一次的考試歷程...有人重考 2次、3次...」。這暗示使用者認為系統有存這些資料。
    - **技術限制**：目前的系統架構 (`ExamRecord` update logic) **確實沒有**保存歷史紀錄。
    - **變通方案**：
        1.  **若必須顯示歷史**：必須先修改後端架構，新增 `ExamHistory` 表，並在每次 `submit_exam` 時寫入。
        2.  **若僅能顯示當前資訊**：目前只能顯示「第 X 次嘗試」，無法列出前 X-1 次的時間與分數。
    - **決策**：鑑於這是「列印功能修復」，而非「系統架構重構」，且使用者語氣堅定（"每個考生...都至少會有一次..."），我必須先告知使用者這個系統限制，或者檢查是否有 log 或其他地方存了歷史。
    - **再次檢查**：是否有 `ExamLog` 或類似的？目前的 search results 沒看到。
    - **假設使用者誤解**：或者使用者是指 `attempts` 這個數字？不，他說「清單式的列出每一次的考試時間來」。
    - **關鍵行動**：我必須先跟使用者確認或建議新增資料表。但在此 Plan 模式下，我應該先規劃「如何實作」，如果發現不可行，則在計畫中指出缺口。
    - **修正計畫**：為了滿足需求，我們必須 **新增 `ExamHistory` 機制**。這是一個 Full-Stack 的修改。

## 3. 分析
使用者需求：
- **位置**：第一頁，訓練計畫名稱與簽名欄之間。
- **內容**：考試歷程清單（日期、成績）。
- **資料來源**：目前後端 **不支援**。`ExamRecord` 只有最後一次的狀態。

**系統架構缺口**：
- `ExamRecord`：記錄「最新」狀態 (Current State)。
- `ExamHistory` (缺失)：記錄「歷次」狀態 (Audit Log / History)。

**解決方案 (Full Stack)**：
1.  **Database**: 新增 `exam_history` 表格 (record_id, exam_record_id, score, is_passed, submit_time)。
2.  **Backend**:
    - 修改 `models.py`：加入 `ExamHistory` model。
    - 修改 `submit_exam`：在更新 `ExamRecord` 前（或後），將當次結果寫入 `exam_history`。
    - 修改 `get_exam_record_detail` API：回傳資料中加入 `history` 列表。
3.  **Frontend**:
    - 修改 `ScoreDetailModal.tsx` 的 type 定義，加入 `history` 欄位。
    - 修改 `ScoreCardPreview.tsx`：在封面頁渲染歷史清單。

## 4. 實作步驟

### 4.1 資料庫與模型 (Backend)
- 新增 `ExamHistory` model：
    ```python
    class ExamHistory(Base):
        __tablename__ = "exam_history"
        id = Column(Integer, primary_key=True)
        record_id = Column(Integer, ForeignKey("exam_records.id"))
        exam_record = relationship("ExamRecord", back_populates="history")
        submit_time = Column(DateTime, default=datetime.now)
        total_score = Column(Integer)
        is_passed = Column(Boolean)
    ```
- 在 `ExamRecord` 加入 relationship: `history = relationship("ExamHistory", back_populates="exam_record")`。

### 4.2 業務邏輯 (Backend)
- 修改 `routers/exam_center.py` 的 `submit_exam`：
    - 每次提交時，建立一筆 `ExamHistory`。
- 修改 `routers/exam_center.py` 的 `get_exam_record_detail`：
    - 查詢並回傳該 record 的所有 history。

### 4.3 前端整合 (Frontend)
- 更新 `ScoreDetail` 介面定義。
- 在 `ScoreCardPreview.tsx` 的封面頁區塊（簽名欄上方），加入一個表格或清單：
    - 標題：考試歷程 / Exam History
    - 內容：
        - 第 1 次: 2026/01/01 - 50分 (未通過)
        - 第 2 次: 2026/01/03 - 80分 (通過)

## 5. 替代方案 (若不想動 DB)
如果不想現在改 DB，目前只能顯示：
- 「目前為第 N 次考試，最後一次考試時間為 YYYY/MM/DD」
- 無法列出前幾次的時間與分數。

**決定**：鑑於使用者的明確需求，我將規劃「新增 ExamHistory」的完整實作。但這需要先執行 DB Migration。若使用者是「現在」就要印舊資料的歷史，那是**不可能的**（因為資料已經遺失）。只能從「程式更新後」的考試開始記錄。

**重要確認**：請使用者確認是否接受「舊的重考紀錄無法回溯，只有新系統上線後的重考才會被記錄」。

- 重考一次 -> 檢查 history 有 2 筆。
- 列印成績單 -> 確認封面頁出現歷程清單。
